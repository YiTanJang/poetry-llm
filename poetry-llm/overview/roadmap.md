# 로드맵

## Phase 0 — 기반 설계 (현재 단계)
**목표:** 프로젝트 설계 완료, 데이터 전략 수립, 모델 후보 선정

- [ ] OKF 번들 초안 작성
- [ ] 데이터 소스 목록 및 저작권 검토
- [ ] 모델 후보 평가 기준 정의
- [ ] 전처리 파이프라인 설계 (특수 토큰 전략)
- [ ] 평가 프레임워크 설계

## Phase 1 — 데이터 수집 및 SFT 학습 데이터 생성
**목표:** 학습 가능한 형태의 시 코퍼스 구축 + SFT 데이터 생성

**데이터 수집:**
- [ ] 저작권 만료 한국시 / 오픈 데이터 수집
- [ ] 스캔 데이터 OCR 및 정제 파이프라인 구축
- [ ] `<행갈이>`, `<연갈이>` 토큰 삽입 스크립트 작성
- [ ] 발음 토크나이저 통합

**SFT 학습 데이터 생성 (씨앗 → 스크래치패드 + 시):**
- [ ] Claude/GPT-4를 이용해 5~7개 페르소나로 (씨앗 + 스크래치패드 + 시) 트레이스 생성
- [ ] 창작 도구 호출 패턴 포함 트레이스 생성 (random_words, news_snippet 등)
- [ ] 페르소나 간 출력 거리 측정 실험 — 3개 페르소나가 실제로 다양성을 만드는지 검증
- [ ] 도구 사용 트레이스 / 비사용 트레이스 비율 결정
- [ ] 시론/시평론 구조화 (Q&A, 설명 데이터 변환)

## Phase 2 — CPT → SFT → DPO 파인튜닝
**목표:** 베이스 모델을 시 생성에 특화된 모델로 학습

**CPT (Continual Pretraining):**
- [ ] 시 코퍼스 + 시론/평론으로 언어 분포 적응
- [ ] 특수 토큰 (`<행갈이>`, `<연갈이>`, `<시작>`, `<끝>`) 학습 검증

**SFT (Supervised Fine-Tuning):**
- [ ] Phase 1에서 생성한 (씨앗 → 스크래치패드 + 시) 데이터로 학습
- [ ] 스크래치패드 + 시 전체 어시스턴트 토큰으로 학습
- [ ] 파일럿: Qwen2.5-10.7B로 먼저 검증, 이후 32B

**DPO (Direct Preference Optimization):**
- [ ] SFT 모델로 3 페르소나 × 씨앗당 시 생성
- [ ] 외부 판정 모델 (Claude/GPT-4)로 승자 선택 → (chosen, rejected) 쌍 구성
- [ ] DPO 학습 (β=0.1, lr=5e-7)
- [ ] ablation: 비평가/독자 에이전트 추가 시 DPO 쌍 품질 변화 측정

## Phase 3 — 생성 파이프라인 구축 및 평가
**목표:** 창작 도구 하네스 + 생성 루프 완성, 품질 측정

**생성 파이프라인:**
- [ ] 창작 도구 하네스 구현 (random_words, news_snippet, dice 등)
- [ ] 씨앗 → 스크래치패드 + 도구 호출 + 시 루프 구현
- [ ] sanity gate 구현 (비어 있지 않음 / 반복 과다 없음 / 3~100행)

**평가:**
- [ ] 자동 지표 구현 (novelty, 구조 정확도)
- [ ] 인간 평가 설계 (블라인드 평가단, 0.50 가중치)
- [ ] LLM 심사 프롬프트 설계 (0.20 가중치)
- [ ] 평가 결과 기반 데이터 큐레이션 개선

## Phase 4 — LoRA 스타일 어댑터 (선택적)
**목표:** DPO 모델에서 스타일별 LoRA 어댑터 학습

- [ ] 스타일 레이블된 데이터 구축 (감각형 / 개념형 / 형식실험형)
- [ ] 각 스타일별 LoRA 학습
- [ ] 추론 시 LoRA 선택 / 혼합 인터페이스 설계
- [ ] temperature × LoRA 조합 실험
