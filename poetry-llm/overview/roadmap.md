# 로드맵

## Phase 0 — 기반 설계 (현재 단계)
**목표:** 프로젝트 설계 완료, 데이터 전략 수립, 모델 후보 선정

- [ ] OKF 번들 초안 작성
- [ ] 데이터 소스 목록 및 저작권 검토
- [ ] 모델 후보 평가 기준 정의
- [ ] 전처리 파이프라인 설계 (특수 토큰 전략)
- [ ] 평가 프레임워크 설계

## Phase 1 — 데이터 수집 및 전처리
**목표:** 학습 가능한 형태의 시 코퍼스 구축

- [ ] 저작권 만료 한국시 / 오픈 데이터 수집
- [ ] 스캔 데이터 OCR 및 정제 파이프라인 구축
- [ ] `<행갈이>`, `<연갈이>` 토큰 삽입 스크립트 작성
- [ ] 발음 토크나이저 통합
- [ ] 시론/시평론 구조화 (Q&A, 설명 데이터 변환)
- [ ] 학습 데이터 생성 (다양한 포맷: 완성, 수정, 창작 노트)

## Phase 2 — 파인튜닝 실험
**목표:** 기반 모델 선정 및 초기 파인튜닝

- [ ] 베이스 모델 선정 (Qwen2.5-32B / LLaMA-3.1-70B 등 후보 비교)
- [ ] 소규모 데이터로 파일럿 파인튜닝
- [ ] 행갈이/연갈이 토큰 학습 검증
- [ ] 창작 노트(CoT) 데이터 파인튜닝
- [ ] 반복 수정 파이프라인 구현

## Phase 3 — 생성 파이프라인 구축
**목표:** 창작 노트 → 시 → 수정의 루프 완성

- [ ] 창작 노트 초안 생성 프롬프트 설계
- [ ] 시 초안 생성 프롬프트 설계
- [ ] 수정 에이전트 설계 (자기 비평 → 재작성)
- [ ] 반복 횟수 / 종료 조건 설계
- [ ] 최종 시 선택 로직

## Phase 4 — 평가 및 개선
**목표:** 생성된 시의 품질 측정 및 모델 개선

- [ ] 자동 평가 지표 구현 (novelty, 구조 정확도)
- [ ] 인간 평가 설계 (블라인드 평가단 모집)
- [ ] 평가 결과 기반 데이터 큐레이션 개선
- [ ] 필요시 추가 파인튜닝
