## 데이터 영역

### Q1. 시론이 시 생성 능력에 실제로 기여하는가?
- **상태**: 🟡 부분 답변 — 강한 간접 근거 확보
- **프로젝트 단계**: Phase 1 — 데이터 수집 및 전처리
- **우선순위**: ★★★★★ (가장 중요 — 시론 수집의 정당성을 내부적으로 검증해야 함)
- **업데이트**: COIG-Writer (arXiv 2510.14763)가 중국어 창의적 글쓰기에서 "과정 데이터(사고 과정 포함 데이터)"가 없는 경우 체계적 창의성 결핍이 발생함을 보임. 시론은 시적 판단의 언어화된 과정 데이터에 해당한다. 시론 없이 훈련 vs. 시론 포함 훈련 비교 ablation이 없으나, 과정 감독(process supervision)의 효과를 지지하는 근거가 분명히 존재한다.
- **남은 과제**: 한국어 시론의 효과를 한국어 현대시 특화 ablation으로 직접 측정. COIG-Writer의 실험 설계를 참조 가능.
- **관련 논문**: [COIG-Writer](https://arxiv.org/abs/2510.14763)

### Q2. 최적 데이터 혼합 비율은?
- **상태**: 🔴 미답 — 실험 필요
- **프로젝트 단계**: Phase 1 — 데이터 수집 및 전처리
- **우선순위**: ★★★★☆
- **업데이트**: 합성 데이터 다양성 연구 (arXiv 2511.01490)에서 데이터 품질 필터링이 혼합 비율보다 중요할 수 있음을 시사. 즉, 혼합 비율 최적화 전에 **각 소스의 품질 기준**을 먼저 정해야 한다.
- **남은 과제**: 현대시 : 시론 : 외국어시 : 보조예술 비율 파일럿 실험. 품질 필터 기준 먼저 설계 필요.
- **관련 논문**: [Synthetic Eggs in Many Baskets](https://arxiv.org/abs/2511.01490)

### Q3. 외국어 시가 한국어 시 생성에 어떻게 기여하는가?
- **상태**: 🔴 미답 — 탐색 필요
- **프로젝트 단계**: Phase 1 — 데이터 수집 및 전처리
- **우선순위**: ★★★☆☆
- **업데이트**: 다국어 모델의 개념적 전이(Cross-lingual Conceptual Transfer) 연구 (arXiv 2406.15267)에 따르면, 고차원적이고 추상적인 이미지(이미저리)는 언어 간 공유되는 시맨틱 공간에서 정렬된다. 이는 외국어 시의 번역본 또는 원문을 함께 학습할 때, 한국어 시의 표현 다양성이 확장될 수 있음을 시사한다.
- **남은 과제**: 일본어 시(하이쿠 포함) 포함 시 한국어 생성 품질 변화 측정. 한자문화권 공유 이미저리의 활성화 패턴 분석.
- **관련 논문**: [Evaluating Diversity in Automatic Poetry Generation](https://arxiv.org/abs/2406.15267)

## 모델 영역

### Q4. 풀 파인튜닝이 정말 LoRA보다 시 생성에 더 유리한가?
- **상태**: 🔴 미답 — 실험 필요
- **프로젝트 단계**: Phase 2 — 파인튜닝 실험
- **우선순위**: ★★★★☆
- **업데이트**: 고품질 도서 파인튜닝이 전문가 수준 글쓰기를 만들어낸다는 근거 (arXiv 2601.18353)는 도메인 특화 전체 가중치 업데이트의 효과를 간접 지지함.
- **남은 과제**: 동일 데이터로 Full FT vs LoRA 비교 실험.
- **관련 논문**: [Can Good Writing Be Generative?](https://arxiv.org/abs/2601.18353)

### Q5. 특수 토큰 초기화 방법이 학습에 영향을 미치는가?
- **상태**: 🔴 미답 — 실험 필요
- **프로젝트 단계**: Phase 0 — 기반 설계
- **우선순위**: ★★★☆☆
- **업데이트**: 학습된 임베딩 공간에서 완전히 새로운 토큰을 무작위 초기화(random initialization)하는 것은 학습 초기 손실 함수의 불안정성을 유발하고 느린 수렴을 야기한다. 특수 토큰을 줄바꿈(`\n`)이나 쉼표(`,`) 등 시의 구조적 분절 기능을 수행하는 기존 토큰의 평균 임베딩 값으로 초기화하는 방안(arXiv 2310.04834)이 유효할 것으로 추정된다.
- **남은 과제**: `<행갈이>`와 `<연갈이>` 특수 토큰을 의미/기능상 유사한 기존 어휘 임베딩의 평균으로 초기화한 경우와 무작위 초기화한 경우의 Perplexity 감소 속도 비교.
- **관련 논문**: [Initializing New Word Embeddings for Language Model Fine-Tuning](https://arxiv.org/abs/2310.04834)

### Q6. 창작 노트가 별도 학습 단계를 필요로 하는가?
- **상태**: 🟡 부분 답변 — COIG-Writer 설계에서 힌트
- **프로젝트 단계**: Phase 2 — 파인튜닝 실험
- **우선순위**: ★★★★☆
- **업데이트**: COIG-Writer는 CoT 데이터를 처음부터 함께 학습하는 방식을 채택하고 효과적임을 보임.
- **남은 과제**: 한국어 시 도메인에서 두 방식 비교.
- **관련 논문**: [COIG-Writer](https://arxiv.org/abs/2510.14763)

## 생성 영역

### Q7. 반복 수정 횟수의 최적값은?
- **상태**: 🔴 미답 — 실험 필요
- **프로젝트 단계**: Phase 3 — 생성 및 평가 파이프라인 구축
- **우선순위**: ★★★★☆
- **업데이트**: LLM의 자기 수정(Self-correction) 연구 (arXiv 2308.03188)에 따르면, 피드백 루프가 3-5회를 초과하여 반복되면 텍스트 품질이 급격히 정체되거나 오히려 초기 미학적 신선함이 거세되는 경향(퇴화)이 발생할 수 있다.
- **남은 과제**: 수정 횟수(1, 2, 3, 5, 10회) 별 인간 평가 + perplexity 추적 실험.
- **관련 논문**: [Is Self-Correction via LMs Ready?](https://arxiv.org/abs/2308.03188)

### Q8. 자기 비평과 외부 비평 모델 중 어느 것이 더 효과적인가?
- **상태**: 🟡 부분 답변 — 외부 비평 우세 근거 확보
- **프로젝트 단계**: Phase 3 — 생성 및 평가 파이프라인 구축
- **우선순위**: ★★★★★ (파이프라인 설계 핵심 결정)
- **결정**: 외부 모델(Claude/GPT-4)을 심사위원으로 사용. 자기 비평은 구조적으로 편향됨 — 자신이 쓴 시를 반복 수정할수록 좋다고 평가하게 된다.
- **업데이트**: LLM Review (arXiv 2601.08003)는 블라인드 동료 평가가 자기 비평보다 창의적 글쓰기 향상에 더 효과적임을 보임.
- **남은 과제**: 한국어 시 도메인에서 직접 비교. 비평 언어(한국어 vs 영어) 영향도 확인 필요.
- **관련 논문**: [LLM Review](https://arxiv.org/abs/2601.08003), [Igniting Creative Writing](https://arxiv.org/abs/2508.21476)

### Q9. 소재 novelty를 어떻게 유도할 것인가?
- **상태**: 🔴 미답 — 설계 필요
- **프로젝트 단계**: Phase 3 — 생성 및 평가 파이프라인 구축
- **우선순위**: ★★★★☆
- **업데이트**: Multi-Novelty (arXiv 2502.12700) 연구에서는 서로 다른 관련이 없거나 희귀한 개념들을 의미적으로 연결하는 프롬프팅 기법이 참신한 비유 생성에 매우 효과적임을 보였다.
- **방향**: 학습 데이터 설계(소재 희귀성 주입) + inference-time 브레인스토밍 병행.
- **관련 논문**: [Multi-Novelty](https://arxiv.org/abs/2502.12700)

## 평가 영역

### Q10. AI 생성시가 어떤 기준에서 인간 시와 가장 다른가?
- **상태**: 🔴 미답 (한국어 시 특화) — 일반 근거만 확보
- **프로젝트 단계**: Phase 3 — 생성 및 평가 파이프라인 구축
- **우선순위**: ★★★★★
- **업데이트**: 문학적 글쓰기 평가 연구 (arXiv 2601.18353)에 따르면, 일반적인 LLM 시는 표면적인 운율과 문법적 정합성은 뛰어나나, 시적 화자의 내면적 일관성(subtext)과 거시적인 은유의 완결성(macro-metaphorical cohesion)에서 인간 시와 뚜렷한 격차를 보인다.
- **남은 과제**: 한국 현대시 전문가 패널을 활용한 블라인드 평가 실험 설계.
- **관련 논문**: [Can Good Writing Be Generative?](https://arxiv.org/abs/2601.18353)

### Q11. Novelty와 미학적 품질의 상관관계는?
- **상태**: 🟡 부분 답변 — 측정 방법론 확보
- **프로젝트 단계**: Phase 3 — 생성 및 평가 파이프라인 구축
- **우선순위**: ★★★★☆
- **업데이트**: **novelty와 미학적 품질은 관련은 있지만 별도 축이다.** 전문가는 novelty 높은 시를, 일반 독자는 coherence 높은 시를 선호할 가능성이 있다.
- **남은 과제**: 한국어 시에서 novelty 점수와 전문가 미학 평가의 상관관계 직접 측정.
- **관련 논문**: [Measuring LLM Novelty](https://arxiv.org/abs/2504.09389), [NoveltyBench](https://arxiv.org/abs/2504.05228)

## 신규 질문 (2026-06-27 추가)

### Q14. 에코챔버 효과를 평가 단계에서 어떻게 방지할 것인가?
- **상태**: 🔴 미답 — 설계 필요
- **프로젝트 단계**: Phase 3 — 생성 및 평가 파이프라인 구축
- **우선순위**: ★★★★☆
- **업데이트**: Tang Poetry (arXiv 2510.15313) 연구에 따르면, 자신이 생성하거나 자신과 유사한 학습 파이프라인을 거친 결과물을 평가할 때 모델 심사위원의 자기 편향(self-bias) 및 에코챔버 효과가 극대화된다.
- **질문**: 평가 모델(LLM-as-a-Judge)이 훈련 데이터와 같은 편향을 공유하지 않도록 하려면?
- **관련 논문**: [Tang Poetry](https://arxiv.org/abs/2510.15313)

### Q15. CrPO(Creative Preference Optimization)를 시 도메인에 직접 적용할 수 있는가?
- **상태**: 🔴 미답 — 설계 필요
- **프로젝트 단계**: Phase 3 — 생성 및 평가 파이프라인 구축
- **우선순위**: ★★★★☆
- **업데이트**: Creative Preference Optimization (arXiv 2505.14442)은 창의적 생성물의 극적인 참신성을 확보하기 위해 전형성(coherence)과 참신성(novelty)의 균형을 맞추는 손실 마진(loss margin) 조정을 제안한다.
- **질문**: 한국 현대시에서 CrPO의 선호 쌍(preference pairs)을 어떻게 구성할 것인가?
- **관련 논문**: [Creative Preference Optimization](https://arxiv.org/abs/2505.14442)

### Q16. 멀티 에이전트 비평이 단일 모델 반복 수정보다 한국어 시에서 더 유효한가?
- **상태**: 🔴 미답 — 실험 필요
- **프로젝트 단계**: Phase 3 — 생성 및 평가 파이프라인 구축
- **우선순위**: ★★★★★
- **업데이트**: Multi-agent Poetry (arXiv 2409.03659) 연구에서는 서로 다른 문체나 역할을 부여받은 에이전트들이 피드백을 공유하고 논쟁할 때, 단일 모델의 단순 반복 수정 대비 은유의 독창성과 구조적 일관성이 크게 향상됨을 밝힘.
- **질문**: 같은 모델의 자기 수정 N회 vs. 다른 페르소나(A=감각형, B=개념형, C=형식실험형) 분리 방식 중 어느 것이 더 유효한가?
- **관련 논문**: [Multi-agent Poetry](https://arxiv.org/abs/2409.03659)

### Q17. 한국어 시에서 인간 전문가의 평가와 일반 독자의 평가는 어떻게 다른가?
- **상태**: 🔴 미답 — 조사 필요
- **프로젝트 단계**: Phase 0 — 기반 설계
- **우선순위**: ★★★★★
- **업데이트**: *Can Good Writing Be Generative?* (arXiv 2601.18353)에서 전문 비평가 집단은 서사적 미학의 해체 및 조어(novel coinage)에 높은 점수를 부여하는 반면, 일반 독자군은 이해 용이성과 감정적 이입을 미학적 완성도로 오인하는 경향이 크다고 설명함.
- **질문**: 이 프로젝트의 평가는 어떤 집단을 대상으로 설계해야 하는가? Cohen's κ와 Krippendorff's α 중 어느 것을 쓸 것인가?
- **관련 논문**: [Can Good Writing Be Generative?](https://arxiv.org/abs/2601.18353)

- **프로젝트 단계**: Phase 1 — 데이터 수집 및 전처리
- **우선순위**: ★★★★☆
- **업데이트**: *Synthetic Eggs in Many Baskets* (arXiv 2511.01490)는 대규모 합성 CoT 데이터 생성 시 인위적인 '망설임(hesitations)'이나 '사고의 실패 후 경로 재수정'과 같은 인지적 노이즈가 누락될 때, 모델이 매우 정형화되고 생기 없는 정답만을 산출하는 '퇴화(degradation)' 현상을 유발함을 보임.
- **배경**: 역추론으로 생성한 창작 노트가 너무 매끄러울 위험이 있어, 실제 창작처럼 망설임과 실패(노이즈)를 인위적으로 주입해야 할지 결정이 필요하다.
- **관련 논문**: [Synthetic Eggs in Many Baskets](https://arxiv.org/abs/2511.01490)

### Q19. 한국어 형태소 경계와 시적 리듬의 관계를 모델이 학습하는가?

- **우선순위**: ★★★☆☆
- **업데이트**: 한국 현대시의 율격(rhythm)은 음절 수(음수율)와 어절/의미 단위의 휴지(음보율)에 기반한다. 서브워드 토크나이저는 형태소 경계를 완벽히 따르지 않아 생성 시 운율 왜곡이 발생할 수 있으며, KoNLPy 등의 사전 형태소 분절 후 특수 기호 처리가 필요할지 검증해야 한다.
- **질문**: 형태소 분석기(KoNLPy 등)를 이용한 전처리가 시 생성 품질에 영향을 미치는가?

### Q20. CreativeBench의 측정 방식이 한국어 시에 적용 가능한가?

- **업데이트**: CreativeBench (arXiv 2603.11863)는 텍스트의 미학적 자산(Metaphor, Novelty, Emotion)을 평가하는 최신 다차원 프레임워크를 제공하나, 한국어 특유의 한자어 조어력과 어미 변화(존칭, 문체)에 따른 뉘앙스 변화를 포착하기 위해서는 한국어 전용 가중치 튜닝이 필수적이다.
- **관련 논문**: [CreativeBench](https://arxiv.org/abs/2603.11863)

### Q21. 시적 형식 숙성에 따른 커리큘럼 러닝이 가능한가?
- **상태**: 🔴 미답 — 설계 필요

- **우선순위**: ★★★★☆

- **질문**: 초기에 단시(2-4행)를 먼저 학습시킨 후 점차 장시로 확장하는 커리큘럼 설계가 시적 정밀도 향상에 기여하는가?

### Q22. 발음 레이어(Phoneme-level Embedding) 도입이 유효한가?
- **상태**: 🔴 미답 — 실험 필요
- **프로젝트 단계**: Phase 1 — 데이터 수집 및 전처리
- **우선순위**: ★★★☆☆
- **업데이트**: 시의 청각적 요소(자음동화, 두음법칙, 각운 등)는 시각적 서브워드 토큰만으로는 완전한 모델링이 불가능하다. 자소(grapheme)를 발음기호(phoneme)로 변환해 임베딩 레이어에 결합하는 다중 모달(multi-modal) 형태의 전처리가 한국어 시의 음악성(musicality)을 높일 것이라는 가설이 제기된다.
- **질문**: 발음 기호를 병렬 입력하거나 발음 예측 손실 함수를 추가하는 것이 음운적 조화를 지닌 시어 생성에 기여하는가?

### Q23. 계보별(Lineage-specific) 조건부 파인튜닝이 시상의 일관성에 영향을 주는가?
- **상태**: 🔴 미답 — 실험 필요
- **프로젝트 단계**: Phase 0 — 기반 설계
- **우선순위**: ★★★★☆
- **업데이트**: 상이한 미학적 계보(예: 극단적 파격과 해체를 추구하는 미래파 vs 서정적 정제미를 추구하는 전통서정시)의 데이터를 구분 없이 대량 학습시키면 시상의 상충이 발생한다. 메타데이터 태그(`[해체시]`, `[서정시]`)를 접두어로 주입하여 잠재 공간을 분리하는 조건부 파인튜닝 기법이 유효할 것이다.
- **질문**: 미학 계보 태그를 부여하여 조건부 파인튜닝을 수행할 때, 단일 통합 학습 대비 미학적 농도가 향상되는가?

### Q24. 목소리 일관성 유지를 위해 Few-shot ICL과 풀 파인튜닝 중 어느 것이 적합한가?
- **상태**: 🔴 미답 — 실험 필요
- **프로젝트 단계**: Phase 2 — 파인튜닝 실험
- **우선순위**: ★★★★☆
- **업데이트**: 특정 스타일을 재현할 때, 프롬프트 Context 내에 예시 시를 제공하는 Few-shot ICL은 추론 자원 소모가 크고 문맥 윈도우 한계로 장기적인 스타일 일관성이 떨어지지만, 풀 파인튜닝(Full FT)은 해당 스타일을 가중치 깊숙이 내재화하는 대신 일반 글쓰기 및 어휘의 붕괴(overfitting) 위험을 안고 있다.
- **질문**: 특정 시인의 대표 시 3편을 Few-shot으로 제공하는 방식 vs. 해당 스타일로 풀 파인튜닝하는 방식 중 화자의 정체성 유지와 어휘 다양성 보존 면에서 어느 쪽이 우세한가?

### Q25. 모델 양자화(Quantization)가 시의 창의적 일탈 어휘 생성에 악영향을 미치는가?
- **상태**: 🔴 미답 — 실험 필요
- **프로젝트 단계**: Phase 2 — 파인튜닝 실험
- **우선순위**: ★★★☆☆
- **업데이트**: 신경망 양자화(INT4/INT8) 시 하위 비트 연산으로 인해 가중치의 세밀한 변동이 소실된다. 이는 샘플링 단계에서 시적 허용이나 조어(nonce words)와 같은 저확률 일탈 어휘(low-probability creative tokens)가 활성화되는 것을 억제해 시의 참신성을 크게 떨어뜨릴 위험이 있다.
- **질문**: 32B FP16 모델을 INT4/INT8로 양자화했을 때 Distinct-k 다양성 점수 및 시적 조어 발생 비율의 감소폭은 어느 정도인가?

### Q26. 3개 페르소나가 실제로 의미 있게 다른 출력을 만드는가?
- **상태**: 🔴 미답 — Phase 1 실험 필요
- **프로젝트 단계**: Phase 1 — SFT 학습 데이터 생성 직전
- **우선순위**: ★★★★★
- **배경**: 감각형 / 개념형 / 형식실험형 페르소나를 사용하지만, 동일 기반 모델(또는 Claude/GPT-4)에서 시스템 프롬프트만으로 충분히 다른 출력이 나오는지 검증되지 않았다. 페르소나 간 출력이 수렴한다면 DPO 판정이 무의미해지고 학습 신호가 노이즈가 된다.
- **검증 방법**:
  1. 동일 씨앗 10~20개에 대해 3개 페르소나로 시 생성
  2. 임베딩 코사인 거리 (페르소나 A vs B, A vs C, B vs C)
  3. 어휘 중복률 (ROUGE-L, n-gram overlap)
  4. 사람이 직접 읽고 "다르게 느껴지는가" 평가
  5. 비교 기준: 같은 페르소나 × 다른 temperature 샘플링 대비 페르소나 간 거리가 유의미하게 큰가
- **질문**: 3개 페르소나가 temperature 다양성보다 큰 의미 있는 출력 분산을 만드는가? 만약 아니라면 페르소나 정의를 강화하거나 수를 조정해야 한다.
- **파생 질문**: Phase 0 SFT 데이터 생성에서 페르소나를 5~7개로 늘렸을 때 추가 다양성이 실제로 생기는가?

- **상태**: 🔴 미답 — 실험 필요

- **우선순위**: ★★★☆☆
- **배경**: 페르소나가 미학 방향(축 1)을, temperature가 실행 대담성(축 2)을 조정한다. 두 축 모두 포함된 DPO 쌍이 학습에 더 유용할 것으로 가정하나, 실제로 어떤 조합이 최선인지 모른다.
