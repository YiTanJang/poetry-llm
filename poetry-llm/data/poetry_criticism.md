# 시론 및 시평론 데이터

## 왜 시론이 핵심인가

시론(詩論)은 단순한 해설이 아니다.
시인이 **언어를 왜 이렇게 선택했는지**, 행을 **왜 여기서 끊었는지**,
이미지를 **왜 이 방식으로 배치했는지**를 설명하는 텍스트다.

이것은 모델에게 단순히 "시처럼 보이는 텍스트"를 넘어
**시의 논리**를 학습시키는 가장 밀도 높은 경로다.

> 시론은 시 학습 데이터 대비 10배의 교육적 밀도를 가질 수 있다.

## 학습 데이터 변환 전략

시론/시평 텍스트는 모델이 시 창작 시 거치는 내밀한 추론 루프(CoT; Chain of Thought)의 논리적 뼈대를 제공한다. 이를 모델 학습에 활용하기 위해 다음과 같은 변환 방식을 사용하며, 특히 SFT Stage 3(CoT 및 제약 창작) 단계에서 활용하기 위해 고밀도 대화 템플릿(Dialogue Template)으로 변환하는 가이드라인과 스키마 규칙을 적용한다.

### 1. SFT Stage 3 대화형 템플릿 변환 (Dialogue Template for Poetic Logic)

단순한 서술형 평론이나 시 해설 텍스트를 모델이 대화 맥락 속에서 시인의 의사결정 논리를 추론하는 형태로 재구성한다.

#### 1) 변환 원칙 (Conversion Principles)
- **논리적 인과성 강화**: "비평가가 시인의 특정한 형식적/의미적 선택에 대해 질문하고, 시인이 창작자로서의 미학적 이유를 상세히 답하는 형태"의 의사 소통 구도를 재현한다.
- **추상적 개념의 구체화**: 단순한 주제 해설(예: "이 시는 고독을 노래한다") 대신, 특정 시어 선택, 행갈이/연갈이 지점, 이질적 이미지의 병치 등 **물리적인 시 구조의 미학적 인과관계**를 추적하는 질의로 전환한다.
- **수사적 추론 과정(Rhetorical Reasoning Trace) 유도**: 답변은 단답형이 아니어야 하며, [시적 상황 인식 $\rightarrow$ 수사적 대안 대비 $\rightarrow$ 선택된 시적 장치의 효과]로 이어지는 명확한 추론 단계를 포함한다.

#### 2) 대화 데이터 스키마 규칙 (Schema Rules)
밀도 높은 평론 텍스트에서 QA Trace를 구성할 때 다음과 같은 JSON 스키마 구조를 엄격히 따른다.

#### 3) 질의응답 Trace 구성 규칙 (QA Trace Generation Rules)
- **Q (질문) 구성 규칙**:
  - 반드시 시 속의 **특정 구절(단어 X)과 대안적 구절(단어 Y)을 대비**하거나, **특수 토큰(`<행갈이>`, `<연갈이>`)의 분절 위치**를 정확히 지적해야 한다.
  - 질문 템플릿 예시:
    - *"이 시의 2연 3행에서 '주황빛 불빛' 대신 '차가운 형광등'과 같은 상반된 속성의 사물을 대비한 미학적 이유는 무엇입니까?"*
    - *"'<행갈이>' 토큰이 '밤의'와 '가느다란' 사이에 위치함으로써 생기는 호흡의 유예와 의미적 낯설게 하기 효과는 무엇입니까?"*
- **A (답변) 구성 규칙**:
  - 비평 텍스트에 들어 있는 개념적 주장들을 **3단계 논리 구조(Trifold Logic)**로 정렬하여 답변을 서술한다.
    1. **구조적/의미론적 대비**: 사용된 어휘/행갈이가 시 전체의 정서와 어떻게 충돌하거나 공명하는지 대비한다.
    2. **미학적 인과성**: 해당 선택이 독자의 인식을 어떻게 지연시키거나 낯설게(Defamiliarization) 만드는지 설명한다.
    3. **정서적/주제적 낙착**: 이 선택이 결국 시가 도달하고자 하는 최종적 미학(예: 소외감, 실존적 허무 등)을 어떻게 완성하는지 설명한다.

#### 4) 실제 텍스트 변환 가공 예시
- **원문 평론 (Raw Criticism Text)**:
  > *"김수영의 이 시에서 '어깨'는 단순한 해부학적 신체가 아니다. 그것은 군중 속에서 끊임없이 타인과 스치며 마멸되는 현대인의 고독과 규격화된 일상을 의미한다. 특히 '지하'라는 하강적 이미지의 공간 속에서 '어깨'는 무수한 개별자들이 서로의 고독을 인지하지 못한 채 점멸하는 기계적 톱니바퀴의 일부처럼 묘사된다. 시인이 여기서 '어깨' 뒤에 즉각적인 행갈이를 수행한 것은, 군중 속에 섞이면서도 단절되는 현대인의 물리적 고립감을 시각적 여백으로 형상화하려는 의도이다."*

## 학습 데이터 혼합 비율 (Mixing Ratio)

시론 및 비평 데이터는 모델이 시적 비유와 구조적 논리를 학습하는 데 필수적이지만, 과도한 혼합은 모델이 시를 쓰기보다 시에 대한 평론이나 산문 조의 설명글을 출력하는 부작용(Criticism Collapse)을 낳을 수 있다. 따라서 학습 단계별로 엄격히 통제된 혼합 비율을 적용한다.

1. **CPT (Continual Pre-Training) 단계**:
   - **혼합 비율**: **시 평론/시론 텍스트 : 시 텍스트 = 1:3 ~ 1:5**
   - **설명**: 비평 텍스트는 일반 웹 텍스트나 일반 논문에 비해 시적 어휘의 밀도가 매우 높으므로, 비교적 높은 비율로 주입하여 시적 개념어와 메타 지식의 결합도를 높인다.
2. **SFT (Supervised Fine-Tuning) 단계**:
   - **혼합 비율**: **시론/비평 기반 지시 데이터 : 시 창작 지시 데이터 = 1:10 (약 9%)**
   - **설명**: 비평 Q&A, 시 해석, 특정 수사적 제약조건 분석 등 메타 데이터는 전체 SFT 데이터의 약 10% 내외로 제한한다. 이를 넘어서면 창작 프롬프트에 대해서도 시가 아닌 해설을 먼저 길게 작성하는 부작용이 발생할 수 있다.

## 학습 단계별 활용 전략 (CPT vs SFT)

비평 데이터의 형태와 가공 난이도에 따라 CPT와 SFT 단계에 배치할 데이터를 분리하여 학습의 효율성을 극대화한다.

| 학습 단계 | 데이터 유형 | 가공 및 학습 방식 | 학습 효과 및 목적 |
| :--- | :--- | :--- | :--- |
| **CPT** | - 단행본 수준의 시론서<br>- 학술지 및 등단 비평 전문<br>- 시인의 산문/에세이 전집 | - 문단 단위 슬라이딩 윈도우 텍스트화<br>- 시어 및 비평 용어 특화 토크나이저 어휘 사전 반영 | - 시적 어휘(Poetic Vocabulary)의 임베딩 정렬<br>- 문맥에 따른 시적 상징성 이해도 제고 |
| **SFT** | - 시-비평 매칭 Q&A<br>- 시론 기반 개념 사전 추출 데이터<br>- 행갈이/연갈이 근거 분석 데이터 | - `질문-답변` 포맷으로 가공<br>- **CoT(창작 노트)** 템플릿의 근거 생성부 소스로 활용 | - 시 창작 시 '의도적 행갈이' 및 '소재 결합'의 논리적 수행<br>- 사용자 비평 요구(Style/Theme 제어) 수용 능력 배가 |

### 단계 결정 세부 원칙 (Decision Rules)
* **비가공 원문(Raw Text)**: 학술 논문이나 문학 평론집처럼 연속적인 텍스트 흐름이 중요한 자료는 무조건 **CPT**에 배치한다.
* **구조화된 관계 정보(Poetic Logic Mapping)**: 특정 시의 특정 시행에 대응하는 해설, 특정 시론가의 고유 개념 정의 등은 데이터셋 가공을 거쳐 **SFT**의 지시쌍(Instruction-Response Pair)으로 활용한다.

## 외국어 시론 번역 및 품질 관리 (Translation QA)

해외 저명 시론 및 평론(예: T.S. 엘리엇, 에즈라 파운드, 발레리 등)은 현대 한국어 시 창작에 신선한 소재와 미학적 다양성(Novelty)을 제공하는 핵심 자산이다. 그러나 번역의 왜곡이나 번역 투의 어색한 한국어는 생성 모델의 언어 미학을 해칠 수 있으므로 엄격한 번역 QA 가이드라인을 준수한다.

1. **문학/시론 전문 용어 일관성 (Terminology Consistency)**:
   - 번역 시 임의의 기계 번역어 사용을 금지하고, 국내 문학계에서 통용되는 표준 번역어가 반영된 **특화 용어집(Bilingual Glossary)**을 구축하여 활용한다.
   - *예시*:
     * Objective Correlative $\rightarrow$ 객관적 상관물 (객관적 상관관계 등 혼용 금지)
     * Enjambment / Lineation $\rightarrow$ 행갈이 / 줄나눔
     * Defamiliarization $\rightarrow$ 낯설게 하기
   - 번역 LLM의 시스템 프롬프트 또는 사전 룰베이스에 해당 Glossary 매핑을 강제 주입한다.
2. **비평적 어조 및 문체 유지 (Register & Style Preservation)**:
   - 단순 직역이 아닌 한국어 평론가 및 문필가의 고유 문체(예: 문어체, '~이다', '~라 할 수 있다' 등 경어체 지양)를 일관되게 모방하도록 프롬프트를 조정한다.
   - 번역 문장에 '에 대한', '을 통한', '의하여' 등 번역 투 표현(Translationese)을 극소화하는 포스트 에디팅 룰을 적용한다.
3. **인용 시(Cited Poetry) 번역 가이드라인**:
   - 비평 본문 내에 인용된 외국어 시는 단순 번역하지 않고, 국내에서 공인된 번역본(학계 검증판 등)이 존재할 경우 이를 우선적으로 사용한다.
   - 공인본이 없는 경우, 미학적 행갈이와 연갈이 구조를 보존한 채 고품질로 완역하고, 원문 시를 주석이나 병기 형식으로 데이터에 포함한다.
4. **품질 검수(Translation QA) 프로세스**:
   - **1단계 (LLM Translate)**: 용어집이 강제된 LLM 번역 수행.
   - **2단계 (Linguistic Filtering)**: 한국어 자연스러움 필터링 (Grammar Check 및 Perplexity 기준 충족 여부 확인).
   - **3단계 (Expert Review)**: 가공된 데이터의 5% 샘플을 무작위 추출하여 현대 문학 전공자/비평가가 미학적 및 문체적 적합성을 최종 검수.
