# 보조 예술 데이터

## 핵심 가설

> LLM은 데이터를 많이 넣을수록 똑똑해진다.
> 미술, 소설, 극의 언어는 시와 **미학적 회로**를 공유한다.

서로 다른 예술 형식은 각각 고유한 **미학적 언어**를 갖는다.
이를 학습하면 모델은 단일 장르의 관습에 갇히지 않고
더 넓은 감각적 레퍼토리를 갖춘 시를 생성할 수 있다.

## 예술 장르별 학습 가치

### 소설/산문 문학
**학습 가치**: 서사적 긴장, 디테일의 힘, 감각 묘사의 밀도

- 체호프 단편 (장면의 압축)
- 카프카 (부조리한 이미지의 논리)
- 한강 소설 (감각적 밀도, 신체 언어)
- 김애란, 최은영 — 현대 한국 단편
- 세계문학 단편 선집 (보르헤스, 알모도바르 등)

**시에 미치는 영향**: 이미지 하나로 긴 서사를 함축하는 능력

---

### 시각 예술 (에크프라시스 텍스트)
**학습 가치**: 정지된 이미지를 언어로 번역하는 방법, 공간적 사고 (상세 번역 방법 및 메타 프롬프트 예시는 아래 [비언어 예술의 시적 번역 메타 프롬프트 설계](#비언어-예술의-시적-번역-메타-프롬프트-설계) 참조)

- 미술 평론/에크프라시스 텍스트 (그림을 시로 번역한 텍스트들)
- 김우창, 「미술가의 눈」 류의 예술 에세이
- 현대 미술 작가 노트 (개념 미술가의 언어)
- Rilke의 로댕 에세이 — 조각을 시로 번역

**시에 미치는 영향**: 공간적 배치 감각, 색/형태/빛을 언어로 변환

---

### 극/희곡
**학습 가치**: 언어의 리듬과 템포, 침묵과 여백, 대화의 긴장

- 베케트 희곡 — 언어의 극한 절약, 반복과 변주
- 이오네스코 — 언어 해체, 부조리 대화
- 김정옥, 오태석 — 한국 현대극
- 브레히트 — 소외 효과와 언어

**시에 미치는 영향**: 행갈이의 타이밍, 침묵(여백)의 기능, 목소리의 다층성

---

### 음악 이론/악보 해설
**학습 가치**: 리듬, 반복, 변주, 구조적 긴장 (상세 번역 방법 및 메타 프롬프트 예시는 아래 [비언어 예술의 시적 번역 메타 프롬프트 설계](#비언어-예술의-시적-번역-메타-프롬프트-설계) 참조)

- 음악 분석 텍스트 (소나타 형식, 대위법 해설)
- 작곡가 노트 (바르토크, 쇤베르크, 케이지의 작곡 일기)
- 한국 전통 음악 이론 (정간보, 시조 창법)

**시에 미치는 영향**: 시의 음악적 구조 — 반복, 변주, 클라이맥스

---

### 영화 시나리오
**학습 가치**: 장면 전환, 시점의 전환, 시각적 언어 (상세 번역 방법 및 메타 프롬프트 예시는 아래 [비언어 예술의 시적 번역 메타 프롬프트 설계](#비언어-예술의-시적-번역-메타-프롬프트-설계) 참조)

- 영화 시나리오 오픈 데이터베이스 (IMSDB 등)
- 영화 분석 텍스트

## 비언어 예술의 시적 번역 메타 프롬프트 설계

비언어 예술(회화, 음악, 영화)의 고유한 미학적 문법을 시적 언어로 번역하기 위해, 각각의 예술적 구성 요소를 시의 요소(행갈이, 이미지, 리듬, 여백)로 매핑하는 구조화된 메타 프롬프트를 정의한다. 이 프롬프트들은 데이터 전처리 및 인스트럭션 튜닝(Instruction Tuning) 단계에서 학습 데이터의 생성 및 변환 가이드라인으로 사용된다.

### 1. 시각 예술 (미술/회화) -> 시 번역: 에크프라시스(Ekphrasis) 분석법

회화의 공간적 배치, 색채의 대조, 붓 터치의 질감 등을 시의 시각적 심상과 연갈이로 변환한다.

#### 번역 프롬프트 구조 (Ekphrastic Translation Template)
```json
{
  "instruction": "제공된 회화 작품의 분석 정보를 바탕으로, 해당 회화의 공간감과 질감을 시각적으로 복원하는 시를 창작하십시오.",
  "meta_prompts": {
    "구도와 공간 (Composition)": "그림의 전경/중경/배경의 대비를 시의 연(Stanza) 구도로 대응시킬 것 (예: 1연=전경의 디테일, 2연=중경의 흐름, 3연=원경의 소멸).",
    "색채 스펙트럼 (Color Palette)": "색상의 명도와 채도 변화를 시의 형용사와 동사로 번역할 것 (예: '타오르는 황토색', '빛을 빨아들이는 군청').",
    "질감과 붓놀림 (Texture & Brushstroke)": "임파스토(impasto) 기법의 두터운 질감이나 옅은 번짐을 시적 호흡과 단어의 밀도로 표현할 것 (예: 호흡이 가쁜 짧은 시구 vs 스며드는 호흡의 긴 시구)."
  }
}
```

#### 구체적 적용 예시 (정선, 〈인왕제색도(仁王霽色圖)〉)
- **입력 (회화 분석 메타 정보)**:
  - **작품**: 정선의 〈인왕제색도〉 (조선 후기 수묵화)
  - **구도**: 비 온 뒤 안개가 피어오르는 인왕산의 육중한 바위산을 위에서 내려다보는 부감법과 아래에서 올려다보는 앙시법의 절묘한 결합.
  - **색채/명암**: 먹의 짙고 옅음(농담)의 극적인 대조. 젖은 바위의 거대한 검은 질감과 안개의 흰 여백.
  - **질감**: 붓을 뉘어 쓸어내리는 찰법(擦法)과 도끼로 찍은 듯한 부벽준(斧劈皴)의 단단한 질감.
- **번역 인스트럭션**:
  "인왕제색도의 먹 농담과 바위의 거친 질감을 시로 번역하십시오. 거대한 검은 바위의 덩어리감은 무거운 독백(명사 중심)으로, 피어오르는 흰 안개는 가볍게 흩어지는 서술어(용언 중심)로 표현하고, 바위의 갈라진 틈은 행갈이(`<행갈이>`)로 번역하십시오."
- **시 생성 결과 예시**:
  ```markdown
  젖은 바위가 공중을 누르고 앉아 있다 <행갈이>
  시커먼 골짜기마다 붓을 뉘어 쓸어내린 자국 <행갈이>
  비가 그치자 <행갈이>
  비로소 드러나는 살을 깎는 뼈들 <연갈이>
  흰 안개는 명사를 버리고 <행갈이>
  골짜기 사이로 가볍게 흘러가고 <행갈이>
  검은 이마만 남은 인왕의 정수리 아래 <행갈이>
  여백이 거대한 숨을 들이쉬고 있다
  ```

### 2. 청각 예술 (음악) -> 시 번역: 조성/동역학 및 대위법적 매핑

음악의 시간적 흐름, 템포, 셈여림(Dynamics), 다성음악(Polyphony)의 대위법적 구조를 시의 리듬감, 호흡의 속도, 다층적 목소리(폴리포니)로 변환한다.

#### 번역 프롬프트 구조 (Musical Poetics Template)
```json
{
  "instruction": "제공된 음악적 분석 노트를 바탕으로, 음악의 리듬과 템포, 구조적 대위법을 시의 호흡과 행갈이 구조로 번역하십시오.",
  "meta_prompts": {
    "템포와 다이내믹스 (Tempo & Dynamics)": "빠르기표(알레그로, 아다지오)와 셈여림(포르테, 피아노)을 음절 길이와 구두점, 행갈이 빈도로 매핑 (예: 알레그로=3-4조의 짧은 음절 배치와 잦은 행갈이; 크레센도=연이 길어질수록 어휘의 강도 강화).",
    "대위법과 다성성 (Counterpoint & Polyphony)": "서로 다른 두 개의 주제 선율이 얽히는 구조를 시에서 대화체와 독백체의 교차 혹은 괄호(`()`)를 사용한 내부 목소리의 삽입으로 시각화.",
    "종지법과 여백 (Cadence & Silence)": "음악이 멈추는 페르마타(쉼표)나 종지를 3회 연속의 `<행갈이>` 혹은 빈 행인 `<연갈이>`로 변환하여 잔향을 형성."
  }
}
```

#### 구체적 적용 예시 (베토벤, 〈교향곡 5호 운명, 1악장〉)
- **입력 (음악 분석 메타 정보)**:
  - **작품**: 베토벤 교향곡 5호 다단조 Op.67, 1악장
  - **리듬/동기**: 유명한 4음 동기 (단-단-단-장: ♪♪♪♩)가 악장 전체를 지배하며 끊임없이 변주됨.
  - **구조**: 강렬한 포르티시모의 첫 동기 제시 후, 서로 다른 악기들이 이를 모방하며 상승하고 충돌하는 소나타 형식.
- **번역 인스트럭션**:
  "베토벤 5호 교향곡 1악장의 '단-단-단-장' 리듬 동기를 시의 3음보-4음보 변주로 번역하십시오. 첫 연은 강렬한 노크 소리처럼 시작하고, 이어지는 연들은 각 악기가 선율을 이어받듯 시의 행마다 주어를 바꾸어 가며 리듬을 점층(크레센도)시키십시오. 마지막 결미는 갑작스러운 멈춤으로 여백을 남기십시오."
- **시 생성 결과 예시**:
  ```markdown
  그가 온다 <행갈이>
  문 두드리는 세 번의 짧은 타격, 그리고 긴 침묵 <연갈이>
  바람이 받고 <행갈이>
  현(絃)이 팽팽하게 당겨지고 <행갈이>
  마른 잎새가 땅 위에 뒹굴며 소리를 모아 <행갈이>
  하늘 끝까지 치솟는 소리의 거대한 기둥 <연갈이>
  쿵, <행갈이>
  멎은 심장 속에서 <행갈이>
  귓바퀴만 뜨겁게 타오르는 정적
  ```

### 3. 시공간 예술 (영화) -> 시 번역: 몽타주와 미장센 기법

영화의 컷 전환(몽타주), 카메라 앵글(익스트림 클로즈업/롱샷), 미장센(공간 배치 및 조명)을 시의 연상 작용, 시선 이동, 심상의 명암 대비로 번역한다.

#### 번역 프롬프트 구조 (Cinematic Translation Template)
```json
{
  "instruction": "제공된 영화 씬의 연출 방식을 분석하여, 카메라의 움직임과 몽타주 기법을 시의 구조적 흐름으로 변환하십시오.",
  "meta_prompts": {
    "시선과 렌즈 (Lens & Angle)": "클로즈업(미세한 사물의 극단적 묘사)과 롱샷(풍경의 조망)을 매 연마다 교차시켜 독자의 시선 거리를 강제로 조정.",
    "몽타주와 쇼트 전환 (Montage & Cuts)": "전혀 다른 두 이미지의 충돌로 새로운 의미를 만드는 소련 몽타주 기법(쿨레쇼프 효과 등)을 콤마(,)나 급격한 행갈이(`<행갈이>`)를 통한 명사-명사 충돌로 표현.",
    "조명과 미장센 (Lighting & Mise-en-scène)": "화면 내 광원의 위치(역광, 측광)와 소품의 배치를 시각적 은유로 변환 (예: 인물의 얼굴 절반이 어둠에 묻힌 로우키 조명 -> 이중적인 자아를 나타내는 두 개의 대비되는 형용사 사용)."
  }
}
```

#### 구체적 적용 예시 (안드레이 타르코프스키, 〈희생(The Sacrifice)〉의 마지막 불타는 집 씬)
- **입력 (영화 분석 메타 정보)**:
  - **작품**: 타르코프스키의 영화 〈희생〉 마지막 롱테이크 (6분 이상의 고정 숏)
  - **연출**: 주인공 알렉산더가 서약대로 자신의 모든 소유물인 집을 불태움. 집이 화염에 휩싸여 무너지는 동안, 카메라는 수평으로 천천히 트래킹하며 패닉에 빠진 인물들과 구급차, 불타는 집의 파멸적 시각을 한 화면에 담아냄.
  - **소리**: 타오르는 불꽃의 미세한 크랙 소리, 멀리서 들리는 구급차 사이렌, 그리고 바흐의 〈마태수난곡〉 중 '나를 불쌍히 여겨소서(Erbarme dich)'가 겹침.
- **번역 인스트럭션**:
  "타르코프스키의 롱테이크 카메라 트래킹을 재현하듯, 끊어지지 않고 길게 이어지는 문장(연과 행의 구분을 최소화하는 롱테이크 구조)을 사용하십시오. 타오르는 집의 시각적 파괴와 마태수난곡의 애절한 청각적 요소를 동시에 한 행에 병치하여 몽타주 효과를 만드십시오."
- **시 생성 결과 예시**:
  ```markdown
  대들보가 타들어 가는 소리를 바흐의 첼로 현이 길게 잡아 늘이고 있었다 <행갈이>
  바퀴는 진흙 구덩이 속에 빠져 헛도는데 정원 앞의 목조 가옥은 제 가슴을 찢어 붉은 혀를 내밀고 <행갈이>
  다 타버린 기둥이 쓰러지는 초 단위의 고통을 붙잡아두려는 듯 카메라는 멈추지 않고 젖은 들판을 수평으로 기어간다 <행갈이>
  구원이라는 단어가 재가 되어 공중으로 흩어질 때 <행갈이>
  그는 붉은 웅덩이 위에 엎드려 울고 있었고, 엎드린 그의 등 뒤로 무너지는 지붕, 연기가 긴 호흡의 문장을 끈질기게 이어나간다
  ```

## 혼합 비율 제안 (미결)

```
핵심 데이터 (시, 시론):     70%
보조 문학 (소설, 극):       20%
비언어 예술 텍스트:         10%
```

> 이 비율은 초기 파일럿 결과에 따라 조정 필요.
> 보조 데이터가 너무 많으면 시 생성 능력이 희석될 수 있음.

## 미결 사항

- 에크프라시스 및 음악 번역 메타 프롬프트의 평가 자동화: 번역된 시가 실제 원작 비언어 예술의 미학적 특징(예: 정선의 먹 농담, 베토벤의 4음 동기 리듬)을 얼마나 잘 보존하고 있는지 정량적으로 평가할 수 있는 유사도 지표(Similarity Metric)나 인간 평가 가이드라인을 어떻게 정의할 것인가?
- 멀티모달 모델(예: Qwen2-VL)의 활용 여부: 현재는 비언어 예술을 '분석된 텍스트 정보'로 변환하여 LLM에 입력하는 파이프라인을 사용하지만, 이미지/비디오/오디오 입력이 가능한 멀티모달 LLM을 직접 파인튜닝하여 비언어 예술 원본을 입력받아 시를 창작하게 하는 프롬프트/파인튜닝 파이프라인과의 성능 비교는 어떻게 수행할 것인가?
- 비언어 예술 데이터의 오버핏(Overfitting) 방지 및 편향 완화: 비언어 예술 분석 텍스트나 메타 프롬프트 데이터가 학습에 과도하게 개입할 경우, 자유시/서정시 고유의 리듬이 왜곡되거나 난해한 개념 미술적 산문시 형태로만 편향될 우려가 있는데, 이에 대한 최적의 데이터 혼합 비율(Data Mixture Ratio) 및 정규화(Regularization) 방안은 무엇인가?
