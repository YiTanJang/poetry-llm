# 데이터 정제 및 품질 필터링 가이드라인

## 1. 메타데이터 표준 스키마 (Detailed Metadata Schema)

시 생성 모델의 지도 파인튜닝(SFT) 및 지속 사전학습(Continued Pre-training)을 효과적으로 제어하기 위해, 수집되는 모든 데이터는 정규화된 스키마에 따라 메타데이터가 관리되어야 한다. 데이터 유형별 세부 스키마는 다음과 같다.

### 1-1. 한국 현대시 및 고전시 스키마 (Poetry Metadata Schema)
```json
{
  "$schema": "http://json-schema.org/draft-07/schema#",
  "title": "PoetryMetadata",
  "type": "object",
  "properties": {
    "id": { "type": "string", "description": "고유 식별자 (예: KO-MOD-0001)" },
    "title": { "type": "string", "description": "시 제목" },
    "author": { "type": "string", "description": "시인 이름" },
    "author_birth_death": { "type": "string", "description": "시인 생몰년도 (예: 1917-1945, 고전시 정제용)" },
    "book_title": { "type": "string", "description": "출판된 시집 제목" },
    "publisher": { "type": "string", "description": "출판사" },
    "publish_year": { "type": "integer", "description": "최초 출판 연도 (YYYY)" },
    "pages": { "type": "string", "description": "시집 내 페이지 범위 (예: p.12-14)" },
    "language": { "type": "string", "enum": ["ko", "en", "ja", "zh", "de"], "description": "원문 언어 코드" },
    "era": { "type": "string", "enum": ["classical", "contemporary"], "description": "시대 구분" },
    "source_type": { "type": "string", "enum": ["scan", "epub", "web", "synthetic"], "description": "수집 출처 유형" },
    "copyright_status": { "type": "string", "enum": ["public_domain", "licensed", "fair_use"], "description": "저작권 상태" },
    "has_hanja": { "type": "boolean", "description": "한자 혼용 여부" }
  },
  "required": ["id", "title", "author", "publish_year", "language", "era", "source_type", "copyright_status"]
}
```

## 2. 시 데이터 정제 규칙 (Poetry Cleaning Rules)

시 데이터는 일반적인 산문 코퍼스(웹 덤프, 위키백과 등)와 달리 **행(Line)과 연(Stanza)의 구조, 여백(들여쓰기), 의도적인 띄어쓰기 파괴** 등이 그 자체로 시적 리듬과 미학을 구성한다. 따라서 정제 시 고도의 보존 규칙이 적용되어야 한다.

### 2-1. OCR 노이즈 정밀 보정
스캔본 PDF에서 OCR을 거친 텍스트는 다음과 같은 빈번한 오류를 동반하므로 패턴 매칭 및 휴리스틱 규칙으로 정제한다.

| OCR 오류 유형 | 예시 | 보정 규칙 |
|---------------|------|-----------|
| 자음/모음 분리 | `ㅇㅏ침` $\rightarrow$ `아침` | 유니코드 정규화(NFC)를 적용하여 초/중/종성 결합 상태 복원. |
| 행 병합 오류 | 두 시행이 하나의 행으로 붙어버림 | OCR 레이아웃 JSON의 바운딩 박스(Bounding Box) 좌표를 참조하여, 수평 좌표 격차가 임계값을 넘을 경우 개행을 복원. |
| 마침표/쉼표 오인식 | 깨진 점이나 인쇄 흔적이 `.` 또는 `,`로 오인식 | 원문 대조 필터링을 통해 시 전체의 구두점 밀도를 측정하여, 일반적인 한국어 문장에 비해 비정상적으로 구두점이 흩어져 있는 경우 정규식으로 제거. |
| 페이지 번호/헤더 삽입 | 시 본문 내 `45` 또는 `김수영 전집` 삽입 | 시 본문의 시작과 끝 바운딩 박스 바깥에 위치한 텍스트(여백 영역)는 자동 슬라이싱으로 배제. |

### 2-2. 띄어쓰기 및 구두점 보존 규칙
- **원칙**: 시인의 의도적인 띄어쓰기 무시나 비표준 맞춤법은 **절대 수정하지 않는다.**
  > [!IMPORTANT]
  > 이상의 「오감도」에 등장하는 "제1의아해가무섭다고그리오" 등 띄어쓰기가 없는 문장은 인위적으로 띄어쓰기를 적용할 경우 시의 불안과 속도감이라는 미학적 의도가 붕괴된다.
- **예외적 정제**: 동일 행 내에서 단일 공백이 3개 이상 연속으로 나타나는 경우, 이것이 의도적인 들여쓰기/여백인지 아니면 OCR의 자간 인식 오류인지 구분해야 한다.
  - 행의 시작 부분 공백: **들여쓰기로 간주하여 유지 (또는 `<tab>` 토큰으로 변환)**.
  - 단어 사이의 다중 공백: 폰트 스타일 및 레이아웃 좌표 분석을 거쳐 자간 오류로 판명될 경우 **단일 공백으로 치환**.

### 2-3. 한자(Hanja) 및 고어 처리
- **현대시 한자**: 현대 한국어 시에 포함된 한자는 시어가 지닌 시각적 한계와 압축성을 반영하므로 **한자를 그대로 보존**하되, 모델이 음독을 학습할 수 있도록 발음 레이어를 추가로 부여한다.
- **고전시 및 개화기 시**: 아래아(`ㆍ`), 반치음(`ㅿ`) 등 소실 자모가 포함된 텍스트는 현대어 표기로 일괄 변환하지 않고, **원문 코퍼스**와 **현대어 번역 코퍼스**의 페어로 데이터셋을 구축한다.

### 2-4. 의도된 시적 여백/줄바꿈 보존 및 레이아웃 오개행 정제 규칙
스캔 및 OCR 과정에서 종이책의 물리적 페이지 여백(Scanner Margin)이나 단 분할 레이아웃으로 인해 단어 도중에 발생하는 **오개행(Layout-induced Carriage Returns)**과, 시인이 의도하여 배치한 **시행(Poetic Line Breaks) 및 빈 줄(Poetic Spacing)**을 정확히 판별하기 위한 기준을 적용한다.

1. **시적 빈 줄(Poetic Empty Lines)의 보존**
   - 시인이 연(Stanza)의 구분 또는 호흡의 단절을 위해 의도적으로 남겨둔 빈 줄(`\n\n`)은 전처리 과정에서 보존되며, 최종 토큰화 단계에서 `<연갈이>` 특수 토큰으로 변환된다.
   - 단, 스캔 페이지 경계부에서 기계적으로 발생하는 페이지 구분선 및 중복 빈 줄은 메타데이터(출처 도서의 페이지 범위 등)와 비교 검증하여 자동 병합하고, 순수한 시행 사이의 빈 줄만 유지한다.

2. **레이아웃 오개행(Scanner Margin Breaks) 식별 및 복원 규칙**
   - **형태소적 연결성 판정 (Morphemic Continuity)**: 
     개행문자(`\n`) 앞단어와 뒷단어의 형태소적 결합 관계를 형태소 분석기로 분석한다. 개행의 앞쪽이 용언의 어근이거나 체언(명사 등)이고, 뒤쪽이 필수 격조사('가', '이', '을', '를' 등) 또는 종속적 어미인 경우, 물리적인 레이아웃 강제 개행으로 간주하여 개행을 제거하고 단어를 결합한다.
   - **물리적 레이아웃 좌표 기반 룰셋**:
     OCR 엔진이 제공하는 개별 글자/단어 단위의 바운딩 박스(Bounding Box) 좌표를 대조한다. 
     - *강제 개행 판정*: 특정 시행의 종결 단어 우측 경계 좌표가 해당 텍스트 블록의 최대 우측 여백 한계선(Margin Limit)에 밀착(90% 이상 점유)해 있고, 다음 행의 첫 단어가 좌측 한계선에 밀착해서 시작되는 동시에, 앞 단어와 결합성이 높다면 이를 인쇄 지면 제약에 의한 줄 바꿈으로 판정하고 병합한다.
     - *의도된 시행 판정*: 우측 여백이 많이 남아 있음에도 불구하고 개행이 된 경우에는 시인의 의도적 개행으로 분류하여 원형대로 유지한다.
   - **참조 모델 PPL 기반 변동성(Delta-PPL) 대조**:
     형태소 및 좌표 분석으로도 판정이 모호한 경우, [개행 유지]와 [개행 제거 후 공백 결합] 두 가지 텍스트 버전에 대하여 참조 언어 모델(Gemma 4 9B-IT)을 사용해 각각 Perplexity를 측정한다.
     $$\Delta \text{PPL} = \text{PPL}_{\text{split}} - \text{PPL}_{\text{merged}}$$
     병합 후의 PPL이 유의미하게 감소하여 $\Delta \text{PPL} > \theta_{\text{merge}}$ (검증용 골드 데이터로 캘리브레이션된 임계값)을 만족할 경우, 레이아웃 오류에 의한 오개행으로 판단하여 병합 처리를 적용한다.

## 3. 데이터 품질 필터 (Data Quality Filters)

학습 코퍼스 구축 단계에서 저품질 데이터(스캔 불량으로 파쇄된 텍스트, 비시적 텍스트의 혼입 등)를 걸러내기 위해 다단계 정량 필터를 적용한다.

```
[입력 데이터] ──> [1단계: 레이아웃 검증] ──> [2단계: PPL 필터] ──> [3단계: 시적 다양성 필터] ──> [최종 코퍼스]
```

### 3-1. Layout 무결성 필터 (Layout Integrity Filter)
- **개행 비율 검사**: 총 글자 수 대비 개행 문자(`\n`)의 비율을 계산한다.
  - 시 데이터의 개행 비율 임계값: **0.05 ~ 0.25 (글자 100자당 개행 5~25회)**.
  - 이 범위를 벗어나는 데이터(예: 개행이 전혀 없는 텍스트)는 소설이나 설명문 등의 오분류 데이터로 판정하여 배제한다. (산문시는 수동 예외 처리 테이블 등록 필요).
- **빈 줄(연 구분) 검사**: 전체 텍스트에서 2회 연속 개행(`\n\n`)의 존재 여부를 파악한다. 연 구분이 전혀 없는 장형 시의 경우, 특수 구조 메타데이터를 확인하여 검증한다.

### 3-2. Perplexity (PPL) 기반 노이즈 필터
OCR 문자 인식 실패, 스캐닝 시 노이즈 유입으로 인한 횡설수설(gibberish) 및 무작위 문자열 필터링을 위해, 대규모 언어 모델을 활용한 Perplexity 임계값 검증 파이프라인을 운영한다.

- **참조 언어 모델 (Reference LM) 선정**
  한국어 맥락 이해도와 문장 구조 일관성 평가 역량이 높은 **Gemma 4 9B-IT** 또는 **Solar-10.7B** 모델을 참조 모델로 적용하여 토큰 Likelihood를 계산한다.

- **PPL 계산 및 노이즈 수치화 메커니즘**
  입력 텍스트 시퀀스 $X = (x_1, x_2, \dots, x_N)$에 대하여, 각 토큰의 조건부 확률의 기하평균의 역수로 Perplexity를 정의한다.
  $$\text{PPL}(X) = \exp \left( -\frac{1}{N} \sum_{i=1}^N \log P_{LM}(x_i \mid x_{<i}) \right)$$
  의미론적 연결성이 완전히 단절된 OCR 오인식 문자열이나 기형적인 기호 나열 등은 조건부 확률 $P_{LM}(x_i \mid x_{<i})$이 극도로 작아져 PPL 값이 기하급수적으로 폭증하게 된다.

- **품질 필터링 임계값(Thresholding Rules) 및 동적 보정 규칙**
  1. **표준 컷오프(Standard Cutoff)**:
     - 검증된 고품질 한국어 현대시 레퍼런스 데이터셋의 PPL 평균을 $\mu_{\text{ref}}$, 표준편차를 $\sigma_{\text{ref}}$라 할 때, 기본 필터링 컷오프는 다음과 같다.
       $$\text{Threshold}_{\text{base}} = \mu_{\text{ref}} + 2.5 \cdot \sigma_{\text{ref}}$$
     - 계산된 $\text{PPL}(X) > \text{Threshold}_{\text{base}}$를 초과하는 데이터는 저품질 OCR 파쇄 텍스트로 간주하여 자동 정제 대상에서 배제한다.
  2. **난해시 및 고어 혼용 시에 대한 동적 보정 계수 ($k$) 적용**:
     - 이상의 시와 같이 띄어쓰기를 의도적으로 파괴했거나, 한자 혼용율이 높은 시, 혹은 1920-30년대 개화기 국한문 혼용체의 경우 레퍼런스 모델이 비정상적으로 높은 PPL을 산출할 수 있다.
     - 메타데이터(예: `has_hanja`, `era`)와 본문 내 고어 형태소 비율을 실시간 추정하여 동적 보정 계수 $k$를 부여한다.
       $$\text{Threshold}_{\text{dynamic}} = \mu_{\text{ref}} + (2.5 + k) \cdot \sigma_{\text{ref}} \quad (\text{단, } k = f(\text{HanjaRatio}, \text{ArchaicWordRatio}))$$
     - *예외 작가 화이트리스트*: 사전에 지정된 실험적 난해시 작가군(예: 이상 등)의 경우, PPL 필터 컷오프를 $\mu_{\text{ref}} + 4.0 \cdot \sigma_{\text{ref}}$로 대폭 완화하는 대신, 2단계 수동 검증 및 원문 레이아웃 보존 규칙을 필수로 우회 통과하게 조치한다.

### 3-3. 내용 중복 및 유사도 필터
- 동일한 시가 서로 다른 소스(스캔본, 웹 공개 데이터 등)에서 이중 수집되어 중복 학습되는 것을 방지하기 위해 **MinHash LSH** 알고리즘을 사용한다.
- **임계치 설정**: Jaccard 유사도 기준 **0.90 이상**인 시편 쌍이 발견될 경우, 메타데이터 신뢰도가 높은 소스(예: 공식 전자책 출처)의 텍스트를 남기고 나머지는 중복 제거한다.

## 4. 라이선싱 및 저작권 준수 전략 (Licensing & Copyright)

한국의 저작권법을 준수하면서 현대시집 약 2,000권 규모의 고품질 데이터셋을 합법적으로 구축하기 위한 법률적·기술적 전략을 수립한다.

### 4-1. 저작권법 제35조의3 (공정 이용) 적용 검토
대한민국 저작권법 제35조의3은 "저작물의 통상적인 이용 방법과 충돌하지 아니하고 저작자의 정당한 이익을 부당하게 해치지 아니하는 범위" 내에서 저작물을 이용할 수 있는 **일반적 공정 이용(Fair Use)** 규정을 두고 있다.

- **텍스트 및 데이터 마이닝(TDM) 면책**:
  학습용 데이터 수집은 해당 시집을 독자에게 대체 판매하는 행위가 아니며, 모델 학습이라는 비상업적 연구 목적 및 기술적 가치 창출에 해당하므로 공정이용의 성격을 지닌다.
- **위험 최소화 조치**:
  - 수집된 원문 데이터의 외부 유출을 원천 차단한다. (폐쇄형 내부 서버 내에서만 학습 진행).
  - 생성된 시가 기존 원문의 3-gram 이상을 그대로 복제하여 출력하지 않도록 **Generation Novelty Filter**를 추론 단계에 필수 적용한다.

### 4-2. 저작권 만료 및 권리 보장 협약
- **공공 도메인 활용 극대화**: 사망 후 70년이 경과한 시인(윤동주, 이상, 정지용 등)의 작품은 출처만 명시하고 자유롭게 활용한다.
- **현대 시인의 권리 보장 (Poet-in-the-Loop)**:
  - 주요 현대 시인 협회 및 문학 전문 출판사(창비, 문학과지성사 등)와 학술·연구 목적으로 한 한정적 데이터 사용 MOU를 체결한다.
  - 작가들의 데이터 제공 거부권(Opt-out)을 공식 보장하며, 요청 시 즉각 학습 데이터셋에서 배제하는 파이프라인을 유지한다.
