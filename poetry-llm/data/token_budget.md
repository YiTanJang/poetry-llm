# 토큰 예산 및 데이터 규모 계산

## 기본 가정

한국어 텍스트 토큰화 특성 (BPE 기반, LLaMA/Mistral 계열 기준):
- 한국어는 영어 대비 토큰 효율이 낮다
- 영어: 1단어 ≈ 1~1.3 토큰
- 한국어: 1어절 ≈ 2~4 토큰 (형태소 분리 방식에 따라 다름)
- 한국어 특화 토크나이저 (EXAONE, HyperCLOVA 계열): 1어절 ≈ 1.5~2 토큰

이 문서에서는 **일반 BPE 기준** (1어절 ≈ 3 토큰)과
**한국어 최적화 기준** (1어절 ≈ 1.8 토큰) 두 가지를 병기한다.
실제 토크나이저 선택 후 재계산 필요.

---

## 1. 수집 데이터별 토큰 수 추정

### 1-1. 한국 현대시집 (~2,000권)

시집 1권의 구조:
- 평균 시편 수: 50~70편 (중간값 60편)
- 시편당 평균 행 수: 15~25행 (중간값 20행)
- 행당 평균 어절 수: 4~7어절 (중간값 5어절)
- 시편당 평균 어절: 20행 × 5어절 = 100어절

시집 1권 기준:
- 시 본문: 60편 × 100어절 = 6,000어절
- 제목, 메타데이터, 목차: +500어절
- 권당 어절: ~6,500어절

| 토크나이저 | 권당 토큰 | 2,000권 총 토큰 |
|------------|----------|----------------|
| 일반 BPE | 6,500 × 3 = ~19,500 | ~39M 토큰 |
| 한국어 최적화 | 6,500 × 1.8 = ~11,700 | ~23M 토큰 |

**2,000권 추정: 약 23~39M 토큰**
중간 추정치: **30M 토큰**

### 1-2. 시론/평론서

시론서 1권의 구조:
- 일반 산문 단행본 기준
- 평균 페이지: 250~350쪽 (중간값 300쪽)
- 페이지당 어절: 300~400어절 (중간값 350어절)
- 권당 어절: 300쪽 × 350어절 = 105,000어절

| 토크나이저 | 권당 토큰 |
|------------|---------|
| 일반 BPE | ~315,000 토큰 |
| 한국어 최적화 | ~190,000 토큰 |

**시론서 1권 ≈ 190,000~315,000 토큰**
중간 추정치: **250,000 토큰 (0.25M)**

### 1-3. 수집 목표 전체 토큰 추정

`index.md` 기준 수집 목표:

| 데이터 유형 | 추정 분량 | 중간 추정 토큰 |
|-------------|----------|--------------|
| 한국 현대시 (~2,000권) | 2,000권 | 30M |
| 저작권 만료 한국 시 | ~300권 추정 | 4M |
| 신춘문예 / 문예지 수록시 | ~10,000편 추정 | 5M |
| 한국 시론/시평론 | ~200권 추정 | 50M |
| 외국어 시 (~500권) | 500권 | 8M |
| 외국어 시론/평론 | ~300권 추정 | 75M |
| 보조 예술 데이터 | 적정 비율 | 20M |

**총 추정: 약 192M 토큰**
보수적 추정: **150M 토큰** / 낙관적 추정: **250M 토큰**

주목할 점: 시론/평론이 시 본문보다 **총 토큰이 많다**.
시 한 편은 짧지만 시론 한 권은 길고, 시론의 교육적 밀도는
시 원문보다 높다고 가정하고 있다. (→ `poetry_criticism.md` 참고)

## 2. 학습에 필요한 토큰 수

### 2-1. 도메인 적응(Domain Adaptation)에 필요한 최소 토큰

기반이 되는 연구 및 경험적 관찰:

**사전학습 완료된 모델의 도메인 적응 (Continued Pre-training):**
- LLaMA-2 계열 도메인 적응 실험에서 10B~30B 모델 기준
  1B~10B 토큰 추가 학습으로 도메인 내 성능 유의미하게 향상
- 특수 도메인 (의학, 법률 등)에서 1~5B 토큰이 실질적 하한선으로 관찰됨
- 단, 시는 일반 텍스트보다 도메인이 훨씬 좁으므로 이보다 작은 규모도 가능

**파인튜닝 (Instruction Tuning):**
- SFT (Supervised Fine-tuning): 10K~100K 예시면 충분 (포맷 데이터 기준)
- 시 생성 능력 파인튜닝: 50K~200K 예시 추정 (행갈이 학습 포함)
- 토큰으로 환산: 포맷 3 기준 1예시 ≈ 500~1,000 토큰 → 50K 예시 = 25~50M 토큰

**20~40B 파라미터 모델 도메인 적응 최소 토큰 추정:**

| 단계 | 최소 권고 | 이상적 | 비고 |
|------|----------|--------|------|
| Continued Pre-training | 500M 토큰 | 2B~5B 토큰 | 도메인 어휘 흡수 |
| SFT (포맷 데이터) | 25M 토큰 | 100M 토큰 | CoT 포맷 학습 |
| 합계 | ~525M 토큰 | ~5B 토큰 | |

**현실적 결론:** 이 프로젝트의 수집 목표(~150~250M 토큰)는
Continued Pre-training 최소 권고(500M)에 **미달한다**.
단, 시 도메인의 특수성 (포맷이 매우 압축적임)을 고려하면
300M 토큰도 의미 있는 효과를 낼 수 있다는 실험적 근거가 있다.

### 2-2. 데이터 부족 시 합성 데이터 보완 계획

수집 가능한 실제 데이터: 약 **150M 토큰** (보수적 추정)
Continued Pre-training 목표: **500M 토큰**
부족분: **약 350M 토큰**

합성 데이터 비율 상한 30% 적용 시:
- 허용 합성 데이터: 총 데이터의 30% = 150M × (100/70) × 0.30 ≈ 64M 토큰

즉, 합성 데이터를 최대한 생성해도 총 학습 데이터는
실제 150M + 합성 64M = **214M 토큰** 수준.

목표 500M에는 여전히 부족하다.

**해결 전략:**

옵션 1: Epoch 수 증가 (권장하지 않음)
- 같은 데이터를 3~5회 반복 학습
- 소규모 데이터에서 과적합 위험 있음
- 단, epoch 2~3회까지는 성능 향상 관찰된 사례 있음

옵션 2: 합성 데이터 비율 상한 완화 (조건부 허용)
- 합성 창작 노트, 수정 이력 등의 **형식 데이터**는 50%까지 허용 가능
- 이유: 이 데이터는 시 원문에서 파생되므로 독립적인 새 정보가 아님
- 단, 시 원문 자체의 합성(새 시 생성)은 여전히 30% 상한 유지

옵션 3: 합성 데이터 집중 생성
- 포맷 3(창작 노트): 2,000권 시집 × 60편 × 4개 = 480,000 예시
  → 1예시 ≈ 700 토큰 = **336M 토큰** (합성)
- 이 경우 총 데이터: 150M + 336M = 486M 토큰 (목표 근접)
- 단, 합성 비율이 69%로 상한을 크게 초과함

**권고 경로:** 옵션 2와 3의 중간.
합성 창작 노트를 **포맷 형식 데이터**로 분류하여 별도 상한(50%)을 적용.

```
실제 데이터:           150M 토큰 (기준)
합성 형식 데이터 (50%): 150M 토큰 (창작 노트, 수정 이력)
합성 시 원문 (30%):    ~64M 토큰 (제한적 사용)
────────────────────────────────
총 학습 데이터:        ~364M 토큰
```

여전히 500M 목표에 미달하지만, 2~3 epoch 학습으로 **700M+ 유효 토큰** 달성.

### 2-3. Epoch 수와 실제 학습 토큰의 관계

```
유효 학습 토큰 = 데이터 토큰 × epoch 수

364M 데이터, epoch 2회 → 유효 728M 토큰
364M 데이터, epoch 3회 → 유효 1,092M 토큰 (1B+)
```

권고 epoch 수:
- Continued Pre-training: **2 epoch** (3 epoch부터 과적합 징후 모니터링)
- SFT: **3~5 epoch** (SFT 데이터는 작으므로 epoch 늘려도 위험 낮음)

과적합 조기 발견 지표:
- 검증 손실(validation loss)이 학습 손실 대비 2배 이상 발산 시 중단
- 시 생성 다양성 측정 (같은 프롬프트에 대한 출력 분산이 줄어들면 과적합 신호)

## 3. 수집 목표 달성 가능성

### 3-1. 스캔 기반 수집 시 현실적 상한

| 유형 | 수집 목표 | 예상 토큰 |
|------|----------|---------|
| 한국 현대시 (스캔) | 2,000권 | 30M |
| 한국 시론/평론 (스캔) | 200권 | 50M |
| 외국어 시/시론 | 500권 | 83M |
| 보조 예술 데이터 | 적정량 | 20M |
| 합성 데이터 | 창작 노트 역추론 등 | 100~150M |
| **총합** | | **280~330M 토큰** |

### 3-2. 합성 데이터 보완 계획

합성 창작 노트(포맷 데이터)를 실제 데이터 파생물로 분류하면 상한 50%까지 허용:

```
실제 데이터:             ~200M 토큰
합성 형식 데이터 (50%):  ~150M 토큰 (창작 노트, 수정 이력)
────────────────────────────────
총 학습 데이터:          ~350M 토큰 × 2 epoch = 700M 유효 토큰
```

충분히 목표 범위(500M+) 달성 가능.

## 계산 요약

```
목표:
  실제 데이터:    200M 토큰  (스캔 완료 기준)
  합성 데이터:    150M 토큰
  총:             350M 토큰 × 2 epoch = 700M 유효 토큰

최소 기능 임계점 (추정):
  실제 데이터:    50M 토큰
  합성 데이터:    50M 토큰
  총:             100M 토큰 × 3 epoch = 300M 유효 토큰
  → 이 수준에서는 일반 스타일 모방 가능, 창작 CoT는 약함
```

토큰 예산 계산은 실제 토크나이저 결정 후 재계산한다.
특히 한국어 특화 BPE vs. 범용 BPE 선택이 토큰 수를 2배까지 바꿀 수 있다.
