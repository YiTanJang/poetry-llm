# 저작권 Opt-out 필터링 시스템

본 문서는 살아있는 시인들의 작품과 관련된 저작권 문제를 해결하기 위한 실시간 Opt-out 시스템 및 데이터 거버넌스 설계를 다룬다. 모델 학습 및 배포 과정에서 시인의 권리를 보호하고 투명성을 유지하는 것이 핵심 목표다.

## 실시간 Opt-out 시스템

생존 시인이나 유족이 자신의 작품을 학습 데이터에서 제외하도록 요청할 수 있는 권리를 보장한다.

- **요청 채널 구축**: 전용 웹 폼 및 이메일 창구를 통해 Opt-out 접수.
- **실시간 필터링 적용**:
  - 데이터 전처리 파이프라인에서 접수된 시인/저작물의 목록을 기반으로 즉시 제외 처리 (블랙리스트 기반 필터링).
  - 추후 학습(Fine-tuning) 및 RAG(Retrieval-Augmented Generation) 시스템 구성 시 실시간으로 참조하여 결과물에 포함되지 않도록 차단.
- **처리 투명성**: Opt-out 요청 후 처리 결과(데이터셋 제외 및 캐시 삭제)를 즉시 신청자에게 통지하고 기록을 남긴다.

## 데이터 거버넌스 및 윤리 가이드라인

데이터 거버넌스 체계를 통해 저작물의 활용 범위를 명확히 규정하고, 무단 도용 방지 장치를 마련한다.

- **출처 명시 의무화**: 모델이 생성한 시 또는 CoT(Chain-of-Thought) 창작 노트에서 특정 시인의 스타일을 차용했거나 영향을 받았을 경우, 참고한 메타데이터/원작 정보를 추적하여 표기할 수 있는 구조 설계.
- **생성물의 Novelty 한계 설정**: 기존 작품과 임베딩 거리(Cosine Similarity)가 일정 수준 이상(예: 0.85 이상) 가까울 경우, 이를 모방으로 간주하고 생성 필터링(Reject) 처리.
- **저작권 협의 기구 연계**: 주요 문인 협회 및 출판사와 사전 협의하여, 명시적인 Opt-in 권장 프로그램과 더불어 보편적 Opt-out 권리를 병행.

## 시스템 아키텍처 연동

- **DB 관리**: `opt_out_registry` DB를 두어 시인명, 작품명, 출판사, ISBN 등의 해시값을 저장.
- **학습 파이프라인**: 텍스트 데이터 로드 시 `opt_out_registry`와 교차 검증하여 일치하는 텍스트 블록은 마스킹 혹은 드롭.
- **생성 파이프라인**: 최종 텍스트 출력 전, 생성된 텍스트가 Opt-out 리스트와 유사도/n-gram 중복을 가지지 않는지 검사하는 안전망 구축.
