# 데이터 수집 전략

## 저작권 트랙

### Track A — 공공 도메인 (즉시 사용 가능)
- 저자 사망 후 70년 이상 경과 작품
- 위키문헌, Project Gutenberg, Internet Archive
- → [korean_classical.md](korean_classical.md), [foreign_poetry.md](foreign_poetry.md)

### Track B — 오픈 라이선스
- CC 라이선스 저작물
- 저자/출판사가 연구 목적 사용 허락한 텍스트
- 한국문학번역원(KLTI) 자료

### Track C — 연구 목적 협의
- 주요 출판사(문학과지성사, 창작과비평사, 문학동네 등)와 협의
- 시인 유족/에이전시 접촉
- 대학 도서관 연구 협약 활용

### Track D — 공정 이용 (Fair Use) 검토
- 소량 인용 기반 학습 데이터 생성의 법적 지위 검토
- 한국 저작권법 제28조 (교육/연구 목적 이용) 검토
- **법률 자문 필요** — 특히 AI 학습용 스캔에 관한 최신 판례

## 스캔 파이프라인

```
1. 물리적 시집 입수
   └── 도서관 대출 / 구매 / 기증

2. 스캔
   └── 고해상도 (300 DPI 이상)
   └── 행/연 구분이 명확하게 보이도록

3. OCR
   └── 도구: Tesseract (한국어), Naver Clova OCR, ABBYY
   └── 특수 케이스: 타이포그래피 실험시 수동 교정 필요

4. 후처리
   └── 행갈이 정규화
   └── 연갈이 표시
   └── 메타데이터 삽입 (시집명, 시인, 연도, 쪽수)

5. 품질 검증
   └── 샘플 수동 검토
   └── 원문 대조
```

## OCR 주의 사항

### 한국어 시 특수성
- **행갈이**: OCR이 한 시행을 두 줄로 읽는 오류 빈발
- **연갈이**: 빈 줄이 OCR 처리 과정에서 사라질 수 있음
- **들여쓰기**: 일부 시는 들여쓰기로 리듬을 표현 → 보존 필요
- **타이포그래피 실험** (황지우 등): 수동 처리 필수

### 고해상도 스캔의 중요성
저화질 스캔은 OCR 정확도를 크게 낮춤.
특히 한자 혼용 시, 세로쓰기 시는 300 DPI 미만에서 오류 급증.

## 메타데이터 표준

모든 시에 다음 메타데이터 첨부:

```yaml
시인: 홍길동
시집: 시집 제목
출판사: 출판사명
출판연도: 1990
시제목: 시 제목
원고지면: p.45
저작권상태: 보호중 | 공공도메인 | 협의완료
언어: ko
장르: 현대시
```

## 우선순위 행동 목록

1. 한국 공공도메인 시 수집 (즉시 가능)
2. Project Gutenberg / Wikisource 외국어 시 수집 (즉시 가능)
3. 주요 출판사 협의 시작 (Track C)
4. OCR 파이프라인 테스트 (소규모 파일럿)
5. 저작권 법률 자문 (Track D 결정)
