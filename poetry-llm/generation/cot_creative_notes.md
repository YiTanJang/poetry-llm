# 창작 노트(CoT) 설계

## 핵심 아이디어

> 시인은 즉흥적으로 시를 쓰지 않는다.
> 쓰기 전에 — 혹은 쓰면서 — 아이디어를 구체화하고,
> 실패한 표현을 버리고, 언어를 선택하고, 구조를 설계한다.

이 창작 과정을 모델이 외화(externalize)하도록 학습시키는 것이
CoT 창작 노트의 목표다. 결과물의 추적 가능성을 높이고,
수정 루프에서 창작 노트를 기준으로 시를 개선할 수 있다.

## 창작 노트의 구조

### 초기 아이디어 포착
```
## 씨앗 이미지
오늘 지하철에서 본 것: 잠든 남자의 손 위에 낙엽이 한 장 떨어졌다.
아무도 치우지 않았다. 남자는 모른다.
→ 무지와 아름다움이 공존하는 순간
```

### 개념 구체화
```
## 개념 탐색
- 낙엽 + 잠 → 계절과 의식의 교차
- 손 → 행위의 도구, 지금은 쉬고 있음
- 지하철 → 일시성, 이동, 모두가 통과하는 공간
- 무지 → 비극인가, 축복인가?

## 다루고 싶은 것
직접 말하지 않고 이 순간 자체가 말하게 하고 싶다.
"아름답다"는 말 없이 아름다움이 느껴지게.
```

### 수사/기법 계획
```
## 형식 결정
- 짧게: 너무 설명하면 순간이 죽는다
- 행갈이: 시선의 이동에 따라 끊기
- 시제: 현재형 — 지금 벌어지는 일처럼

## 피해야 할 것
- "낙엽처럼" — 직유 금지
- "쓸쓸하다" — 감정 직접 서술 금지
- 결론 달지 않기
```

### 초안
```
## 초안
잠든 남자의 손 위에
낙엽이 내린다
남자는 모른다
가을이 그를 택했다는 걸
```

## 창작 노트 형식 (JSON)

```json
{
  "씨앗이미지": "...",
  "개념탐색": ["...", "..."],
  "다루고싶은것": "...",
  "형식결정": {
    "길이": "짧게",
    "행갈이원칙": "시선 이동에 따라",
    "시제": "현재형"
  },
  "금지목록": ["직유", "감정 직접 서술", "결론"],
  "초안": "[시 텍스트]"
}
```

## 창작 노트의 진화

창작 노트는 수정 과정에서 함께 변형된다:

```
Round 1: 초기 창작 노트 + 시 초안
Round 2: "초안의 4행이 설명적이다" → 창작 노트 업데이트 → 시 수정
Round 3: "가을이라는 단어가 직접적" → 노트 추가 → 재수정
```

→ [iterative_refinement.md](iterative_refinement.md) 참조

## 학습 데이터 생성 방법

실제 시인의 창작 노트는 드물다. 대안:

1. **역추론**: 완성된 시 + 시인의 에세이/인터뷰 → LLM으로 창작 노트 역추론
2. **시론 활용**: 시론에서 발췌한 원칙을 특정 시에 적용한 창작 노트 생성
3. **시인 인터뷰 변환**: 인터뷰에서 창작 배경을 설명하는 부분을 구조화
