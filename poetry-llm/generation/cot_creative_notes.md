# 창작 노트(CoT) 설계

## 핵심 아이디어

> 시인은 즉흥적으로 시를 쓰지 않는다.
> 쓰기 전에 — 혹은 쓰면서 — 아이디어를 구체화하고,
> 실패한 표현을 버리고, 언어를 선택하고, 구조를 설계한다.

이 창작 과정을 모델이 외화(externalize)하도록 학습시키는 것이
CoT 창작 노트의 목표다. 결과물의 추적 가능성을 높이고,
수정 루프에서 창작 노트를 기준으로 시를 개선할 수 있다.

## 창작 노트의 구조 (자유형 스크래치패드)

모델은 지정된 템플릿(JSON이나 고정된 섹션)을 채우는 대신, 시작점인 `씨앗:`과 제약조건인 `금지:` 사이에서 자유롭게 창작 과정을 탐색합니다.

```text
씨앗: 오늘 지하철에서 본 것: 잠든 남자의 손 위에 낙엽이 한 장 떨어졌다. 아무도 치우지 않았다. 남자는 모른다.

낙엽 + 잠 → 계절과 의식의 교차.
남자는 모른다 — 무지와 아름다움이 공존하는 순간이다.
직접적으로 아름답다고 말하지 않고, 이 순간 자체가 말하게 하고 싶다.

형식: 너무 설명하면 순간이 죽는다. 짧게 쓰자.
시선의 이동에 따라 행을 끊고, 현재형으로 써서 지금 벌어지는 일처럼 느껴지게 만들자.

금지: "낙엽처럼"(직유), "쓸쓸하다"(감정 직접 서술), 결론 달기
```

위와 같은 자유형 탐색 과정을 거친 후 최종 초안이 도출됩니다. JSON 형식의 엄격한 구조화는 프론티어 모델의 확장 사고(extended thinking) 트렌드와 맞지 않으므로 폐기합니다.

## 창작 노트의 진화

창작 노트는 수정 과정에서 함께 변형된다:

```
Round 1: 초기 창작 노트 + 시 초안
Round 2: "초안의 4행이 설명적이다" → 창작 노트 업데이트 → 시 수정
Round 3: "가을이라는 단어가 직접적" → 노트 추가 → 재수정
```

→ [iterative_refinement.md](iterative_refinement.md) 참조

## 학습 데이터 생성 방법

실제 시인의 창작 노트는 드물다. 대안:

1. **역추론**: 완성된 시 + 시인의 에세이/인터뷰 → LLM으로 창작 노트 역추론
2. **시론 활용**: 시론에서 발췌한 원칙을 특정 시에 적용한 창작 노트 생성
3. **시인 인터뷰 변환**: 인터뷰에서 창작 배경을 설명하는 부분을 구조화
