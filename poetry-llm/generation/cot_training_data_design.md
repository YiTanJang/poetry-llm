# 학습 데이터 생성 — 다양한 트레이스, 외부 판정, DPO 파이프라인

## 이전 설계에서 무엇이 바뀌었는가

이전 설계는 두 가지 핵심 가정에 의존했다:

**가정 1:** 역추론(완성된 시 → 창작 노트 생성)으로 학습 데이터를 만든다.
**가정 2:** 품질 필터는 규칙 기반 하드 필터(n-gram 겹침, 핵심개념 길이 등) + LLM 소프트 필터로 구성한다.

두 가정을 모두 수정한다.

**가정 1 수정:** 역추론 데이터도 사용하지만, 주된 경로는 **3 시인 페르소나의 직접 생성**이다. 역추론은 기존 시 코퍼스를 활용하는 보조 방법으로 남는다.

**가정 2 수정:** 규칙 기반 필터를 제거하고 **훈련된 보상 모델(discriminator)**로 교체한다. 인간 선호 데이터를 학습한 소형 모델이 품질을 판정한다.

---

## 전체 파이프라인 구조

```
[Phase 1] 씨앗 소재 수집
          한국 현대시 코퍼스 + 생성된 프롬프트
                │
[Phase 2] 3-시인 트레이스 생성
          페르소나 A/B/C가 독립적으로 (스크래치패드 + 시) 생성
          소재당 3개 트레이스
                │
[Phase 3] 외부 판정
          Claude / GPT-4가 3개 트레이스를 비교 평가
          승자 1개 선택, 패자 2개 식별
                │
[Phase 4] 선호 쌍 구성
          chosen: 승자 (스크래치패드 + 시)
          rejected: 패자 (스크래치패드 + 시)
          소재당 최대 2개 DPO 쌍
                │
[Phase 5] 보상 모델 필터링
          훈련된 보상 모델이 chosen 샘플의 절대 품질을 평가
          임계값 미달 샘플 제거 (낮은 품질의 승자도 제거)
                │
[Phase 6] SFT 데이터 + DPO 데이터 분리
          SFT: 상위 품질 (스크래치패드 + 시) 쌍 전체
          DPO: 선호 쌍 전체
```

## Phase 2: 트레이스 생성 세부 설계

### 입력 다양화

같은 소재에 대해 여러 형식의 입력을 사용한다:

```
형식 1: 구체적 소재
"지하철 안 잠든 남자의 손 위에 낙엽이 내려앉은 장면으로 시를 써달라."

형식 2: 추상적 주제
"모르는 사이에 일어나고 지나가는 것들에 관한 시를 써달라."

형식 3: 이미지 목록
"손, 낙엽, 잠을 소재로 시를 써달라."

형식 4: 감정/분위기
"누군가 모르는 사이에 그를 스쳐간 것에 관한 시를 써달라."
```

같은 소재라도 입력 형식에 따라 트레이스가 다르게 전개된다.
다양한 입력 형식으로 생성하면 모델이 다양한 입력에 대응하는 능력도 함께 학습한다.

### 페르소나별 생성 설정

각 페르소나는 독립 실행이지만, 온도(temperature)를 다르게 설정하는 것도 고려한다:

| 페르소나 | 미학 지향 | 권장 온도 | 스크래치패드 기대 스타일 |
|---------|---------|---------|----------------------|
| A | 구체 감각/이미지 | 0.8 | 감각 묘사, 언어 선택 비교 |
| B | 추상/개념/낯선 논리 | 1.0 | 개념 탐색, 방향 전환 |
| C | 형식 실험 | 0.9 | 형식 결정 우선, 배치 실험 |

온도 설정은 초기값이다. 실험으로 조정한다.

## Phase 3: 외부 판정 세부 설계

### 판정 프롬프트

```
다음 세 편의 시와 각각의 스크래치패드를 읽어라.
모두 동일한 소재에서 독립적으로 생성되었다.

[소재]
---
{소재 텍스트}
---

[시 A + 스크래치패드 A]
---
{스크래치패드A}

{시A}
---

[시 B + 스크래치패드 B]
---
{스크래치패드B}

{시B}
---

[시 C + 스크래치패드 C]
---
{스크래치패드C}

{시C}
---

각 시를 다음 기준으로 1~5점 평가하라:

1. 신선함 (1~5): 이 이미지 조합 또는 언어 선택이 기존 한국 현대시에서 드문가
2. 필연성 (1~5): 이 시의 각 언어 선택이 대체 불가능하게 느껴지는가
3. 여운 (1~5): 시를 다 읽은 후 무언가 남는가
4. 트레이스 정직성 (1~5): 스크래치패드의 탐색이 실제로 이 시로 이어진 것처럼 보이는가

합산 점수가 가장 높은 시를 승자로 선택하라.
동점이면 신선함 점수를 우선한다.

출력 형식:
- 승자: A | B | C
- 각 시의 점수 (신선함/필연성/여운/트레이스정직성)
- 승자 선택 이유 (2~4문장)
- 패자 각각의 주요 약점 (1~2문장씩)
```

### 판정 신뢰도

외부 판정 모델은 완벽하지 않다. 신뢰도를 높이는 방법:

- **복수 판정 모델:** Claude와 GPT-4 두 모델이 독립 판정 → 일치하는 경우만 사용 (또는 앙상블 평균)
- **무작위 순서:** 시 A/B/C의 순서를 무작위로 섞어 제시 → 순서 편향 제거
- **최소 점수 차이:** 승자와 차점 패자의 합산 점수 차이가 2점 미만이면 "판정 불명확"으로 처리 → DPO 쌍에서 제외

## Phase 5: 보상 모델 — 규칙 대신 학습된 판별기

### 왜 규칙 기반 필터를 제거하는가

이전 설계는 n-gram 겹침, 핵심개념 길이, 금지 항목 위반 여부 같은 규칙으로 품질을 판단했다.

**문제:**
- 좋은 시가 n-gram 기준을 위반할 수 있다 (새로운 언어 조합은 낯선 n-gram을 사용한다)
- 나쁜 시가 규칙을 통과할 수 있다 (규칙 준수 ≠ 미학적 품질)
- "novelty"는 n-gram으로 포착되지 않는다

**대안:** 인간 선호를 학습한 소형 보상 모델(reward model)이 품질을 판단한다.

### 보상 모델 설계

**아키텍처:** 소형 인코더 모델 (예: 수백 MB 규모의 한국어 BERT 계열)
- 입력: (스크래치패드 + 시) 쌍
- 출력: 품질 점수 (스칼라)

**학습 데이터:**
1단계: 외부 판정 모델(Claude/GPT-4)의 점수 데이터로 초기 학습
2단계: 인간 평가자의 소규모 선호 데이터로 추가 학습 (RLHF 스타일)

**학습 신호 (1단계):**
외부 판정 모델이 각 시에 부여한 4개 점수(신선함/필연성/여운/트레이스정직성)의 합산이 보상 모델의 학습 레이블이 된다.

```
보상 모델 학습 예시:
  입력: [스크래치패드A + 시A]
  레이블: 판정 모델 합산 점수 / 20 (0~1 정규화)
```

### 보상 모델의 역할

Phase 5에서 보상 모델은 두 가지를 한다:

1. **절대 품질 임계값 적용:** 외부 판정에서 승자가 됐더라도 보상 모델 점수가 낮으면 제거한다. "상대적으로 가장 좋은" 샘플이 "절대적으로 충분히 좋은" 샘플과 다를 수 있다.

2. **DPO 쌍의 마진 확인:** chosen 점수와 rejected 점수 차이가 충분히 크지 않으면 해당 쌍을 DPO에서 제외한다. 모호한 선호 쌍은 학습 신호가 약하다.

```
임계값 (초기값, 실험으로 조정):
  SFT 포함 조건: 보상 모델 점수 ≥ 0.7
  DPO 포함 조건: chosen 점수 - rejected 점수 ≥ 0.15
```

## Phase 6: SFT 데이터와 DPO 데이터 구성

### SFT 데이터 포맷

```json
{
  "messages": [
    {
      "role": "system",
      "content": "당신은 한국 현대시를 쓰는 시인이다. 시를 쓰기 전에 자유형 스크래치패드를 작성하라. 스크래치패드에는 반드시 씨앗(출발이 되는 이미지나 감각)과 금지(쓰지 않을 것들)를 포함한다. 나머지는 형식 없이 자유롭게 탐색하라."
    },
    {
      "role": "user",
      "content": "{소재 입력}"
    },
    {
      "role": "assistant",
      "content": "씨앗: {씨앗}\n\n{자유형 스크래치패드}\n\n금지: {금지}\n\n{시}"
    }
  ]
}
```

어시스턴트 토큰은 스크래치패드부터 시까지 전체를 커버한다.
모델은 사고하고 생성하는 전체 과정을 학습한다.

### DPO 데이터 포맷

```json
{
  "prompt": "{소재 입력}",
  "chosen": "씨앗: ...\n\n{스크래치패드 chosen}\n\n금지: ...\n\n{시 chosen}",
  "rejected": "씨앗: ...\n\n{스크래치패드 rejected}\n\n금지: ...\n\n{시 rejected}"
}
```

DPO는 SFT 이후에 적용된다.
SFT로 기본 포맷과 품질 수준을 확보한 뒤,
DPO로 선호 방향(더 신선하고, 더 필연적이고, 더 여운이 남는 방향)을 학습한다.

## 역추론(Reverse-Inference) 데이터 생성 파이프라인

기존 한국 현대시 코퍼스에서 고품질 시를 추출하여 역으로 스크래치패드(창작 노트)를 재구성하는 역추론 데이터는 SFT 학습의 풍부성을 위해 필수적이다. 완성된 시의 미학적 다양성을 확보하기 위해 다음 4단계 역추론 파이프라인을 구축한다.

### 1단계: 시 해체 및 속성 추출 (Deconstruction)
*   **입력:** 현대시 코퍼스의 최종 완성 시
*   **추출 대상:**
    *   **씨앗(Seed):** 시적 발상이 시작되었을 법한 최소 단위의 관찰, 핵심 이미지, 감각적 모티프.
    *   **개념적 금지어(Prohibitions):** 해당 시를 쓸 때 작가가 의도적으로 기피하거나 배제했을 상투적 어휘나 클리셰 비유 목록.
    *   **잠재 페르소나:** 해당 시의 스타일(구체 묘사 중심, 개념 중심, 실험 중심)에 부합하는 시인 페르소나 설정.

### 2단계: 역퇴행 초안 생성 (Draft Regression)
*   최종 완성 시를 바로 사용하여 사고 과정을 역추론하면 '정답을 아는 편향'이 발생하므로, 의도적으로 질이 떨어지는 '초안(Raw Draft)'을 먼저 생성한다.
*   최종 시의 가장 강력하고 낯선 은유나 결말부를 상투적인 문장으로 변형하여 원시적인 초기 구상을 모델링한다.

### 3단계: 점진적 퇴고 트레이스 작성 (Iterative Revision Trace Generation)
*   초안에서 시작하여 최종 완성 시에 도달하기까지의 사고 과정을 생성한다.
*   중간 단계마다 시어의 삭제, 행갈이 수정, 제약 조건 위반 여부 점검 등의 구체적 피드백 루프를 스크래치패드에 서술하도록 강제한다.

### 4단계: 모순 검증 및 포맷팅 (Validation & Formatting)
*   **정합성 필터:** 스크래치패드에서 금지어로 설정한 단어가 최종 시에 들어있거나, 스크래치패드에서 계획한 형식이 최종 시에서 지켜지지 않은 정합성 오류 샘플을 자동 필터링한다.
*   **SFT 최종 포맷화:** 검증 완료된 (씨앗 + 금지 + 스크래치패드 + 시) 형태를 JSON으로 가공하여 저장한다.

## 정답을 알고 쓰는 편향(Smooth-Inference Bias) 완화 방법론

역추론이나 합성 CoT 생성 시 발생하는 가장 큰 한계는 모델이 최종 결과물(시)을 이미 인지한 상태에서 사고의 흐름을 짜 맞춘다는 점이다. 이로 인해 스크래치패드가 실제 인간의 창작 과정과 달리 너무 매끄럽고, 결정론적이며, 우연성이 결여된 '정답 지향적 사고(Smooth-Inference)'에 빠지게 된다. 이를 방지하고 '생산적인 헤맴'을 모방하기 위해 다음 네 가지 구체적 기법을 강제한다.

### 1. 인위적 머뭇거림 및 갈등(Artificial Hesitation) 주입
생성 프롬프트 내에 최소 1회 이상의 어휘 갈등 단계를 작성하도록 제한 조건을 설정한다.
*   **예시 템플릿:** `"여기서 [단어 A]를 쓰면 감정이 너무 날것으로 드러난다. 대신 [단어 B]를 써서 감정을 물화(reify)하는 것이 좋을까? 아니면 아예 문장을 생략하고 비워둘까? 리듬상 B가 더 나으니 B를 채택하되 앞 구절을 다듬자."`
*   특수 지시를 통해 스크래치패드 내에 사유의 단절이나 대안 비교를 강제함으로써, 모델이 시어 선택 과정의 심사숙고를 학습하게 한다.

### 2. 실패한 초안 및 자가 교정(Self-Correction) 시나리오
의도적으로 실패한 변주를 생성한 뒤, 퇴고 과정에서 이를 수정해 나가는 패턴을 트레이스에 의무화한다.
*   **기법:**
    1.  스크래치패드 중반부에 `[시도]` 블록을 열어 상투적이거나 정형화된 연을 작성한다.
    2.  이후 `[반성/분석]` 블록에서 해당 시도가 실패한 이유(예: "너무 유치한 비유다", "리듬이 깨진다", "금지어와 겹친다")를 서술한다.
    3.  `[수정]` 블록에서 단어 순서를 바꾸거나, 낯선 시어로 대체하는 정밀 퇴고 트레이스를 보여준다.

### 3. 노이즈 및 우연성(Noisy & Contingency) 주입
작가 페르소나의 주의력 분산이나 주변 환경의 개입, 혹은 무작위적 연상을 모사하는 노이즈를 트레이스에 섞는다.
*   프롬프트 생성 매개변수에 무작위 우연성 키워드를 제공한다. (예: `돌발 소재: 깨진 유리, 지나가는 바람 소리`)
*   시인은 스크래치패드 작성 도중 이 돌발 요소를 깨닫고, 원래 쓰려던 방향과 결합하여 낯선 미학적 비약을 이끌어내도록 구성한다.
*   **예시:** `"잠에 대해 쓰고 있었는데 문득 책상 위 깨진 유리컵이 눈에 들어온다. 잠을 깨진 유리의 파편들에 비유할 수 없을까? 잠의 틈새로 뾰족한 것들이 밀려들어오는 이미지로 방향을 급선회해보자."`

### 4. 정보 비대칭(Information Asymmetric) 2-Pass 프롬프팅
교사(Teacher) LLM에게 한 번에 완성 시를 다 보여주지 않고 정보 격차를 두어 CoT를 생성하는 아키텍처를 도입한다.
*   **Pass 1 (Blind Draft Generator):** 교사 모델에게 시의 제목, 씨앗 소재, 그리고 1연의 초안만을 제공하고 2연과 3연에 들어갈 수 있는 다양한 시적 탐색 노트와 가상의 스크래치패드를 작성하게 한다.
*   **Pass 2 (Context Integrator):** 1차로 작성된 거친 탐색 노트와 실제 코퍼스의 전체 시를 교사 모델에게 함께 준다. 그리고 `"당신이 Pass 1에서 구상했던 다양한 생각 중에서 어떤 고민을 거쳐 최종적으로 Pass 2의 완성 시로 이행하게 되었는지, 논리적이고 미학적인 징검다리를 놓아라"`라고 요청한다.
*   이를 통해 사후 확신 편향(Hindsight Bias)이 근본적으로 차단된, 생생하고 입체적인 창작 노트를 획득할 수 있다.

## 미결 사항

- [Ph1] 역추론 정보 비대칭 2-Pass 프롬프팅에서 1차 Pass 초안의 다양성 제어: 1차 Pass에서 LLM이 너무 엉뚱한 방향으로 가면 2차 Pass에서 실제 코퍼스의 완성 시로 수렴할 때 논리적 단절이 지나치게 커질 수 있다. 초안의 '적절한 오차 범위'를 어떻게 정량화하고 통제할 것인가?
- [Ph1] 인위적 노이즈 및 머뭇거림 패턴의 모델 전이 여부: SFT 학습을 통해 모델이 스크래치패드에서 보여준 '실패 후 극복' 행동 패턴을 성공적으로 습득하는지, 아니면 단지 '의미 없는 무작위 문장 삽입'으로만 간주하여 노이즈로 작용하게 되는지 검증할 수 있는 평가 메트릭은 무엇인가?
- [Ph1] 보상 모델(Reward Model)의 편향 완화 검증: 편향이 완화된 CoT 트레이스(머뭇거림, 퇴고가 포함된 복잡한 경로)가 보상 모델 평가에서 오히려 '일관성 없음'이나 '낮은 정합성'으로 오판받을 위험이 있다. 보상 모델이 '완벽하고 매끄러운 트레이스'보다 '현실적으로 갈등하는 트레이스'를 고평가하도록 보상 루브릭을 어떻게 수정해야 하는가?
