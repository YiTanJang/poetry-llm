# CoT 학습 데이터 실제 제작 방법

## 문제 상황

학습 데이터의 핵심은 **[창작 노트 → 시]** 쌍이다.
그러나 실제 시인의 창작 노트는 거의 공개되지 않는다.
윤동주의 메모, 최승자의 초고 일부가 있지만 규모가 매우 작다.

결론: 창작 노트를 역추론으로 생성해야 한다.
완성된 시 → LLM으로 역추론 → 그 시를 썼을 법한 창작 노트 생성

이 방법의 위험은:
- 역추론이 "답을 알고 쓰는 창작 노트"가 되어 너무 매끄러울 수 있다
- 실제 창작 과정의 불확실성, 실패, 탐색이 빠질 수 있다
- 시에 없는 내용이 창작 노트에 등장할 수 있다 (할루시네이션)

이를 해결하는 품질 보장 방법을 설계한다.

---

## 역추론 프롬프트 설계

### 프롬프트 A: 기본 역추론 (씨앗이미지 중심)

```
당신은 숙련된 시인이다.
아래 완성된 시를 읽고, 이 시를 쓰기 전에 당신이 작성했을 법한 창작 노트를 복원하라.

[시]
---
잠든 남자의 손 위에
낙엽이 내린다
남자는 모른다
---

규칙:
1. 씨앗이미지: 이 시의 시작이 되었을 구체적 감각 경험을 1~3문장으로 서술하라.
   "낙엽"이나 "잠든 남자"라는 단어 자체가 아니라, 그것을 발견한 맥락과 감각을 서술하라.

2. 핵심개념: 이 시가 실제로 말하는 것을 1문장으로 압축하라.
   시에 나온 단어를 그대로 쓰지 마라.

3. 언어후보: 이 시를 쓰기 전에 고려했을 법한 표현 후보 3~5개를 제시하라.
   최종적으로 선택되지 않은 후보도 포함하라.

4. 형식결정: 길이(short/medium/long), 구조(자유시/연시/산문시), 시제 선택의 이유를 서술하라.

5. 금지: 쓰지 않기로 결정했을 표현을 최소 2개 제시하라.
   이 시에 실제로 등장하지 않는 표현이어야 한다.

주의: 역추론이지만 탐색 과정의 불확실성을 포함하라.
      너무 매끄러운 창작 노트는 실제 창작 과정을 반영하지 못한다.
```

### 프롬프트 B: 불확실성 강조 역추론

```
당신은 시를 분석하는 편집자다.
아래 시를 읽고, 이 시인이 쓰기 전에 무엇을 고민했을지 역추론하라.
단, 모르는 것은 솔직하게 "이 부분은 불분명하다"고 표시하라.

[시]
---
[시 텍스트]
---

다음 형식으로 작성하라:

씨앗이미지: [확실한 것 + 추측인 것을 구분하여 서술]
핵심개념: [1문장, 시에 없는 단어로]
포기한 방향: [이 시인이 고려했다가 포기했을 법한 접근 2~3가지]
언어후보: [선택된 것과 탈락된 것을 모두]
형식결정: [왜 이 형식인가]
금지: [이 시에 없는 표현들 — 의도적 배제로 보이는 것]
불확실: [이 역추론에서 확신하기 어려운 부분]
```

### 프롬프트 C: 다중 해석 역추론

```
아래 시에 대해 서로 다른 두 가지 창작 노트를 역추론하라.
두 창작 노트는 같은 시를 썼지만 출발점이 달랐을 두 버전이다.

[시]
---
[시 텍스트]
---

버전 1: [이 시인이 도시적 고독에서 출발했다면]
버전 2: [이 시인이 시간의 무관심함에서 출발했다면]

각 버전은 완전한 창작 노트 형식을 갖춰야 한다.
씨앗이미지, 핵심개념, 형식결정, 금지를 모두 포함하라.
```

**프롬프트 A는 표준 역추론에 사용, B는 탐색적 분석이 필요한 복잡한 시에 사용,
C는 데이터 다양성 확보를 위해 한 시당 여러 버전 생성 시 사용.**

## 역추론 결과 품질 기준

### 좋은 창작 노트의 조건

**1. 씨앗이미지의 감각 구체성**
- 씨앗이미지가 시에 등장하는 단어를 그대로 반복하지 않아야 한다
- 감각(시각, 청각, 촉각 등)이 명시되어야 한다
- 시보다 더 많은 디테일을 포함해야 한다 (시는 압축된 결과물이므로)

나쁜 예: `씨앗이미지: 낙엽이 손 위에 떨어진 장면`
좋은 예: `씨앗이미지: 출근 후 텅 빈 지하철 안, 맞은편에 잠든 중년 남자. 형광등이 납빛으로 그의 손등을 비추고 있었다. 어느 순간 낙엽 한 장이 열린 창문으로 들어와 그 손 위에 내려앉았는데, 흔들리는 전동차에서도 낙엽은 미끄러지지 않았다.`

**2. 핵심개념의 추상화 수준**
- 시에 등장하는 단어를 그대로 쓰면 안 된다
- 시보다 한 단계 더 추상화되어야 한다
- 문장 길이: 20~60자

나쁜 예: `핵심개념: 낙엽이 남자 손 위에 내려앉고 남자는 모른다`
좋은 예: `핵심개념: 모르는 사이에 아름다운 일이 일어나고 지나간다`

**3. 언어후보의 탐색 흔적**
- 최종 시에 채택된 표현과 탈락된 표현이 모두 있어야 한다
- 탈락 이유가 명시되면 더 좋다
- 후보 간 뉘앙스 차이가 실질적이어야 한다 (동의어 나열 불가)

**4. 금지 항목의 실질성**
- 금지 항목이 실제로 시에 등장하지 않아야 한다 (당연히)
- 금지 항목이 너무 포괄적이면 안 된다 ("진부한 표현" 같은 것)
- 최소 하나는 이 시가 쓰일 맥락에서 실제로 유혹적인 표현이어야 한다

**5. 불확실성의 포함**
- 너무 매끄럽고 완벽한 창작 노트는 역추론의 산물임을 의심한다
- 적당한 망설임, 복수 시도, 의문 표현이 있어야 한다

## 자동 필터링 방법

### 규칙 기반 필터 (Hard Filter)

```python
def hard_filter(creative_note, poem):
    """
    명백히 나쁜 역추론을 걸러내는 규칙 기반 필터.
    모두 통과해야 학습 데이터로 사용.
    """
    checks = []

    # 1. 씨앗이미지가 시의 단어를 50% 이상 반복하는가
    poem_words = set(poem.split())
    seed_words = set(creative_note['씨앗이미지'].split())
    overlap = len(poem_words & seed_words) / len(poem_words)
    checks.append(('씨앗이미지 과잉반복', overlap < 0.5))

    # 2. 핵심개념 길이 기준 충족
    concept_len = len(creative_note['핵심개념'])
    checks.append(('핵심개념 길이', 15 <= concept_len <= 80))

    # 3. 금지 항목이 실제로 시에 없는가
    violations = [f for f in creative_note['금지'] if f in poem]
    checks.append(('금지항목 위반 없음', len(violations) == 0))

    # 4. 언어후보가 최소 3개인가
    checks.append(('언어후보 최소수', len(creative_note.get('언어후보', [])) >= 2))

    return all(v for _, v in checks), checks
```

### 모델 기반 필터 (Soft Filter)

하드 필터를 통과한 샘플에 대해 별도 LLM으로 품질 점수 평가:

```
다음 창작 노트와 시의 쌍을 평가하라.

[창작 노트]
---
{creative_note}
---

[시]
---
{poem}
---

1~5점으로 각 항목을 채점하라:

a. 씨앗이미지의 감각 구체성 (1: 추상적/반복적, 5: 생생하고 독립적)
b. 핵심개념의 추상화 적절성 (1: 시를 그대로 반복, 5: 한 단계 높은 추상)
c. 창작 노트와 시의 인과성 (1: 연결 불분명, 5: 노트가 시를 만들 수 있을 것 같음)
d. 탐색 과정의 진정성 (1: 너무 매끄러움, 5: 실제 고민이 느껴짐)

총점 14점 이상인 경우만 학습 데이터로 승인.
```

### 필터 임계값 기준

| 등급 | 조건 | 처리 |
|------|------|------|
| A | 하드 필터 통과 + 모델 점수 16점 이상 | 학습 데이터로 즉시 사용 |
| B | 하드 필터 통과 + 모델 점수 14~15점 | 인간 검토 후 사용 |
| C | 하드 필터 통과 + 모델 점수 13점 이하 | 재생성 또는 폐기 |
| D | 하드 필터 탈락 | 자동 폐기 |

## 한 편의 시에서 여러 버전 생성 (다양성 확보)

단일 시에서 하나의 창작 노트만 만들면 데이터셋이 편향된다.
같은 시에 대해 서로 다른 해석과 출발점에서 여러 창작 노트를 만든다.

### 다양성 축

**1. 씨앗이미지 다양화**
같은 시가 다른 감각 경험에서 출발할 수 있다:
- 버전 A: 도시 장면에서 출발
- 버전 B: 추상적 관계에서 출발
- 버전 C: 다른 시를 읽다가 촉발

**2. 핵심개념 다양화**
같은 시가 다른 주제를 가졌을 수 있다:
- 버전 A: "모르는 사이에 일어나는 것들에 관해"
- 버전 B: "선택받음의 역설에 관해"
- 버전 C: "계절과 의식의 비동기에 관해"

**3. 형식 결정 경로 다양화**
같은 형식에 도달하는 다른 이유:
- 버전 A: "짧게 — 순간이기 때문에"
- 버전 B: "짧게 — 그 이상은 설명이 되기 때문에"

**4. 실패한 시도 포함 버전**
실제 창작 과정에서는 초안이 실패한다. 이를 반영:
- 처음엔 다른 방향으로 쓰다가 이 시가 나온 버전

### 권장 비율

시 한 편당: 2~4개 창작 노트
- 표준 버전 1개 (프롬프트 A)
- 다른 핵심개념 버전 1개 (프롬프트 C)
- 불확실성 강조 버전 1개 (프롬프트 B, 선택적)

총 학습 데이터: 시 N편 × 평균 2.5개 = 2.5N 샘플

```json
  ]
```


### 입력 소재 형식의 다양화

```json
// 형식 1: 구체적 소재
{ "role": "user", "content": "지하철 안 잠든 남자의 손 위에 낙엽이 내려앉은 장면으로 시를 써달라." }

// 형식 2: 추상적 주제
{ "role": "user", "content": "무지와 아름다움이 공존하는 순간에 관한 시를 써달라." }

// 형식 3: 감정/분위기
{ "role": "user", "content": "모르는 사이에 지나가버린 것들에 대한 시를 써달라." }

// 형식 4: 이미지 제시
{ "role": "user", "content": "손, 낙엽, 잠을 소재로 시를 써달라." }
```

같은 시에 대해 다양한 입력 형식의 샘플을 만들어 모델이 다양한 입력에 대응하게 한다.

## Ablation 설계: 창작 노트 없이 직접 생성 vs CoT 생성 비교

### 실험 목적

창작 노트(CoT)가 실제로 생성 품질을 높이는가, 아니면 없어도 좋은 결과가 나오는가.

### 실험 조건

| 조건 | 설명 |
|------|------|
| Baseline | 소재 입력 → 즉시 시 생성 (창작 노트 없음) |
| CoT-Full | 소재 입력 → 전체 창작 노트(8단계) → 시 생성 |
| CoT-Minimal | 소재 입력 → 필수 필드만 창작 노트 → 시 생성 |
| CoT-NoForbidden | 창작 노트에서 금지 필드만 제거 → 시 생성 |
| CoT-NoConcept | 씨앗이미지만 있고 핵심개념 없음 → 시 생성 |
| CoT-NoTension | 긴장관계 단계 없음 → 시 생성 |

### 평가 지표

**자동 평가:**
- Novelty 점수: 생성된 이미지 조합의 n-gram 코퍼스 유사도 역수
- 금지 항목 준수율: 기본 금지 패턴(직유, 감정 직접 서술, 진부 이미지) 위반 빈도
- 창작 노트-시 추적성 점수 (cot_schema.md 기준)

**인간 평가 (소규모):**
평가자 3인이 시를 조건 레이블 없이 읽고 채점:
- 신선함 (1~5): 이 이미지 조합을 본 적 있는가
- 밀도 (1~5): 불필요한 표현이 없는가
- 여운 (1~5): 읽은 후 무언가 남는가

**평가 대상:**
- 동일 소재 50개에 대해 각 조건별 시 생성
- 총 300편 (50 × 6조건)

### 가설

- H1: CoT-Full > Baseline (CoT가 품질을 높인다)
- H2: CoT-NoForbidden < CoT-Full (금지 필드가 기여한다)
- H3: CoT-NoConcept < CoT-Full (핵심개념이 기여한다)
- H4: CoT-Minimal ≈ CoT-Full (과도한 단계는 한계 수익이 낮다)

H4가 성립하면 추론 비용을 줄이기 위해 최소 버전을 채택할 수 있다.

### 단계별 기여도 측정

각 단계를 하나씩 제거하는 ablation:

```
실험 A: Stage 1 없음 (감각 포착 생략)
실험 B: Stage 2 없음 (개념 추상화 생략)
실험 C: Stage 3 없음 (긴장관계 생략)
실험 D: Stage 4 없음 (언어 탐색 생략)
실험 E: Stage 7 없음 (갭 분석 생략 → 수정 없음)
```

Stage를 제거했을 때 점수 하락폭이 가장 큰 단계가 가장 중요한 단계.
