# 하네스 설계 — 툴 지원 창작 파이프라인

## 설계 원칙

- **도구 사용 패턴 학습**: 학습 데이터에 창작 도구 호출을 포함하여, 모델이 스크래치패드에서 도구를 자유롭게 사용하는 방식을 학습한다.
- **자유형 스크래치패드**: 도구 사용 여부와 시점은 모델이 스스로 결정한다. 강제 단계 없음.
- **외부 판정 독립**: 생성 과정에서 사용한 도구와 무관하게 외부 모델이 최종 시만 평가한다.
- **인간 개입 지점 최소화**: 씨앗(소재) 제공과 금지어 설정이 기본. 나머지는 모델 자율.

---

## 하네스 역할 구조

```
[입력: 씨앗 + 금지어]
         │
    ┌────┴──────┬────────────┐
    ▼           ▼            ▼
  페르소나 A  페르소나 B   페르소나 C
  (감각형)   (개념형)    (형식실험형)
    │           │            │
  [스크래치패드 + 도구 호출 + 시]  ← 각자 독립, 서로 출력 못 봄
    └────────────┴────────────┘
                 │
         [외부 판정 모델]
         Claude / GPT-4
                 │
            [DPO 선호 쌍]
```

각 페르소나는 완전히 독립적으로 실행된다. 서로의 출력을 보지 않는다.

---

## 창작 도구 (Tool Harness)

스크래치패드 안에서 모델이 자유롭게 호출할 수 있는 도구들.
모든 도구는 선택적이다 — 호출 여부와 시점은 모델이 결정한다.

### 제공 도구 목록

| 도구 | 설명 | 반환값 예시 |
|------|------|------------|
| `random_words(n)` | 한국어 랜덤 단어 n개 생성 | ["녹슨", "마루타", "진공"] |
| `news_snippet(topic)` | 뉴스 헤드라인/문장 스크래핑 | "서울 지하 3층에서 발견된 1970년대 타임캡슐" |
| `dice(sides)` | n면 주사위 굴리기 | 4 |
| `syllable_count(text)` | 텍스트 음절 수 반환 | 7 |
| `creative_note_append(text)` | 창작 노트에 메모 추가 | OK |
| `creative_note_read()` | 현재 창작 노트 전체 조회 | "세탁기 드럼 속 형광빛..." |

### 창작 노트 (Creative Note)

- 모델이 스크래치패드 안에서 자유롭게 생성하고 기록하는 메모 공간.
- 시를 쓰기 전 아이디어를 탐색하거나, 시를 쓰는 중 발견한 것을 기록할 수 있다.
- **세션 단위 초기화**가 기본 — 한 시 생성 내에서만 유지. 세션 간 누적은 Phase 1 실험 대상.
- 사용자가 씨앗 외에 창작 노트 내용을 수동으로 제공하는 것도 가능 (선택).

### 학습 데이터에서의 도구 사용 패턴

도구 호출을 포함하는 학습 트레이스 예시:

```
씨앗: 새벽 4시 무인 세탁소

<tool: random_words(5)>
결과: ["형광", "파이프", "뒤집히다", "수신기", "거품"]

세탁기 회전 속 형광빛이 뒤집히는 이미지. 거품이 신호를 보내는 느낌.
금지: 외로움, 쓸쓸함, 기다리다

<시작>
형광 거품이 파이프를 타고
수신기처럼 올라간다
...
<끝>
```

도구를 쓰지 않은 트레이스도 학습 데이터에 포함 — 모델이 도구 없이도 쓸 수 있어야 한다.
학습 데이터에서 도구 사용 트레이스와 비사용 트레이스의 비율은 Phase 1에서 결정.

## 인간 개입 지점

| 지점 | 방식 | 기본값 |
|------|------|--------|
| 씨앗 제공 | 소재 / 이미지 / 상황 제공 | 필수 |
| 금지어 설정 | 쓰지 말아야 할 단어/표현 목록 | 선택 |
| 창작 노트 사전 제공 | 특정 분위기나 의도를 텍스트로 주입 | 선택 |
| 도구 결과 수동 삽입 | 뉴스 기사 등을 직접 제공 (tool 대신) | 선택 |

---

## 컨텍스트 격리

- **판정 모델**: 3개 페르소나의 최종 시 + 스크래치패드(도구 호출 이력 포함)를 모두 받는다.
- **페르소나 간**: 완전 격리. 한 페르소나의 출력이 다른 페르소나에게 전달되지 않는다.
- **도구 결과**: 스크래치패드 내부에 인라인으로 포함. 별도 컨텍스트 필드 없음.

---

## Phase 1 실험 대상

Phase 0에서는 아래 기능을 설계에 포함하지 않는다. Phase 1 ablation으로 검증:

- **비평가/독자 에이전트**: 외부 판정 전에 시를 피드백하는 추가 역할. 합평(workshop critique) 구조 모방. 동일 모델 편향 문제 있음. [→ multi_agent_council.md](multi_agent_council.md)
- **세션 간 창작 노트 누적**: 여러 시에 걸쳐 노트가 유지되면 모델의 창작 맥락이 누적됨. 자유도와 일관성 간 트레이드오프 미검증.
- **사용자 맞춤 도구 추가**: 사용자가 자신만의 도구(특정 사전, 개인 노트 등)를 주입하는 인터페이스.

## 미결 사항

- [Ph1] 도구 호출 결과를 스크래치패드에 어떤 포맷으로 표현하면 SFT 학습 시 모델이 도구 호출 패턴을 잘 학습하는가 (인라인 텍스트 vs. 구조화된 마커)
- [Ph1] 도구를 전혀 사용하지 않은 트레이스와 도구를 사용한 트레이스의 적절한 비율 — 너무 많으면 도구 없이 생성할 때 품질이 저하될 수 있음
- [TODO] `news_snippet` 같은 외부 API 의존 도구의 재현성 문제: 학습 시 기록된 결과와 추론 시 새로 호출한 결과가 다를 수 있음. 학습 데이터에 결과를 고정 기록하는 방식과 추론 시 실시간 호출하는 방식 중 어느 쪽을 택할 것인가
  - 하위 질문: CI/CD 파이프라인 내에서의 Mock-API 주입 방식 대 인프라 레벨의 캐싱(Caching) 전략 간의 트레이드오프는 어떻게 되는가?
