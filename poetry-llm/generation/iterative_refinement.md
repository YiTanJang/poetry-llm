# 반복 수정 루프

## 철학

> 첫 번째 시는 항상 나쁘다.
> 시는 수정으로 완성된다.

이 파이프라인은 모델이 단 한 번에 완성시를 출력하는 것이 아니라,
**자기 비평 → 수정 → 재평가**의 사이클을 반복하여 시를 완성하도록 한다.

## 루프 구조

```
[입력: 소재/주제/분위기 등]
       │
       ▼
┌─────────────────┐
│  1. 창작 노트    │ ← 씨앗 아이디어 구체화
│     생성         │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  2. 시 초안     │ ← 창작 노트 기반 첫 버전
│     생성         │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  3. 자기 비평   │ ← "이 시의 문제는..."
└────────┬────────┘
         │ 문제 발견?
    ┌────┴────┐
   예          아니오
    │          │
    ▼          ▼
┌───────┐  ┌────────┐
│ 수정  │  │ 출력   │
└───┬───┘  └────────┘
    │ (N번 반복)
    └→ 3번으로
```

## 자기 비평 기준

모델이 시를 평가하는 기준 목록:

### 부정적 체크리스트 (이것이 있으면 수정)
- [ ] 직유("처럼", "같이")가 너무 많다
- [ ] 감정을 직접 서술했다 ("슬프다", "아름답다")
- [ ] 결론을 달았다 (독자의 해석 여지를 없앰)
- [ ] 행갈이가 의미 없는 위치에 있다
- [ ] 특정 이미지가 진부하다 ("달", "별", "눈물")
- [ ] 창작 노트의 의도가 시에 반영되지 않았다

### 긍정적 기준 (이것이 있으면 완성에 가까움)
- [ ] 설명 없이도 감각이 전달된다
- [ ] 행갈이가 의미의 단위와 일치한다
- [ ] 첫 행이 독자를 잡아당긴다
- [ ] 마지막 행이 열린 채 끝난다 (완결되지 않은 여운)
- [ ] 한 번도 쓰인 적 없는 이미지 조합이 있다

## 수정 전략

### 국소 수정 (Local)
- 특정 행 또는 단어만 교체
- "3연 2행 수정: '아름답다'를 삭제하고 이미지로 대체"

### 전역 재구성 (Global)
- 구조 자체를 바꿈 (연 순서 재배열, 행 통합/분리)
- 창작 노트의 핵심 개념을 다른 방식으로 표현

### 완전 재시작 (Reset)
- 현재 방향이 막혔을 때
- 창작 노트에 새로운 접근 방식 추가 후 재생성

## 반복 횟수

| 조건 | 행동 |
|------|------|
| 자기 비평 기준 모두 통과 | 출력 |
| 최대 반복 횟수 도달 (기본: 5회) | 가장 높은 점수의 버전 출력 |
| 3회 이상 동일 문제 반복 | 완전 재시작 |

## 버전 관리

모든 중간 버전을 저장:
```json
{
  "round_0": { "창작노트": "...", "시": "...", "비평": "..." },
  "round_1": { "창작노트_수정": "...", "시": "...", "비평": "..." },
  "round_2": { "시": "...", "비평": "완성" },
  "final": "round_2"
}
```

## 미결 사항

- 자기 비평을 동일 모델이 하는가, 별도 비평 모델을 쓰는가
- 비평 기준의 가중치 설계 (어떤 문제가 더 심각한가)
- 외부 인간 피드백을 루프에 통합하는 시점
- 최대 반복 횟수의 최적값 탐색
