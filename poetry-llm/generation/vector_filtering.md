# 벡터 DB 및 필터링 아키텍처

본 문서는 실시간으로 코사인 유사도(Cosine Embedding Distance) 기반의 거리를 계산하고, 대규모 레퍼런스 코퍼스를 대상으로 벡터 DB(FAISS 또는 Milvus)를 활용해 기존에 있는 표현을 필터링(Novelty 보장)하는 아키텍처를 정의합니다.

## 1. 개요 및 목적
- 생성된 시의 문구나 표현이 기존 작품(학습 데이터나 외부 코퍼스)의 단순 표절이거나 진부한 표현인지 판별합니다.
- Novelty를 보장하기 위해 생성된 시의 초안을 즉각적으로 벡터 임베딩화하고, 기존 코퍼스의 임베딩과 코사인 거리를 계산하여 지나치게 가까운(유사한) 결과를 걸러냅니다.

## 2. 벡터 DB 선정 (FAISS vs Milvus)
- **FAISS**: 인메모리 기반으로 밀집 벡터에 대한 극도로 빠른 검색(KNN 등)을 지원합니다. 파일럿 단계나 단일 서버 환경에서 구현이 단순하고 속도가 빠릅니다.
- **Milvus**: 분산 아키텍처 지원으로 클라우드 네이티브 환경에서 대규모(Billion 단위 이상) 확장에 유리합니다.
- **아키텍처 결정**: 초기 파일럿 단계에서는 실시간 검색 속도와 구현 편의성을 위해 **FAISS**를 사용하여 파이프라인을 검증하고, 스케일업이 필요한 시점에 **Milvus**로 전환할 수 있도록 모듈화된 인터페이스를 설계합니다.

## 3. 실시간 임베딩 유사도 계산 파이프라인
1. **청킹(Chunking)**: 생성된 시를 행(Line) 혹은 연(Stanza) 단위의 의미 단위로 청킹합니다.
2. **임베딩 추출(Embedding)**: 한국어 문장 임베딩 모델을 사용하여 텍스트 청크를 실시간으로 다차원 벡터로 변환합니다.
3. **유사도 검색(Retrieval)**: 추출된 임베딩을 FAISS 인덱스(미리 구축된 대규모 시 코퍼스 임베딩)에 쿼리하여 코사인 유사도 기준 가장 가까운 Top-K 이웃을 찾습니다.
4. **필터링(Filtering)**:
   - 코사인 거리(1 - Cosine Similarity)가 특정 임계치(Threshold) 미만으로 매우 가깝게 나타나면 독창성이 떨어지는 것으로 간주합니다.
   - 해당 구절이 기준치를 초과할 경우, 결과물은 필터링되어 재수정 루프(Iterative Refinement)로 넘겨집니다.
