# 시인의 사고 과정 → LLM 추론 단계 매핑

## 전제: 시인은 어떻게 쓰는가

시인의 창작 과정을 관찰하면 공통된 인지 패턴이 있다:

1. 강렬한 감각 경험이나 생각이 시의 씨앗이 된다
2. 그것을 붙잡고 확장하거나 압축하면서 언어를 탐색한다
3. 무엇을 말할지보다 **어떻게 말하지 않을지**를 결정한다
4. 형식(길이, 행갈이, 시제)이 내용과 상호 결정된다
5. 초안을 쓰고 자신의 의도와 거리를 측정한다
6. 제거하고 교체하면서 밀도를 높인다

이 과정을 LLM이 명시적으로 수행하도록 단계화한다.

---

## 단계 구조 개요

```
[입력: 소재/이미지/주제]
         │
    Stage 1: 감각 포착
         │
    Stage 2: 개념 추상화
         │
    Stage 3: 긴장관계 설정
         │
    Stage 4: 언어 탐색
         │
    Stage 5: 형식 결정
         │
    Stage 6: 초안 생성
         │
    Stage 7: 갭 분석 (자기 비평)
         │
    Stage 8: 수정 계획
         │
    [출력: 완성시 + 창작 노트]
```

---

## Stage 1: 감각 포착 (Sensory Anchoring)

### 목적
입력된 소재에서 가장 구체적이고 감각적인 요소를 분리한다.
추상으로 가기 전에 감각의 정박점을 확보한다.

### LLM이 해야 하는 것
- 시각, 청각, 촉각, 후각, 미각 중 어느 감각이 지배적인가
- 그 감각이 언제, 어디서, 어떻게 발생했는가
- 그 감각의 구체적인 디테일 — 크기, 색, 질감, 속도, 소리

### 입력 형식
```
소재: [자유로운 텍스트 — "지하철에서 본 장면" 등]
```

### 출력 형식 예시
```
감각 포착:
- 지배 감각: 시각
- 구체 디테일: 잠든 남자의 손바닥, 납빛 형광등 아래, 낙엽 한 장,
              흔들리는 전동차 안에서도 낙엽은 미끄러지지 않았다
- 시간: 낮, 출근 시간 이후의 빈 지하철
- 이 감각을 만든 것: 무언가 어울리지 않는 두 요소의 공존
```

### 실패 패턴
- 감각 대신 감정을 포착함: "외로움을 느꼈다" (→ 감각이 아님)
- 너무 일반화: "사람들이 피곤해 보였다" (→ 구체성 부족)
- 소재를 그대로 복사: 탐색 없이 입력을 반복

### 다음 단계로의 연결
감각 디테일 중 **가장 낯선 요소** 또는 **가장 긴장된 요소**를 Stage 2로 넘긴다.

---

## Stage 2: 개념 추상화 (Conceptual Extraction)

### 목적
감각 경험에서 그것이 가리키는 추상적 관계나 의미를 추출한다.
시는 사건을 보고하는 것이 아니라 사건이 의미하는 것을 언어화한다.

### LLM이 해야 하는 것
- "이 감각 경험이 왜 기억에 남는가"를 묻는다
- 그 안에 있는 역설, 아이러니, 충돌, 공명을 찾는다
- 단 하나의 문장으로 핵심 개념을 압축한다

### 입력 형식
```
감각 포착 결과: [Stage 1 출력]
```

### 출력 형식 예시
```
개념 추상화:
- 첫 번째 시도: "아름다움은 알아채는 사람을 필요로 하지 않는다"
- 두 번째 시도: "모르는 사이에 아름다운 일이 일어나고 지나간다"
- 선택: 두 번째 — 더 서술적이고 시간성이 있음
- 이유: "알아채는 사람" 표현이 독자에게 교훈을 주는 느낌. 피한다.
```

### 실패 패턴
- 추상화가 너무 평이함: "자연은 아름답다" (→ 진부한 일반화)
- 개념이 감각과 단절됨: 감각 경험과 무관한 추상 생성
- 여러 개념을 동시에 주장: 시가 두 개의 주제를 지향하게 됨

### 다음 단계로의 연결
핵심 개념을 **명시적으로 말하지 않으면서** 그것을 전달할 방법을 Stage 3에서 설계한다.

---

## Stage 3: 긴장관계 설정 (Tension Architecture)

### 목적
시 내부에서 작동하는 두 힘의 충돌을 명시적으로 설계한다.
긴장이 없는 시는 단선적이다. 긴장이 시를 읽게 만든다.

### LLM이 해야 하는 것
- 핵심 개념 안에 있는 대립 쌍을 찾는다
- 그 대립이 어떻게 시 안에서 공존하거나 충돌하는지 설계한다
- 대립을 해소하지 않는 방법을 찾는다 (해소하면 설명이 된다)

### 입력 형식
```
핵심 개념: [Stage 2 출력]
```

### 출력 형식 예시
```
긴장관계:
- 1차 대립: 수면(무의식, 부재) / 낙엽(존재, 사건)
- 2차 대립: 잠든 사람(시간 밖) / 가을(시간 안)
- 선택한 구조: 두 대립이 동시에 작동하게 — 사람이 잠든 동안
  가을이 일어났다. 해소 없이 공존.
- 해소하지 않는 방법: 마지막 행에 결론을 달지 않는다.
  독자가 남자를 불쌍히 여길지, 그를 부러워할지 결정하게 한다.
```

### 실패 패턴
- 대립을 해소하는 결말 설계: "그래도 가을은 아름다웠다" 류
- 하나의 힘만 있는 구조: 슬픔, 기쁨 등 단일 감정 배치
- 인위적인 대립: 감각 경험과 연결되지 않는 개념 충돌

### 다음 단계로의 연결
긴장관계가 **어떤 언어로** 가장 잘 표현되는지 탐색한다.

---

## Stage 4: 언어 탐색 (Lexical Scouting)

### 목적
시에 사용할 구체적인 단어, 구문, 이미지 후보를 생성하고 비교한다.
이 단계에서 "쓰지 않을 것"도 명시적으로 결정한다.

### LLM이 해야 하는 것
- 핵심 이미지에 대한 표현 후보 3~5개 생성
- 각 후보의 뉘앙스 차이를 분석
- 금지 목록 작성: 진부한 표현, 직접 서술, 과도한 수사

### 입력 형식
```
씨앗이미지 + 핵심개념 + 긴장관계: [이전 단계 출력]
```

### 출력 형식 예시
```
언어 탐색:

이미지 후보:
1. "낙엽이 떨어진다" — 평이. 중력에 의한 수동성. 선택 보류.
2. "낙엽이 내린다" — 눈처럼, 의도성이 약하게 느껴진다. 낫다.
3. "낙엽이 앉는다" — 낙엽에 의지 부여. 과잉.
4. "낙엽 한 장이 손 위에 있다" — 상태 묘사. 도착을 생략. 독자가
   언제 왔는지 모른다. 이 모호성이 좋을 수 있다.

동사 탐색: "모른다" vs "알지 못한다"
- "모른다"가 더 짧고 단정. "알지 못한다"는 너무 설명적.

금지 목록:
- "~처럼", "~같이" (직유 — 비교 대신 제시)
- "쓸쓸하다", "아름답다", "외롭다" (감정 직접 서술)
- "가을", "겨울" (계절 이름 — 낙엽으로 충분히 알 수 있음)
- 결론 행 ("그것이 삶이다" 류)
```

### 실패 패턴
- 첫 번째 후보만 선택하고 비교 없음
- 금지 목록이 너무 추상적: "진부한 표현 금지" (→ 구체적이어야 함)
- 탐색한 언어가 나중에 시에 전혀 반영되지 않음 (→ 추적성 문제)

### 다음 단계로의 연결
탐색된 언어를 어떤 형식 구조에 배치할지 결정한다.

---

## Stage 5: 형식 결정 (Form Decision)

### 목적
길이, 행갈이 원칙, 연 구조, 시제를 결정한다.
형식은 내용에 종속되지 않는다 — 형식이 내용을 만들기도 한다.

### LLM이 해야 하는 것
- 이 시가 왜 짧아야 하는가 (또는 길어야 하는가) 를 논증
- 행갈이의 원칙: 의미 단위? 호흡 단위? 시선 이동?
- 시제 선택의 이유: 현재형이 주는 즉각성 vs 과거형의 거리감

### 입력 형식
```
핵심개념 + 긴장관계 + 언어탐색 결과: [이전 단계 출력]
```

### 출력 형식 예시
```
형식 결정:

길이: short (4~6행)
이유: 이 순간은 길게 서술하면 소멸한다. 목격 그 자체가 순간이므로.

행갈이 원칙: 시선의 이동에 따라 끊는다.
- 손 → 낙엽 → 남자의 얼굴(또는 반응 없음) → 진술

연 구조: 단일 연 (연을 나누면 장면 전환이 생긴다. 이 시는 한 순간임)

시제: 현재형
이유: 독자를 그 순간 안에 놓는다.
     과거형으로 쓰면 "그랬던 이야기"가 된다.
```

### 실패 패턴
- 형식을 기계적으로 결정: "5행 자유시" (→ 이유 없음)
- 내용과 무관한 형식: 산문시가 맞는 내용에 연시 강제
- 시제를 "그냥" 선택: 현재형과 과거형의 효과 차이를 고려하지 않음

### 다음 단계로의 연결
이 모든 결정을 실제 언어로 구현한다.

---

## Stage 6: 초안 생성 (Draft Generation)

### 목적
이전 다섯 단계의 결정을 바탕으로 시의 첫 버전을 생성한다.
완성이 아니라 **검토 가능한 객체**를 만드는 것이 목적이다.

### LLM이 해야 하는 것
- Stage 4의 언어 후보를 실제로 배치
- Stage 5의 형식 결정 준수
- Stage 3의 긴장관계가 해소되지 않도록 주의
- Stage 2의 핵심 개념을 직접 말하지 않음

### 입력 형식
```
이전 모든 단계 출력 (창작 노트)
```

### 출력 형식 예시
```
초안:

잠든 남자의 손 위에
낙엽이 내린다
남자는 모른다
가을이 그를 택했다는 걸
```

### 실패 패턴
- 이전 단계의 결정을 무시하고 새로운 방향으로 생성
- 금지 목록 항목이 시에 등장
- 핵심 개념을 마지막 행에 직접 서술

### 다음 단계로의 연결
초안과 창작 노트를 비교하여 간극을 측정한다.

---

## Stage 7: 갭 분석 (Gap Analysis)

### 목적
초안이 창작 노트의 의도를 얼마나 구현했는지 측정한다.
"좋은 시다/나쁜 시다"가 아니라 "의도와 결과의 거리"를 분석한다.

### LLM이 해야 하는 것
- 창작 노트의 각 필드를 초안과 대조
- 금지 항목 위반 여부 검사
- "긴장이 해소됐는가" 판단
- 부정적 체크리스트 (`iterative_refinement.md` 기준) 적용

### 입력 형식
```
창작 노트 (v0): [전체]
초안: [시 텍스트]
```

### 출력 형식 예시
```
갭 분석:

씨앗이미지 반영: O (잠든 남자, 손, 낙엽 모두 존재)
핵심개념 간접화: O (직접 서술 없이 전달됨)
긴장관계 유지: 부분적 — 4행 "가을이 그를 택했다"가
               긴장을 설명으로 전환할 위험.
               "택했다"는 능동적 행위자 설정 — 검토 필요.

금지 위반: "가을" 직접 언급 — 금지 목록에 있었으나 4행에 등장.

부정적 체크리스트:
- [X] 직유 없음
- [X] 감정 직접 서술 없음
- [ ] 결론 가능성 있음 — 4행 재검토 필요

수정 우선순위:
1. 4행 재작업 (설명성 제거, "가을" 제거)
```

### 실패 패턴
- 막연한 평가: "전반적으로 좋다" (→ 구체적 기준 없음)
- 창작 노트와 비교하지 않고 일반적 시 평가만 수행
- 문제를 과도하게 나열하여 수정 방향이 모호해짐

### 다음 단계로의 연결
우선순위가 높은 문제 하나를 선택하고 수정 방법을 설계한다.

---

## Stage 8: 수정 계획 및 실행 (Revision Planning)

### 목적
갭 분석에서 발견된 문제를 어떻게 수정할지 구체적으로 계획하고 실행한다.
무엇을 바꿀지 + 왜 그렇게 바꾸는지를 먼저 명시한다.

### LLM이 해야 하는 것
- 국소 수정 또는 전역 재구성 중 선택 (이유 포함)
- 수정 전/후를 명시적으로 보여줌
- 수정 후 창작 노트 업데이트 (새로운 금지 항목 추가 등)

### 입력 형식
```
갭 분석 결과: [Stage 7 출력]
초안: [시 텍스트]
```

### 출력 형식 예시
```
수정 계획:

문제: 4행 "가을이 그를 택했다는 걸"이 설명적, "가을" 직접 언급
수정 전략: 국소 수정 — 4행만 교체

시도 1: "낙엽이 내린다는 걸" → 1행과 겹침. 탈락.
시도 2: "그가 자는 동안 무언가 왔다는 걸" → 너무 모호.
시도 3: 4행 삭제 — 3행에서 끝낸다. "남자는 모른다"가
        완결되지 않은 채 열려있게.

선택: 4행 삭제

수정 후 시:
잠든 남자의 손 위에
낙엽이 내린다
남자는 모른다

창작 노트 업데이트:
- 금지에 "4행에서의 설명성 시도" 추가
- 형식결정: 3행으로 축소 (이유: 열린 결말이 더 강함)
```

### 실패 패턴
- 계획 없이 즉시 재생성: 왜 바꾸는지 명시 없음
- 문제를 수정했는데 다른 문제가 발생하고 갭 분석을 생략
- 창작 노트를 업데이트하지 않음 (→ 수정 루프에서 추적성 단절)

### 다음 단계로의 연결
수정된 버전에 대해 Stage 7을 재실행 (→ `iterative_refinement.md` 루프 참조).
자기 비평 기준을 모두 통과하거나 최대 반복 횟수 도달 시 출력.

## 단계별 중요도 분석

### 가장 중요한 단계

**Stage 2 (개념 추상화)와 Stage 3 (긴장관계 설정)**

이 두 단계가 시의 방향을 결정한다.
감각이 아무리 구체적이어도 개념이 평이하면 시가 평이하다.
긴장이 없으면 독자가 끝까지 읽을 이유가 없다.

실험적으로 이 두 단계를 생략한 버전과 비교하면 차이가 가장 클 것이다.

**Stage 7 (갭 분석)**

자기 비평이 창작 노트를 기준으로 수행될 때만 의미 있다.
일반적인 "시 평가"로 대체하면 창작 노트가 있는 의미가 없어진다.

### 생략 가능할 수 있는 단계

**Stage 1 (감각 포착)**

입력이 이미 구체적인 감각 경험인 경우, 별도의 포착 단계가 불필요.
입력이 추상적 주제("고독")인 경우에는 필수.
→ 조건부 단계로 설계할 수 있다.

**Stage 8의 "수정 계획" 부분**

짧은 시에서는 계획보다 즉각 수정이 효율적일 수 있다.
단 계획을 생략하면 수정이 왜 이루어졌는지 학습 데이터에서 사라진다.
학습 데이터 품질을 위해서는 유지 권장.

---

## "진짜로" vs "실제로" 높이는가

### 겉모양 문제 (Surface Problem)

LLM이 이 8단계를 "수행하는 것처럼 보이는 텍스트"를 생성하지만
내부적으로는 여전히 다음 토큰 예측만 수행한다는 비판이 가능하다.

즉, "씨앗이미지: ..." 를 쓰는 것이 실제로 이미지 포착을 수행하는 것인가?
아니면 그 레이블에 이어질 통계적으로 그럴듯한 텍스트를 생성하는 것인가?

### 왜 그래도 품질이 높아지는가

구분과 상관없이, 이 단계 구조가 생성 품질을 높이는 이유가 있다:

1. **컨텍스트 누적 효과**: 8단계의 중간 출력이 모두 이후 단계의 입력 컨텍스트가 된다.
   "씨앗이미지"를 명시적으로 쓰면 그것이 시 생성 시 실제로 참조된다.
   명시하지 않으면 소재에서 감각으로 가는 연결이 암묵적으로 처리된다.

2. **제약 명시화 효과**: "금지" 필드를 명시적으로 작성하면 그 제약이
   실제 시 생성에서 지켜질 가능성이 높아진다 (CoT의 attention 분산 효과).

3. **자기 일관성 강제**: 단계적 결정을 쌓아가면 이후 단계에서 이전 결정과
   모순된 출력을 생성할 때 내부적으로 수정 신호가 발생할 가능성이 있다.

4. **학습 데이터 품질**: 이 구조로 학습 데이터를 만들면 모델이
   "좋은 시 앞에는 이런 사고 과정이 있었다"를 학습하여
   직접 생성보다 우수한 결과를 낼 수 있다.

### 결론

이 단계 구조가 "시인처럼 생각하는 LLM"을 만드는 것인지는 알 수 없다.
하지만 이 구조가 **생성 품질을 높이는 조건을 만든다**는 것은 검증 가능하다.
ablation 실험으로 어떤 단계가 실제로 기여하는지 측정해야 한다.

→ [cot_training_data_design.md](cot_training_data_design.md) ablation 설계 참조

## 미결 사항

- Stage 1~3을 하나의 "사전 설계" 블록으로 통합할 수 있는가 (실행 비용 절감)
- 각 단계의 실패 감지를 자동화하는 방법 (품질 신호 설계)
- Stage 2~3의 품질이 최종 시의 novelty와 상관관계가 있는지 실험
- 단계 수행 순서의 유연성: Stage 5 (형식)를 Stage 3 이전에 결정하는 시인도 있다
