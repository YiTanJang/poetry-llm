# 3-시인 앙상블 + 외부 판정 — DPO 학습 루프

## 이전 설계에서 무엇이 바뀌었는가

이전 설계: 시인·비평가·편집자·독자·악마의 변호인 5역할이 라운드 방식으로 협력하여 하나의 시를 개선했다.

**폐기한다.** 이유:

1. **자기비평의 구조적 한계는 역할 분리로 해결되지 않는다.**
   같은 기반 모델에서 파생된 5역할이 서로 비평하는 것은, 단일 모델이 자기비평하는 것과 본질적으로 다르지 않다. 같은 미적 편향과 맹점을 공유한다.

2. **Council은 훈련 신호가 아닌 추론 시 프로세스다.**
   라운드 방식 협력은 고품질 시 생성에 도움이 될 수 있지만, 그 자체가 학습 루프가 되기 어렵다. 우리가 필요한 것은 **선호 데이터를 만드는 구조**다.

3. **비용과 신호 대비 효용이 낮다.**
   5에이전트 × 3라운드는 API 비용이 크고, 그 결과로 나오는 시 1편은 DPO 학습에 충분한 신호를 주지 못한다.

**새 설계:** 다양한 미학 지향을 가진 3가지 시인 페르소나가 **독립적으로** 초안+스크래치패드를 생성한다. 외부 판정 모델이 비교 평가를 수행하고, 그 결과로 DPO 선호 쌍을 구성한다.

이것은 추론 시 파이프라인이 아니라 **학습 데이터 생성 루프**다.

---

## 3 시인 페르소나

페르소나는 **추론 트레이스의 다양성을 확보**하기 위해 존재한다.
같은 소재에 대해 다른 미학 지향에서 출발한 트레이스를 수집함으로써,
모델이 특정 미학에 과적합되는 것을 방지한다.

### 페르소나 A: 구체 감각 / 이미지 중심

**미학 지향:**
언어는 감각 경험을 재현하는 도구다.
이미지가 선명하면 의미는 설명하지 않아도 전달된다.
추상 개념을 직접 말하는 것을 가장 강하게 거부한다.

**트레이스 패턴:**
구체적 장면, 물질, 감각에서 출발한다.
언어 선택이 시각·촉각·청각 경험의 정확성을 기준으로 이루어진다.
추상으로 가는 경로를 의도적으로 지연시킨다.

**시스템 프롬프트 핵심:**
```
당신은 감각 이미지를 제1원칙으로 삼는 시인이다.
시는 장면이다. 장면이 독자에게 직접 전달될 때 의미는 따로 필요 없다.
구체적인 것에서 시작하라. 추상으로 올라가지 마라.
설명하지 마라. 보여주어라.
```

---

### 페르소나 B: 추상 / 개념 / 낯선 논리

**미학 지향:**
시는 일상 언어가 도달하지 못하는 사유의 영역을 탐색한다.
익숙한 논리를 비틀거나 무너뜨려 새로운 인식을 만든다.
이미지는 수단이고, 개념이 목적이다.

**트레이스 패턴:**
추상 개념 또는 관계에서 출발한다.
"이 개념을 어떻게 언어로 낯설게 만들 수 있는가"를 탐색한다.
일상 언어에서 기대되는 논리를 거부한다.

**시스템 프롬프트 핵심:**
```
당신은 개념과 논리를 시의 재료로 삼는 시인이다.
일상 언어의 기대를 비틀어라. 독자가 예상하지 못한 방향으로 언어를 이끌어라.
이미지는 개념을 싣는 그릇이다. 이미지 자체가 목적이 아니다.
```

---

### 페르소나 C: 형식 실험 / 언어 자체가 주제

**미학 지향:**
시의 내용과 형식은 분리되지 않는다.
행갈이, 문장 구조, 어순, 반복 자체가 의미를 만든다.
언어가 언어 자신에 대해 말하는 가능성을 탐구한다.

**트레이스 패턴:**
형식 결정을 먼저 내린다. 내용은 형식에서 발생한다.
언어의 물질성(소리, 리듬, 시각적 배치)에 주목한다.
행갈이 하나가 의미를 어떻게 바꾸는가를 명시적으로 실험한다.

**시스템 프롬프트 핵심:**
```
당신은 언어 자체를 탐구하는 시인이다.
형식이 내용이다. 행갈이, 반복, 구문 구조가 의미를 만든다.
언어가 무엇을 말하는지보다, 언어가 어떻게 작동하는지에 집중하라.
```

## 생성 파이프라인

### 한 소재에서 3개의 독립 트레이스

```
[입력: 소재 또는 씨앗]
         │
    ┌────┴──────┬────────────┐
    ▼           ▼            ▼
  페르소나 A  페르소나 B   페르소나 C
  (독립 실행) (독립 실행)  (독립 실행)
    │           │            │
  [스크래치패드A + 시A]  [스크래치패드B + 시B]  [스크래치패드C + 시C]
    └────────────┴────────────┘
                 │
         [외부 판정 모델]
         Claude / GPT-4
                 │
         [승자 선택 + 이유]
                 │
    ┌────────────┴──────────────┐
    ▼                           ▼
  긍정 샘플                  부정 샘플
  (승자 스크래치패드+시)      (패자 스크래치패드+시 쌍)
    │                           │
    └────────────┬──────────────┘
                 │
           [DPO 선호 쌍]
```

페르소나들은 **서로의 출력을 보지 않는다.** 완전히 독립적으로 실행된다.
이것이 진짜 다양성을 만드는 방법이다.
한 페르소나의 출력이 다른 페르소나의 추론에 영향을 주면 앙상블 효과가 줄어든다.

## 외부 판정 모델

### 왜 외부 모델인가

자기비평(self-critique)의 한계를 피하는 가장 직접적인 방법:
생성 모델과 다른 모델이 평가한다.
현재 단계에서는 Claude 또는 GPT-4를 판정 모델로 사용한다.

> 이것은 임시 방편이 아니다. 외부 모델 판정이 생성한 선호 데이터로 학습된 보상 모델이 나중에 이 역할을 대체할 수 있다. 하지만 그 보상 모델 자체도 처음에는 인간 또는 외부 모델의 신호가 필요하다.

### 판정 기준

판정 모델에게 주는 평가 지시:

```
다음 세 편의 시와 각각의 스크래치패드를 읽어라.
모두 같은 소재에서 출발했다.

평가 기준:
1. 신선함: 이 이미지 조합 또는 언어 선택이 기존 한국 현대시에서 보기 어려운 것인가
2. 필연성: 이 시의 언어 선택이 이 시여야 하는 이유가 있는가, 아니면 교환 가능한가
3. 여운: 시를 다 읽은 뒤 무언가 남는가
4. 스크래치패드 정직성: 스크래치패드의 탐색이 최종 시에 실제로 영향을 준 것처럼 보이는가

세 편 중 한 편을 승자로 선택하고 이유를 설명하라.
패자 두 편에 대해서도 왜 승자보다 약한지를 설명하라.
```

판정 모델의 설명은 학습 데이터에 포함될 수 있다 — 왜 이 시가 저 시보다 나은가의 이유가 추가 감독 신호가 된다.

## DPO 학습 신호 구성

### 선호 쌍 형식

```json
{
  "prompt": "[소재 입력]",
  "chosen": {
    "scratchpad": "[승자 페르소나의 스크래치패드]",
    "poem": "[승자 시]",
    "persona": "A | B | C"
  },
  "rejected": [
    {
      "scratchpad": "[패자 페르소나 1의 스크래치패드]",
      "poem": "[패자 시 1]",
      "persona": "A | B | C"
    },
    {
      "scratchpad": "[패자 페르소나 2의 스크래치패드]",
      "poem": "[패자 시 2]",
      "persona": "A | B | C"
    }
  ],
  "judge_reasoning": "[판정 모델의 승자 선택 이유]"
}
```

DPO 학습에서는 chosen과 rejected 각 1쌍이 필요하다.
3-way 비교에서 두 개의 패자가 나오면 각각 chosen-rejected 쌍을 만든다.
한 소재당 최대 2개의 DPO 선호 쌍이 생성된다.

### SFT와의 관계

SFT는 먼저 수행된다:
- 소재 → (스크래치패드 + 시) 형식의 기본 패턴을 학습
- 3가지 페르소나 스타일 모두에 대해 학습 (멀티 페르소나 SFT)

DPO는 SFT 이후:
- SFT로 기본 패턴을 학습한 모델이 선호 학습으로 품질을 높임
- DPO 신호: 외부 판정에서 높은 평가를 받은 트레이스를 선호

## 이 설계가 Council보다 나은 이유

| 차원 | 구 Council 설계 | 새 앙상블 설계 |
|------|----------------|--------------|
| 다양성 출처 | 같은 모델의 역할 분리 | 다른 미학 지향 페르소나 |
| 판정 주체 | 동일 기반 모델 (자기비평) | 외부 모델 (Claude/GPT-4) |
| 학습 신호 | 없음 (추론 시 파이프라인) | DPO 선호 쌍 직접 생성 |
| 비용 효율 | 5에이전트 × 3라운드 = 15+ API 호출 | 3 페르소나 + 1 판정 = 4 API 호출 |
| 편향 위험 | 기반 모델의 편향 반복 | 외부 판정이 일부 교정 |

## 미결 사항

- 3가지 페르소나의 스타일 다양성이 실제로 충분한가 — 판정 모델이 항상 같은 페르소나를 선택한다면 다양성이 없는 것이다. 판정 편향을 측정해야 한다.
- 외부 판정 모델의 미적 편향 — Claude와 GPT-4는 자신들의 학습 데이터에서 비롯된 미적 선호가 있다. 그 선호가 우리가 원하는 novelty와 일치하는가.
- 3-way 대신 더 많은 페르소나를 사용해야 하는가 — 비용과 다양성 간 트레이드오프.
- 페르소나 외에 같은 페르소나로 여러 번 샘플링하는 것도 다양성에 기여하는가 — 온도 기반 다양성 vs 페르소나 기반 다양성.
- 판정 모델의 설명(judge_reasoning)을 학습 데이터에 포함시키는 것이 실제로 도움이 되는가 — 이 설명이 생성 모델의 품질 판단 능력을 높이는 데 기여하는지 실험 필요.
- 장기적으로 외부 판정 모델을 학습된 보상 모델로 대체하는 일정과 조건.
