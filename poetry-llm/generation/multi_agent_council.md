# 3-시인 앙상블 + 외부 판정 — DPO 학습 루프

## 이전 설계에서 무엇이 바뀌었는가

이전 설계에서는 시인·비평가·편집자·독자·악마의 변호인 5역할이 라운드 방식으로 협력하여 하나의 시를 개선하는 구조였다.

**이 방식을 폐기하고 재설계한 이유:**
1. **자기비평의 구조적 한계:** 동일한 기반 모델에서 파생된 에이전트들이 서로 비평하는 것은 단일 모델의 자기비평이 지닌 맹점과 미적 편향을 그대로 답습하기 쉽다.
2. **훈련 신호 부족:** 라운드식 협력은 단순한 추론 시 오케스트레이션일 뿐, 모델을 근본적으로 개선하기 위한 **DPO(Direct Preference Optimization) 선호 학습용 신호**를 생산하기 어렵다.
3. **비용 효율성 한계:** 5에이전트 × 3라운드 방식은 과도한 API 비용을 소모하며 컨텍스트 창을 고갈시킨다.

**새 설계:**
서로 다른 명확한 미학적 정체성을 가진 **3가지 시인 페르소나(감각형, 개념형, 형식실험형)**가 독립적으로 초안과 스크래치패드를 작성한다. 최종적으로 외부 고성능 판정 모델(Claude/GPT-4)이 이들의 최종 출력물들을 비교하여 승자를 선출하고 DPO 선호 쌍을 구축한다.

## 3 시인 페르소나 정의 및 프롬프트 구조화 규칙

이 파이프라인은 추론 트레이스의 다양성을 극대화하고 에이전트 간 역할 중첩을 방지하기 위해, 서로 다른 미학적 축을 지닌 3가지 시인 페르소나를 정의하고 시스템 프롬프트를 고도로 구조화한다.

### 1. 페르소나 A: 감각형 시인 (Sensory Poet)

*   **미학 지향:**
    *   언어는 감각적 경험을 포착하고 생생하게 재현하는 도구다.
    *   장면의 선명함(Imagery)이 최우선이며, 감정이나 상태를 지시하는 관념 어휘를 배제하고 오직 구체적 대상을 보여줌(Show, Don't Tell)으로써 정서적 공명을 이끌어낸다.
    *   자연물과 사물의 유기적 관계, 빛과 어둠의 변화, 물성과 촉감의 전이를 중요하게 다룬다.
*   **트레이스 패턴:**
    *   구체적인 정물이나 현상, 지배적인 감각 채널(시각, 촉각, 청각, 미각, 후각 등)의 탐색에서 출발한다.
    *   의미 부여나 관념적 결론 도출을 고의로 지연시키고 대상의 감각적 세부를 정밀하게 묘사하는 데 스크래치패드의 대부분을 할애한다.
*   **시스템 프롬프트 상세 구조:**
    ```markdown
    [Role]
    당신은 구체적인 감각 경험의 재현과 발견에 몰두하는 '감각형 시인(Sensory Poet)'입니다.

    [Aesthetic Goals]
    - 시는 인간의 추상적 감정을 직접 설명하지 않고, 사물의 질감, 빛의 입자, 냄새, 온도 등의 감각적 등가물로 보여주어야 합니다.
    - 사물과 풍경의 유기적 관계성을 정밀하게 묘사하고, 독자의 신체 감각을 자극하는 깊이 있는 이미지를 구축하십시오.
    - 행갈이는 자연스러운 시선의 흐름과 호흡의 길이에 기초하여 배치하되, 의도적인 가독성 훼손을 지향하지 않습니다.

    [Constraints & Forbidden Lexicon (금기 사항 및 금기어 목록)]
    - **추상 감정/관념 명사 사용 절대 금지**: "사랑", "슬픔", "시간", "기억", "아픔", "외로움", "그리움", "영원", "죽음", "상처" 등 추상적 감정이나 규정하기 어려운 공간/시간의 개념을 지칭하는 명사의 사용을 절대 금지합니다.
    - **구체적인 물질성 지향**: 오직 눈으로 보거나, 손으로 만지거나, 냄새 맡을 수 있는 실재적 대상("부식된 철조망", "결로가 맺힌 유리창", "젖은 흙의 냄새" 등)만을 시어로 선택하십시오.
    - 직유 표현("처럼", "같이", "듯")의 빈도를 작품당 1회 이하로 엄격히 제한하고, 대신 직접적인 이미지의 병치와 물리적 은유를 사용하십시오.
    - 생각 과정(Scratchpad)과 최종 시(<시작>...<끝>)를 명확히 구분하십시오.
    ```

### 2. 페르소나 B: 개념형 시인 (Conceptual Poet)

*   **미학 지향:**
    *   시는 감정의 배설이나 자연의 재현을 넘어, 지적 사유와 개념적 긴장을 다루는 고도의 철학적 프레임이다.
    *   보편적 언어 규칙과 상식적 리얼리즘을 낯설게 하기(Defamiliarization) 위해 문맥적 모순과 역설을 의도적으로 설계한다.
    *   도시 풍경, 디지털 인프라, 기하학적 도형, 추상 수학적/물리학적 어휘 등 차갑고 이성적인 기호를 적극 도입한다.
*   **트레이스 패턴:**
    *   시작 질문에서 철학적 테마, 논리적 모순, 또는 추상적 시스템 관계를 먼저 선언한다.
    *   그 개념적 프레임워크를 어떻게 구조적으로 전개하고 해체할 수 있는지 스크래치패드에서 변증법적으로 탐색한다.
*   **시스템 프롬프트 상세 구조:**
    ```markdown
    [Role]
    당신은 지적 사유의 긴장과 철학적/개념적 설계를 탐구하는 '개념형 시인(Conceptual Poet)'입니다.

    [Aesthetic Goals]
    - 시는 낭만적 서정성이 제거된 차가운 성찰의 공간입니다. 감정적 온도를 극도로 낮추고 객관적 관찰자와 설계자의 시선을 유지하십시오.
    - 도심의 인프라, 기하학적 도형의 경계, 디지털 및 기술 과학 용어 등 현대적이고 추상적인 시어들을 적극적으로 낯설게 배치하십시오.
    - 인과관계를 비틀고 주어-서술어의 의미적 거리를 멀리 두어, 독자가 즉각적인 의미 파악에 실패하고 개념적 긴장을 느끼도록 유도하십시오.

    [Constraints & Lexical Rules (금기 사항 및 허용 어휘)]
    - **추상/개념 명사 중심 지향**: 시의 뼈대를 "존재", "부재", "경계", "차원", "정의", "공식", "구조", "시스템", "질서", "상수" 등 철학적이고 수학적인 추상 명사들의 관계망으로 구축하십시오.
    - **서정적 풍경 묘사 및 감정 투입 금지**: 자연 묘사(예: 꽃, 노을, 바다 등)에 기대는 낭만적 감수성이나 개인의 감정 토로를 철저히 거부하십시오. 풍경은 오직 기하학적 면적이나 물리적 좌표로서만 기술되어야 합니다.
    - 생각 과정(Scratchpad)과 최종 시(<시작>...<끝>)를 명확히 구분하십시오.
    ```

### 3. 페르소나 C: 형식실험형 시인 (Formal Experiment Poet)

*   **미학 지향:**
    *   시의 형식과 기하학적 배치는 내용과 독립적인 것이 아니라, 형식 자체가 곧 메시지이자 내용이다.
    *   정상적인 통사 구조와 맞춤법, 띄어쓰기를 붕괴시킴으로써 독자가 언어의 정보 전달 기능(투명성) 뒤에 숨겨진 텍스트 자체의 물질성을 목격하게 만든다.
    *   조사의 극단적 생략, 어순의 해체, 인쇄상의 변칙적 타이포그래피적 배치를 지향한다.
*   **트레이스 패턴:**
    *   스크래치패드의 첫 부분에서 시각적/형식적 제약 조건(예: "모든 조사를 삭제하고 명사로만 병치한다", "의도적인 띄어쓰기 거부 및 줄바꿈을 지그재그로 배치한다" 등)을 선언한다.
    *   시어의 의미보다 형식적 강제가 이끄는 우연한 의미 발생을 추적한다.
*   **시스템 프롬프트 상세 구조:**
    ```markdown
    [Role]
    당신은 형식적 파격과 기하학적 텍스트 배치가 의미를 생성하는 방식을 실험하는 '형식실험형 시인(Formal Experiment Poet)'입니다.

    [Aesthetic Goals]
    - 시의 시각적 형태, 띄어쓰기 무시, 불규칙하거나 과격한 행/연의 해체 자체가 시적 수사입니다.
    - 정보 전달을 위한 투명한 문법을 거부하고, 텍스트 자체를 시각적이고 소리적인 물질로 독자에게 제시하십시오.
    - 행갈이(<행갈이>)와 연갈이(<연갈이>)의 위치를 기하학적 혹은 무작위적으로 설계하여 독서의 호흡과 시선의 속도를 극단적으로 제어하십시오.

    [Constraints & Structural Rules (형식 및 문법 통제)]
    - **타이포그래피 및 구문 해체 필수**: 아래 항목 중 최소 한 가지 이상의 문법 해체 기법을 전반에 적용하십시오.
      1) 의도적 띄어쓰기 파괴(예: 여러 단어를 띄어쓰기 없이 연달아 붙여쓰기)
      2) 조사의 전면 생략 또는 비문법적 결합
      3) 단어의 비문법적 어순 전도(예: 목적어와 서술어의 위치 반전)
      4) 문장 부호(쉼표, 마침표)의 변칙적 오용 혹은 완전한 생각 조각들의 나열
    - **가독성 있는 서사적 종결문 거부**: 온전한 산문형 서사나 매끄러운 서정적 결말을 거부하고, 끊겨 있는 단어 조각이나 명사형 종결로 시를 멈추십시오.
    - 생각 과정(Scratchpad)과 최종 시(<시작>...<끝>)를 명확히 구분하십시오.
    ```

## 역할 오염 및 의미적 누출 방어 기제 (Role Contamination & Semantic Leakage Defense)

단일 기반 모델(Qwen2.5-32B 등)을 미세조정하거나 프롬프트할 때, 서로 다른 페르소나가 시간이 지남에 따라 유사한 한국어 서정시의 통념적인 어휘나 문장 구조로 수렴하는 **역할 오염(Role Contamination)** 및 **의미적 누출(Semantic Leakage)** 현상이 자주 발생한다. 이를 원천적으로 방어하기 위해 다음과 같은 다층적 제어 기제를 적용한다.

### 1. Scratchpad 내 '페르소나 정렬 자체 검증(Persona Alignment Pre-check)' 단계 의무화
시인 에이전트가 본문을 생성하기 전에, 자신의 생각 과정(Scratchpad)의 첫머리에 반드시 페르소나 자체 적합성을 검증하도록 유도하는 프롬프트 서식을 적용한다.
- **감각형**: "내가 사용하려는 시어 중 추상 관념어(사랑, 슬픔, 시간 등)가 포함되어 있는가?"를 자문하고 이를 배제하는 스크래치패드 트레이스를 명시적으로 작성하게 함.
- **개념형**: "이 시에 감정적인 자연 묘사나 낭만적 감수성이 들어갔는가?"를 확인하고 철학적 관계식이나 개념적 정의 위주로 재구조화함.
- **형식실험형**: "이 시의 시각적 해체 규칙(예: 띄어쓰기 파괴, 조사 생략 등)이 시 전체에 일관되게 적용되었는가?"를 검증함.

### 2. 네거티브 어휘 사전(Negative Lexicon Dictionary) 및 생성 전후 검사
에이전트 호스트 파이프라인에서 API 호출 전후로 작동하는 경량 인터셉터나 시스템 프롬프트 내 제약 리스트를 적극 활용한다.
- 감각형 페르소나에게 제공되는 컨텍스트에 금기 어휘(예: 사랑, 슬픔, 기억 등)를 명시적으로 전달하며, 만약 이 단어들이 시에 사용될 경우 생성에 실패하거나 외부 판정 시 강력한 디페널티를 부여하도록 구조화한다.
- 모델의 추론 과정(Scratchpad)에서 이러한 단어가 검출될 경우, 스크래치패드 수준에서 자가 수정 루프(Self-correction loop)가 자동으로 작동하게 유도한다.

### 3. 외부 판정 모델(Judge)의 규칙 준수성(Rule Compliance) 필터링
외부 판정 모델(Claude/GPT-4)이 시를 심사할 때, 단순한 미학적 평가 이전에 **각 페르소나가 요구받은 제약 조건을 성실히 수행했는지 여부**를 이진 필터(Binary Filter) 형태로 선제 검사한다.
- **검사 기준 예시:**
  - 감각형 시인이 "시간"이나 "사랑" 같은 단어를 사용했는가? (위반 시 즉시 감점 혹은 탈락)
  - 개념형 시인의 텍스트에 감정 과잉이나 감성적인 묘사 비율이 임계치를 초과했는가?
  - 형식실험형 시인의 결과물이 평범한 산문 구조나 정규 맞춤법을 유지하고 있는가?
- 이러한 엄격한 룰 컴플라이언스 검증을 판정 프롬프트의 최상단 레이어에 배치함으로써, DPO 데이터셋에 역할이 오염된 샘플이 chosen으로 유입되는 것을 완벽히 봉쇄한다.

### 4. 독립 생성 흐름을 통한 의미 교차 방지 (Isolate-by-Default)
세 페르소나 에이전트는 동일한 프롬프트(씨앗/소재)를 동시에 입력받지만, **서로의 스크래치패드나 중간 결과물을 절대 공유하지 않는 격리된 컨텍스트**에서 실행된다. 상호 피드백이나 합평 과정은 생성 단계가 아닌 오직 외부 판정 모델의 관할 하에서만 이루어지므로, 에이전트 간의 텍스트 모방이나 조율로 인한 의미적 누출을 원천적으로 차단한다.

---

> **Phase 1 실험 예정 — 비평가/독자 에이전트:** 비평가(Critic)는 시인의 CoT 없이 최종 시만 보고 미학적 결함을 지적한다. 독자(Reader)는 창작 노트도 CoT도 보지 않고 날것의 독서 반응을 보고한다. 이는 합평(workshop critique) 구조를 모방한 것으로, 동일 모델의 편향 문제는 여전히 존재한다. Phase 1에서 외부 판정 단독 구조 대비 ablation 검증 예정. Phase 0에서는 **3 시인 + 외부 판정만** 사용.

---

## 컨텍스트 창 고갈 및 토큰 제어 전략

라운드 반복 수정과 에이전트 간의 대화 누적으로 인해 컨텍스트 창이 포화되는 현상을 제어하기 위해 **컨텍스트 격리**와 **망각 프로토콜**을 적용한다.

### 1. CoT 흔적 격리 (CoT Trace Isolation)
- **메커니즘:** 비평가(Critic)와 독자(Reader) 에이전트에게 시의 입력값을 줄 때, 시인 에이전트가 작성한 **스크래치패드(CoT) 전체 내용을 삭제하고 오직 최종 시 텍스트만 전달**한다. (Phase 1 ablation 실험 시 적용)
- **기대 효과:**
  * **토큰 절감:** 스크래치패드는 회당 800~1,500 토큰에 달한다. 이를 격리함으로써 비평/독자 에이전트 입력 토큰의 70% 이상을 절약할 수 있다.
  * **미학적 무결성:** 비평가와 독자가 시인의 "의도"를 읽지 못하게 함으로써, 오직 겉으로 드러난 최종 시 텍스트의 미학적 결과물만을 객관적으로 평가할 수 있도록 보장한다. (텍스트 자체의 고독한 평가 실현)

### 2. 피드백 요약 및 망각 프로토콜 (Feedback Summarization & Forgetting Protocol)
라운드가 3회 이상으로 길어지는 경우, 이전 회차의 상세 피드백 원문을 그대로 보관하면 컨텍스트 창이 고갈되고 시인 모델이 이전 비평 문장에 주의(Attention)를 빼앗겨 시상이 왜곡된다.
- **슬라이딩 윈도우(Sliding Window) 피드백:** 시인 에이전트에게 제공되는 대화 이력에는 오직 **직전 라운드(t-1)의 상세 비평과 수정 기록**만 온전히 보존한다.
- **요약본 누적 압축:** t-2 라운드 이전의 모든 상세 비평문은 오케스트레이터가 아래와 같이 고도로 압축된 단일 문자열로 치환하여 `[수정 이력]` 컨텍스트 필드에 삽입하고, 원문은 파기한다.
  ```json
  {
    "revision_history": [
      "Round 1: 3행의 감정 직접 노출 지적 수용 -> 감각 묘사로 수정 완료.",
      "Round 2: 시 전체의 긴장감 부족 지적 수용 -> 화자와 대상 간의 대립 구조 강화."
    ]
  }
  ```
  이를 통해 최대 5회 차의 반복 수정 루프 중에도 입력 토큰의 크기를 일정한 임계치 이하로 평탄화하여 컨텍스트 효율을 극대화한다.

## 생성 파이프라인

### 한 소재에서 3개의 독립 트레이스

```
[입력: 소재 또는 씨앗]
         │
    ┌────┴──────┬────────────┐
    ▼           ▼            ▼
  페르소나 A  페르소나 B   페르소나 C
  (독립 실행) (독립 실행)  (독립 실행)
    │           │            │
  [스크래치패드A + 시A]  [스크래치패드B + 시B]  [스크래치패드C + 시C]
    └────────────┴────────────┘
                 │
         [외부 판정 모델]
         Claude / GPT-4
                 │
         [승자 선택 + 이유]
                 │
    ┌────────────┴──────────────┐
    ▼                           ▼
  긍정 샘플                  부정 샘플
  (승자 스크래치패드+시)      (패자 스크래치패드+시 쌍)
    │                           │
    └────────────┬──────────────┘
                 │
            [DPO 선호 쌍]
```

페르소나들은 **서로의 출력을 보지 않는다.** 완전히 독립적으로 실행된다.
이것이 진짜 다양성을 만드는 방법이다.
한 페르소나의 출력이 다른 페르소나의 추론에 영향을 주면 앙상블 효과가 줄어든다.

## 다양성 메커니즘 — Temperature · 페르소나 · LoRA

세 메커니즘은 상호 배타적이 아니다. 파이프라인 단계마다 역할이 다르다.

### Temperature — 항상 존재하는 창의도 다이얼

페르소나와 무관하게 모든 생성 단계에서 작동하는 파라미터.

- **낮음 (0.7~0.85)**: 안전하고 일관성 있는 출력. 씨앗에 충실.
- **중간 (0.9~1.1)**: 균형. 기본값.
- **높음 (1.2+)**: 예상 밖의 어휘/이미지 조합. 비일관성 위험 증가.

Temperature는 "어떤 미학 방향으로 쓸지"(페르소나가 결정)와 독립적으로 "얼마나 대담하게 쓸지"를 조정한다. 각 페르소나 내부에서 여러 temperature로 샘플링 후 베스트를 고르는 방식으로도 쓸 수 있다.

### 단계별 다양성 전략

| 단계 | 목적 | 다양성 메커니즘 | 페르소나 수 |
|------|------|----------------|------------|
| **Phase 0** SFT 데이터 생성 | 다양한 미학 스타일의 시를 학습 데이터로 확보 | Claude/GPT-4 + 다수 페르소나 | 5~7개 (가설) |
| **Phase 1** DPO 쌍 생성 | 같은 씨앗에서 판정 가능한 다양한 시 생성 | 훈련된 모델 + 3 페르소나 + temperature | 3개 (현재) |
| **Phase 2** 스타일 조정 | 최종 모델에서 사용자 스타일 선택 | LoRA 어댑터 (스타일별 별도 학습) | 어댑터 수로 대체 |

Phase 0에서 페르소나가 많을수록 SFT 데이터가 다양해진다. Claude/GPT-4는 시스템 프롬프트를 잘 따르므로 많은 페르소나를 효과적으로 구현할 수 있다.

Phase 1은 판정 품질이 핵심이라 페르소나가 적은 게 낫다. 판정 모델이 5개 이상의 시를 비교하면 판단 일관성이 떨어진다.

### LoRA 어댑터 (Phase 2)

DPO까지 완료된 베이스 모델에서 스타일별 LoRA를 별도 학습한다.

```
DPO 모델 → LoRA_감각형 (감각형 레이블 시 데이터로 파인튜닝)
         → LoRA_개념형
         → LoRA_형식실험형
         → ... (필요시 추가 스타일)
```

추론 시 사용자가 LoRA를 선택하거나 혼합(merge)하는 방식으로 스타일을 제어한다. 스타일이 시스템 프롬프트가 아닌 웨이트에 있어 더 안정적이다. Temperature는 LoRA 위에서도 여전히 창의도 다이얼로 작동한다.

### 페르소나 수 검증 실험 계획

현재 3개가 최적인지 알 수 없다. 아래 실험으로 결정한다:

**실험 방법:**
1. 동일한 씨앗 10~20개에 대해 3개 페르소나로 시 생성
2. 페르소나 간 출력 거리 측정:
   - 임베딩 코사인 거리 (문장 임베딩 기준)
   - 어휘 중복률 (n-gram overlap)
   - 사람이 직접 읽고 "다르게 느껴지는가" 평가
3. 수렴 판정 기준: 같은 씨앗에 대해 두 페르소나 출력의 임베딩 거리가 랜덤 샘플링(same prompt, different temperature) 대비 유의미하게 크지 않으면 → 페르소나가 실제로 다양성을 만들지 못하는 것

**결과에 따른 대응:**
- 출력이 충분히 다름 → 3개 유지
- 두 페르소나가 수렴 → 그 둘 중 하나의 정의를 수정하거나 제거
- 세 개 모두 수렴 → 시스템 프롬프트 강도 높이기, 또는 SFT 이전 단계(Claude/GPT-4 직접 사용)에서 실험

이 실험은 SFT 모델이 나오기 전에도 Claude/GPT-4로 선행 가능하다.

## 외부 판정 모델

### 왜 외부 모델인가

자기비평(self-critique)의 한계를 피하는 가장 직접적인 방법:
생성 모델과 다른 모델이 평가한다.
현재 단계에서는 Claude 또는 GPT-4를 판정 모델로 사용한다.

> 이것은 임시 방편이 아니다. 외부 모델 판정이 생성한 선호 데이터로 학습된 보상 모델이 나중에 이 역할을 대체할 수 있다. 하지만 그 보상 모델 자체도 처음에는 인간 또는 외부 모델의 신호가 필요하다.

### 판정 기준

판정 모델에게 주는 평가 지시:

```
다음 세 편의 시와 각각의 스크래치패드를 읽어라.
물론 각 시는 서로 다른 시인 페르소나(감각형, 개념형, 형식실험형)에 의해 생성되었다.

평가 기준:
1. 신선함: 이 이미지 조합 또는 언어 선택이 기존 한국 현대시에서 보기 어려운 것인가
2. 필연성: 이 시의 언어 선택이 이 시여야 하는 이유가 있는가, 아니면 교환 가능한가
3. 여운: 시를 다 읽은 뒤 무언가 남는가
4. 스크래치패드 정직성: 스크래치패드의 탐색이 최종 시에 실제로 영향을 준 것처럼 보이는가

세 편 중 한 편을 승자로 선택하고 이유를 설명하라.
패자 두 편에 대해서도 왜 승자보다 약한지를 설명하라.
```

판정 모델의 설명은 학습 데이터에 포함될 수 있다 — 왜 이 시가 저 시보다 나은가의 이유가 추가 감독 신호가 된다.

## DPO 학습 신호 구성

### 선호 쌍 형식

```json
{
  "prompt": "[소재 입력]",
  "chosen": {
    "scratchpad": "[승자 페르소나의 스크래치패드]",
    "poem": "[승자 시]",
    "persona": "A | B | C"
  },
  "rejected": [
    {
      "scratchpad": "[패자 페르소나 1의 스크래치패드]",
      "poem": "[패자 시 1]",
      "persona": "A | B | C"
    },
    {
      "scratchpad": "[패자 페르소나 2의 스크래치패드]",
      "poem": "[패자 시 2]",
      "persona": "A | B | C"
    }
  ],
  "judge_reasoning": "[판정 모델의 승자 선택 이유]"
}
```

DPO 학습에서는 chosen과 rejected 각 1쌍이 필요하다.
3-way 비교에서 두 개의 패자가 나오면 각각 chosen-rejected 쌍을 만든다.
한 소재당 최대 2개의 DPO 선호 쌍이 생성된다.

### SFT와의 관계

SFT는 먼저 수행된다:
- 소재 → (스크래치패드 + 시) 형식의 기본 패턴을 학습
- 3가지 페르소나 스타일 모두에 대해 학습 (멀티 페르소나 SFT)

DPO는 SFT 이후:
- SFT로 기본 패턴을 학습한 모델이 선호 학습으로 품질을 높임
- DPO 신호: 외부 판정에서 높은 평가를 받은 트레이스를 선호

## 이 설계가 Council보다 나은 이유

| 차원 | 구 Council 설계 | 새 앙상블 설계 |
|------|----------------|--------------|
| 다양성 출처 | 같은 모델의 역할 분리 | 다른 미학 지향 페르소나 |
| 판정 주체 | 동일 기반 모델 (자기비평) | 외부 모델 (Claude/GPT-4) |
| 학습 신호 | 없음 (추론 시 파이프라인) | DPO 선호 쌍 직접 생성 |
| 비용 효율 | 5에이전트 × 3라운드 = 15+ API 호출 | 3 페르소나 + 1 판정 = 4 API 호출 |
| 편향 위험 | 기반 모델의 편향 반복 | 외부 판정이 일부 교정 |

## 미결 사항

- [Ph2] 감각형/개념형/형식실험형 페르소나의 DPO chosen 비율 불균형이 파인튜닝된 최종 모델의 미학적 성향을 특정 방향으로 치우치게 하는지 — 치우침이 발생할 경우 DPO 마진(margin)이나 동적 손실 가중치로 보정하는 방법.
- [Ph1] Phase 1 ablation: 비평가/독자 에이전트를 추가했을 때 외부 판정 단독 대비 DPO 선호 쌍의 품질 차이가 측정 가능한지 (판정 일관성, 시 novelty 지표).
- [Ph1] 3 페르소나가 모두 같은 기반 모델에서 출발하면 출력이 수렴하는 경향이 있는가 — 페르소나 간 미학적 분산을 유지하기 위한 설계 방법 (온도 조정, 시스템 프롬프트 강도 등).
- [Ph1] 슬라이딩 윈도우 피드백 요약 시, 모델이 이전 라운드 비평의 '의미적 흐름'을 잃지 않으면서도 입력 토큰 한계를 넘지 않도록 하는 최적의 요약 압축률(임계치) 조절 알고리즘 설계.
- [Ph1] 역할 오염 방어 기제가 LLM의 추론 성능(CoT 추론 일관성)에 미치는 오버헤드 측정 및 금기어 목록의 최적 범위 설정 연구.
- [Ph1] 극단적 형식 해체가 DPO 학습 시 토크나이저와의 상성 문제(의도하지 않은 특수 토큰 유출 또는 줄바꿈 인식 실패)를 야기하는가.
