# 창작 노트 구조화 스키마

## 왜 스키마가 필요한가

`cot_creative_notes.md`의 창작 노트는 내용은 타당하지만 형식이 일관되지 않다.
이 상태로는 모델이 학습할 때 "창작 노트란 무엇인가"를 형식으로 추론하는 것이 어렵다.
학습 데이터 수천 건을 만들면 포맷 자체가 제각각이고, 모델은 내용보다 형식 패턴에 과적합된다.

그러나 스키마를 너무 엄격하게 설계하면 문제가 다른 방향으로 발생한다:
- 모델이 필드를 채우는 형식에 집중하여 실제 사고가 빠진 "껍데기 창작 노트"를 생성
- 창작의 즉흥성과 연상 흐름이 제거되어 노트 자체가 죽음

목표는 **최소한의 구조 + 최대한의 자유**다.

---

## 구조화 vs 자유형식 트레이드오프

| 차원 | 완전 자유형식 | 완전 구조화 |
|------|--------------|-------------|
| 학습 일관성 | 낮음 — 포맷 노이즈 큼 | 높음 — 모델이 형식 학습 쉬움 |
| 창의성 표현 | 높음 — 흐름 그대로 | 낮음 — 칸 채우기 패턴화 |
| 추적성 | 낮음 — 노트→시 연결 불투명 | 높음 — 각 필드가 시에 매핑 |
| 데이터 제작 비용 | 낮음 — 자유롭게 생성 | 높음 — 검증 필요 |
| 모델 추론 길이 | 불예측 | 예측 가능 |

**결론:** 필수 필드는 최소화(4~5개)하고 내용은 자유형식으로 작성하게 한다.
필드 레이블이 구조를 주고, 내용은 시인의 목소리를 유지한다.

## 스키마 정의

### 필수 필드 (항상 있어야 하는 것)

```
씨앗이미지: [1~3문장. 구체적인 감각 경험 또는 발상의 출발점. 추상 금지.]

핵심개념: [1문장. 씨앗이미지에서 추출한 추상적 의미 또는 관계. 시의 "속"에 해당.]

형식결정:
  길이: short | medium | long
  구조: 자유시 | 연시 | 산문시
  시제: 현재 | 과거 | 혼합

금지: [쓰지 않을 표현, 이미지, 수사법의 목록. 최소 2개, 최대 5개.]
```

**각 필드의 목적과 예상 토큰 수:**

| 필드 | 목적 | 예상 토큰 |
|------|------|-----------|
| 씨앗이미지 | 발상의 구체적 출발점 확보 | 40~80 |
| 핵심개념 | 추상화 수준 고정 — 시가 무엇에 관한 것인지 | 20~40 |
| 형식결정 | 생성 제약 설정 | 15~25 |
| 금지 | 진부함 방지, 범위 제한 | 20~40 |

**총 필수 필드: 약 95~185 토큰**

---

### 선택 필드 (있으면 더 좋은 것)

```
언어후보: [3~5개의 구체적 표현, 이미지, 단어 후보. 실제로 쓸 수도 있고 안 쓸 수도 있음.]

긴장관계: [시 내부에서 충돌하는 두 힘 또는 개념. 예: "움직임 / 정지", "앎 / 모름"]

독자위치: [독자가 시를 읽을 때 서 있어야 할 감각적 위치. 예: "지하철 승강장, 멀리서 보는 시점"]

참조거부: [이 시와 혼동될 수 있는 기존 시 또는 클리셰 이미지. 피해야 할 선례.]
```

**각 필드의 목적과 예상 토큰 수:**

| 필드 | 목적 | 예상 토큰 |
|------|------|-----------|
| 언어후보 | 구체적 단어 탐색을 명시적으로 수행 | 30~60 |
| 긴장관계 | 시의 내부 에너지 구조 설계 | 15~30 |
| 독자위치 | 독자 경험 설계 — 어디서 무엇을 보는가 | 20~40 |
| 참조거부 | novelty 확보, 기존 패턴 명시적 거부 | 20~40 |

**선택 필드 포함 시 총 토큰: 약 180~355 토큰**

## 전체 스키마 예시 (마크다운 템플릿)

### 최소 버전 (필수 필드만)

```markdown
## 창작 노트

씨앗이미지: 지하철 안 잠든 남자의 손바닥 위에 낙엽이 한 장 내려앉았다.
아무도 치우지 않았다. 남자는 계속 잔다.

핵심개념: 모르는 사이에 아름다운 일이 일어나고 지나간다.

형식결정:
  길이: short
  구조: 자유시
  시제: 현재

금지: "낙엽처럼", "쓸쓸하다", 계절 이름 직접 언급, 결론 달기
```

### 확장 버전 (선택 필드 포함)

```markdown
## 창작 노트

씨앗이미지: 지하철 안 잠든 남자의 손바닥 위에 낙엽이 한 장 내려앉았다.
아무도 치우지 않았다. 남자는 계속 잔다.

핵심개념: 모르는 사이에 아름다운 일이 일어나고 지나간다.

언어후보:
- "낙엽이 내린다" (동사 선택: 떨어진다가 아니라 내린다 — 의도성 부여)
- "남자는 모른다"
- "가을이 그를 택했다" (가능하나 직접성 검토 필요)
- "손 위에" vs "손바닥 위에" — 후자가 더 구체적

긴장관계: 수면(무의식) / 가을(의식적 계절 변화)

독자위치: 맞은편 좌석에서 바라보는 시점. 개입하지 않는 목격자.

참조거부: 낙엽 + 쓸쓸함 조합 전체 거부. 정지용의 정적 이미지 패턴 참고하되 답습 금지.

형식결정:
  길이: short
  구조: 자유시
  시제: 현재

금지: "낙엽처럼", "쓸쓸하다", 계절 이름 직접 언급, 결론 달기
```

## JSON 스키마 (기계 처리용)

학습 데이터 포맷 및 자동 검증을 위한 구조화 표현:

```json
{
  "씨앗이미지": "string (40~200자)",
  "핵심개념": "string (20~80자)",
  "언어후보": ["string", "..."],
  "긴장관계": "string | null",
  "독자위치": "string | null",
  "참조거부": "string | null",
  "형식결정": {
    "길이": "short | medium | long",
    "구조": "자유시 | 연시 | 산문시",
    "시제": "현재 | 과거 | 혼합"
  },
  "금지": ["string", "..."]
}
```

**필수/선택 구분:**
- 필수: `씨앗이미지`, `핵심개념`, `형식결정`, `금지`
- 선택: `언어후보`, `긴장관계`, `독자위치`, `참조거부`

## 창작 노트 → 시 추적성 설계

### 문제

창작 노트를 만들었는데 최종 시에 어떤 필드가 실제로 반영됐는지 모르면,
- 어떤 필드가 생성 품질에 기여했는지 알 수 없다
- 모델이 노트를 무시하고 시를 쓰는지 탐지할 수 없다
- ablation 실험 설계가 어렵다

### 추적 방법: 인라인 태깅

시의 각 행 또는 이미지 단위에 소스 필드를 태깅한다.
이는 학습 시 사용하는 것이 아니라 **데이터 제작 및 검증 단계**에서 사용한다.

```
[시 원문]
잠든 남자의 손 위에       ← [씨앗이미지]
낙엽이 내린다             ← [언어후보: "낙엽이 내린다"]
남자는 모른다             ← [씨앗이미지, 핵심개념]
가을이 그를 택했다는 걸    ← [긴장관계: 수면/계절변화] [금지 경계: 계절 직접 언급 — 검토 필요]
```

### 추적성 점수

각 창작 노트 필드가 시에 반영된 비율을 계산:

```
추적성 점수 = (시에 반영된 필드 수) / (필수 필드 수 + 사용된 선택 필드 수)
```

**기준:**
- 0.8 이상: 노트와 시의 연결이 강함 — 좋은 학습 데이터
- 0.5~0.8: 부분 반영 — 검토 후 사용
- 0.5 미만: 노트가 시와 분리됨 — 필터링 또는 재생성

### 금지 필드 준수 검사

```python
# 자동 검사 로직 (의사 코드)
def check_forbidden(poem_text, forbidden_list):
    violations = []
    for item in forbidden_list:
        if item in poem_text:
            violations.append(item)
    return violations
```

금지 항목이 시에 등장하면 해당 샘플을 플래그 처리한다.

## 스키마 버전 관리

창작 노트는 수정 과정에서 변형된다. 버전을 추적한다:

```json
{
  "v0": { "씨앗이미지": "...", "핵심개념": "...", "금지": [...] },
  "v1": { "금지": ["추가 금지 항목"], "언어후보": ["수정된 후보"] },
  "v_final": "v1"
}
```

수정 이유(diff)를 함께 기록:
```
v0→v1 변경 이유: 초안에서 "가을이라는 단어가 직접적"이라는 자기 비평 발생 →
                 금지 목록에 "가을" 직접 언급 추가
```

## 미결 사항

- 선택 필드를 포함한 샘플과 최소 버전 샘플의 비율 — 7:3 또는 5:5?
- `언어후보`의 후보가 실제로 시에 등장하지 않아도 되는가? (탐색의 의미로 충분한가)
- 긴장관계 필드가 시의 novelty와 상관관계가 있는지 실험 필요
- 자동 추적성 점수 계산을 위한 NLP 파이프라인 설계 (단순 키워드 매칭 한계)
