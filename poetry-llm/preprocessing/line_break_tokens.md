# 행갈이/연갈이 특수 토큰 설계

## 핵심 아이디어

일반 텍스트로 시를 학습시키면 모델은 행갈이를 단순히 "줄바꿈 문자"로 처리한다.
그러나 시에서 행갈이는 **의미를 생산하는 구조적 결정**이다:

- 어디서 끊는가 → 호흡, 강조, 의미의 단위
- 얼마나 짧게/길게 끊는가 → 리듬과 속도
- 왜 여기서 끊는가 → 시적 의도

특수 토큰 `<행갈이>`, `<연갈이>`를 삽입하면 모델이 이 결정을
**명시적 출력 행위**로 학습하게 된다.

## 토큰 설계

```
<행갈이>    — 시행의 끝. 같은 연(stanza) 내에서의 줄바꿈.
<연갈이>    — 연(stanza)과 연 사이의 구분. 빈 줄에 해당.
<시작>      — 시 텍스트의 시작
<끝>        — 시 텍스트의 끝
```

## 변환 예시

### 원문 (일반 텍스트)

```
눈이 오는가 북쪽엔
함박눈 쏟아져 내리는가

험한 벌판을 달리는
울 오빠는 맘 놓고 있느냐
```

### 토큰 삽입 후

```
<시작>눈이 오는가 북쪽엔<행갈이>
함박눈 쏟아져 내리는가<연갈이>
험한 벌판을 달리는<행갈이>
울 오빠는 맘 놓고 있느냐<끝>
```

## 추가 세분화 (고려 사항)

행갈이에도 **유형**이 있을 수 있다:

| 토큰 | 의미 | 예 |
|------|------|-----|
| `<행갈이>` | 기본 행 끝 | 대부분의 경우 |
| `<행갈이:짧>` | 매우 짧은 행 끝 (1~3음절) | 충격, 강조 |
| `<행갈이:걸침>` | 행걸침(enjambment) — 의미가 다음 행으로 이어짐 | 흐름의 연속 |

> 이 세분화는 모델 복잡성을 높이므로 초기 파일럿 후 결정.

## 발음과의 연계

행갈이는 발음(운율) 결정과 직결된다.
→ [pronunciation.md](pronunciation.md)에서 발음 토큰과 연계 방식 기술.

## 학습 목표

이 토큰 설계로 모델이 학습해야 하는 것:

1. **언제** 행을 끊을 것인가 (행갈이 위치 결정)
2. **언제** 연을 나눌 것인가 (연갈이 위치 결정)
3. 위 결정이 **의미와 리듬에 어떤 효과**를 만드는가 (시론 데이터와 연계)

## 미결 사항

- 행갈이 유형 세분화 여부 (단순 vs 세분화)
- 영어 시의 경우 `<line-break>`, `<stanza-break>`로 별도 처리?
- 타이포그래피 실험시 (들여쓰기, 중앙 정렬 등) 추가 토큰 필요 여부
