# 발음 토큰 및 음성 레이어

## 왜 발음 토큰이 필요한가

시는 **눈으로 읽는 동시에 귀로 듣는 형식**이다.
한국어 현대시도 발화 시 음악적 특성이 강하게 드러난다:

- 음절의 수와 강세 패턴 (4·4조, 3·4조 등의 민요 리듬)
- 두운(頭韻), 각운(脚韻), 내운(內韻)
- 음성 상징 (의성어, 의태어의 음향 효과)
- 장모음 / 단모음의 질감 차이

텍스트만 학습하면 이 정보가 암묵적으로만 습득된다.
발음 토큰을 명시하면 모델이 **소리의 논리**를 명시적으로 배울 수 있다.

## 발음 표기 방식

### 옵션 1 — 국제음성기호 (IPA)
```
봄 → /pom/
눈 → /nun/ (눈(snow)) 또는 /nun/ (눈(eye))
```
장점: 국제 표준, 다국어 통일
단점: 한국어 특유의 현상(경음화, 연음 등) 표기 복잡

### 옵션 2 — 한국어 표준 발음 표기
```
국어 → 구거
않다 → 안타
```
장점: 실제 발화에 가까움
단점: 표준화 필요

### 옵션 3 — 음절 구조 표기 (권장 초안)
```
봄비가 <음절:3> 내린다 <음절:3> — 음절수 표시
눈(snow) <발음:눈> — 동음이의어 명시
꽃이 <연음:꼬치> — 연음 현상
```

## 예시: 발음 토큰 삽입

### 원문
```
진달래꽃
나 보기가 역겨워
가실 때에는
말없이 고이 보내 드리오리다
```

### 발음 토큰 삽입 후
```
<시작>진달래꽃<음절:5><행갈이>
나 보기가 역겨워<음절:7><행갈이>
가실 때에는<음절:5><행갈이>
말없이<연음:마럽씨> 고이 보내 드리오리다<음절:11><끝>
```

## 음악성 정보 레이어

단순 발음을 넘어 음악성 정보를 추가하는 방식:

```
<두운:ㅂ>  — 행 내 두운 시작
<각운:다>  — 행 끝 각운
<반복:봄>  — 특정 어휘의 반복
<장모음>   — 길게 발음되는 부분
```

## 구현 방법

1. **형태소 분석기** + **표준발음법 룰**: KoNLPy, Kiwi 등으로 형태소 분석 후 표준발음법 적용
2. **음절 카운터**: 간단한 규칙 기반
3. **운율 분석기**: 연구 목적 도구 활용 또는 자체 개발

## 미결 사항

- 발음 표기 방식 최종 선택 (IPA vs 표준발음 vs 음절 구조)
- 얼마나 세밀하게 표기할 것인가 (음절수만 vs 완전 IPA)
- 영어/일본어 시에 동일한 레이어를 적용할 것인가
- 음악성 정보 레이어를 별도 토큰으로 할 것인가, 임베딩 레이어로 할 것인가
