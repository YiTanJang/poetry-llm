# 학습 데이터 생성 포맷

## 기본 원칙

같은 시 텍스트를 **여러 관점의 학습 데이터**로 변환하면
모델이 시를 다각도로 이해한다. 단순히 텍스트를 외우는 것이 아니라
시의 **구조, 의도, 효과, 과정**을 배우게 된다.

## 포맷 1 — 완성시 (기본)

가장 기본적인 형태. 시 텍스트 그 자체.

```
[시 텍스트 with 행갈이/연갈이 토큰 + 발음 토큰]
```

## 포맷 2 — 시 + 시론 페어

시와 그 시에 대한 비평/분석을 페어로 학습.

```json
{
  "시": "[시 텍스트]",
  "분석": "[시론 또는 평론 텍스트에서 추출한 이 시에 대한 분석]"
}
```

## 포맷 3 — 창작 노트 → 시 (CoT 파이프라인의 핵심)

시인의 산문(에세이, 인터뷰, 창작 노트)에서 창작 과정을 역추론하여
"창작 노트 → 시" 형태의 학습 데이터를 생성.

```json
{
  "창작노트": "봄비가 내리는 날 어머니 생각이 났다. 비의 소리가 어머니의 목소리처럼 들렸다. 비를 어머니로 치환하되 직접 말하지 않고 싶었다...",
  "시": "[결과 시 텍스트]"
}
```

> **주의**: 실제 시인의 창작 노트가 없는 경우,
> 강력한 LLM(Claude, GPT-4o)을 사용하여 역추론 창작 노트를 생성할 수 있음.
> 단, 생성된 노트의 품질 검증 필요.

## 포맷 4 — 수정 과정 데이터

시의 초고 → 수정 → 최종본 과정을 학습 데이터로.
(시인의 실제 수정 원고가 있다면 최상, 없으면 생성)

```json
{
  "초고": "[시 초안]",
  "수정의견": "3연 2행의 이미지가 너무 직접적이다. 더 우회적으로...",
  "수정본": "[수정된 시]"
}
```

## 포맷 5 — 시 → 설명 (역방향)

완성시에서 시인의 의도와 기법을 역추론하는 형태.

```json
{
  "시": "[시 텍스트]",
  "해설": "이 시는 __를 통해 __를 표현한다. __연에서 행갈이가 _에 위치한 이유는..."
}
```

## 포맷 6 — 빈칸 채우기 (구조 학습)

행갈이/연갈이 위치를 빈칸으로 만들어 위치 예측 학습.

```
눈이 오는가 북쪽엔 [?] 함박눈 쏟아져 내리는가 [?] ...
정답: <행갈이> ... <연갈이>
```

## 포맷 7 — 소재 → 시 (창작 요청)

특정 소재/개념이 주어졌을 때 시를 생성하는 지시 튜닝.

```json
{
  "instruction": "다음 소재로 시를 써라: 지하철역 환승 통로에서 마주친 낯선 사람",
  "output": "[시 텍스트]"
}
```

## 포맷 8 — Novelty 실험 데이터

기존에 없던 소재 조합을 의도적으로 실험.
LLM으로 "시에서 잘 쓰지 않는 소재 + 전통 시 기법" 조합 생성.

```json
{
  "실험": "소재: 반도체 공정 / 기법: 시조 율격",
  "시": "[생성 시 텍스트]",
  "novelty_score": 0.87
}
```

## 데이터 믹스 비율 (초안)

| 포맷 | 비중 | 비고 |
|------|------|------|
| 포맷 1 (완성시) | 30% | 기본 언어 패턴 |
| 포맷 2 (시+시론) | 20% | 메타 이해 |
| 포맷 3 (창작노트→시) | 20% | CoT 학습 핵심 |
| 포맷 4 (수정 과정) | 15% | 반복 수정 학습 |
| 포맷 5 (역방향 해설) | 10% | 비평 능력 |
| 포맷 6~8 (기타) | 5% | 특수 능력 |
