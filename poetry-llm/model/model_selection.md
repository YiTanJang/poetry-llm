# 베이스 모델 선정

## 선정 기준

| 기준 | 중요도 | 설명 |
|------|--------|------|
| 한국어 능력 | ★★★★★ | 한국 현대시의 언어 감각 기반 |
| 파라미터 수 | ★★★★☆ | 20~40B — 언어 깊이와 컴퓨팅 비용의 균형 |
| 오픈 가중치 | ★★★★★ | 풀 파인튜닝 가능성 필수 |
| 컨텍스트 길이 | ★★★☆☆ | 창작 노트 + 시 = 충분한 컨텍스트 필요 |
| 다국어 지원 | ★★★☆☆ | 외국어 시 학습 데이터 처리 |

## 주요 후보 모델

### Qwen2.5-32B (권장 1순위)
- **한국어 능력**: 상 — 한국어 포함 다국어 사전학습
- **컨텍스트**: 128K 토큰
- **특이사항**: 32B는 A100 80GB 2장으로 풀 파인튜닝 가능 (ZeRO-3)
- **라이선스**: Apache 2.0

### LLaMA-3.1-70B
- **한국어 능력**: 중상 — 영어 중심이나 다국어 능력 양호
- **컨텍스트**: 128K 토큰
- **특이사항**: 70B는 풀 파인튜닝에 A100 80GB 8장 이상 필요 — 비용 높음
- **라이선스**: LLaMA 3 Community License

### Mistral-22B (Mixtral-8x22B)
- **한국어 능력**: 중 — 영어/유럽어 중심
- **특이사항**: MoE(Mixture of Experts) 구조 — 파인튜닝 복잡성 증가
- **라이선스**: Apache 2.0

### EXAONE-3.5-32B (LG AI Research)
- **한국어 능력**: 최상 — 한국어 특화 모델
- **특이사항**: 한국어 특화이므로 시 도메인 전이가 용이할 수 있음
- **라이선스**: EXAONE AI Model License (상업적 이용 제한 확인 필요)

### SOLAR-10.7B (Upstage)
- **파라미터**: 10.7B — 목표 범위 하한
- **한국어 능력**: 상 — 한국어 특화
- **특이사항**: 소형 모델로 실험 속도 빠름 → 파일럿에 적합

## 권장 전략

```
Phase 1 (파일럿): SOLAR-10.7B 또는 Qwen2.5-7B
  └── 빠른 실험 사이클로 전처리/데이터 파이프라인 검증

Phase 2 (본 파인튜닝): Qwen2.5-32B
  └── 한국어 능력 + 오픈 가중치 + 컴퓨팅 실현 가능성의 균형

Phase 3 (선택): LLaMA-3.1-70B 또는 EXAONE-32B 비교 실험
```

## 컴퓨팅 요구사항 (추정)

| 모델 | 풀 파인튜닝 (fp16) | ZeRO-3 필요 GPU |
|------|------------------|----------------|
| 10.7B | ~21 GB VRAM | A100 40GB 1장 가능 |
| 32B | ~64 GB VRAM | A100 80GB 2장 |
| 70B | ~140 GB VRAM | A100 80GB 4~8장 |

## 미결 사항

- 한국어 특화 모델(EXAONE) vs 범용 다국어 모델(Qwen) 최종 결정
- 외국어 시 학습 데이터와 한국어 모델의 궁합
- 컴퓨팅 비용 확보 방법 (개인 GPU 서버 vs 클라우드)
