# SFT 훈련 태스크 분류

## 설계 원칙

**"하네스에 맞춰서 훈련한다."**

하네스가 모델에게 요구하는 능력:
1. 씨앗을 읽고 스크래치패드를 쓴다
2. 스크래치패드에서 도구를 호출한다
3. 도구 결과를 스크래치패드에 통합한다
4. 스크래치패드에서 시로 전환한다
5. 특수 토큰(`<시작>`, `<끝>`, `<행갈이>`, `<연갈이>`)을 정확하게 쓴다

각 훈련 태스크는 이 다섯 가지 능력을 단독으로, 또는 조합으로 키우도록 설계된다. "시를 잘 쓰는 모델"이 아니라 "하네스 안에서 잘 작동하는 모델"을 만드는 것이 목표다.

> 탐색중 — 각 그룹 내 태스크 행 추가 가능. 7개 그룹 구조와 각 그룹의 핵심 목적은 확정.

---

## 태스크 그룹 분류

### Group 1 — 포맷 학습

하네스 특수 토큰과 출력 형식을 기계적으로 학습한다. 창의성보다 정확성이 목적.

| 태스크 | 입력 | 출력 |
|--------|------|------|
| 토큰 삽입 | 평문 시 (행 구분 있음) | `<시작>` + `<행갈이>` + `<연갈이>` + `<끝>` 적용 버전 |
| 토큰 파싱 | `<시작>...<끝>` 형식의 시 | 행 번호별 텍스트 추출 |

비중 목표: 전체의 ~5%

---

### Group 2 — 시 이해 (판별형)

시의 어휘·구조·미학적 긴장을 이해하는 능력을 키운다. 생성보다 판별이 목적.

| 태스크 | 입력 | 출력 | 훈련 목적 |
|--------|------|------|---------|
| 단어 빈칸 채우기 | 시에서 단어 하나 `[MASK]` 처리 | 원래 단어 | 어휘 선택 감각 |
| 행 빈칸 채우기 | 시에서 행 하나 제거 | 제거된 행 | 행 수준 리듬·맥락 |
| 연 빈칸 채우기 | 시에서 연 하나 제거 | 제거된 연 | 연 수준 구조 |
| 어울리지 않는 행 찾기 | 시 + "이 시에서 가장 이질적인 행은?" | 해당 행 번호 + 이유 | 일관성 감각 |
| 시 비교 | 동일 씨앗에서 나온 두 시 | 어느 쪽이 더 참신한지 + 이유 | 판단 능력 (reward model 사전 학습) |

비중 목표: 전체의 ~20%

---

### Group 3 — 직접 생성

가장 기본적인 생성 능력. 스크래치패드 없는 단순 형식부터 전체 하네스 플로우까지.

| 태스크 | 입력 | 출력 | 훈련 목적 |
|--------|------|------|---------|
| 씨앗 → 시 | 씨앗 이미지 | 완성된 시 (스크래치패드 없음) | 기본 생성 |
| 씨앗 + 금지어 → 시 | 씨앗 + 금지어 목록 | 금지어를 쓰지 않은 시 | 제약 준수 |
| 씨앗 → 스크래치패드 → 시 | 씨앗 | 스크래치패드 + 시 | 전체 하네스 플로우 |
| 시 앞부분 → 이어 쓰기 | 시의 처음 N행 | 나머지 행들 | 일관성 있는 연장 |
| 형식 제약 생성 | 씨앗 + "7행 이내" / "현재 시제" 등 | 제약을 만족한 시 | 형식 제어 |

비중 목표: 전체의 ~25%

---

### Group 4 — 스크래치패드 프로세스

스크래치패드가 시와 **인과 관계**를 갖도록 학습시킨다. 단순히 앞에 붙이는 텍스트가 아니라 시를 만드는 실제 탐색 과정이 되어야 한다.

| 태스크 | 입력 | 출력 | 훈련 목적 |
|--------|------|------|---------|
| 역방향 스크래치패드 | 완성된 시 | "이 시가 나오기까지 어떤 탐색이 있었을까" — 사후 재구성 스크래치패드 | 스크래치패드-시 인과 구조 |
| 스크래치패드 이어 쓰기 | 씨앗 + 미완성 스크래치패드 | 나머지 스크래치패드 + 시 | 탐색 흐름 파악 |
| 스크래치패드 연결성 판단 | 스크래치패드 + 시 | "스크래치패드가 실제로 이 시에 반영됐는가?" + 근거 | 자기 점검 능력 |

비중 목표: 전체의 ~15%

> **역방향 스크래치패드가 핵심이다.** 실제 인간 시인의 창작 노트를 확보하기 어려우므로, 기존 시에서 역으로 스크래치패드를 생성하는 방법으로 훈련 데이터를 확보할 수 있다. 이 태스크를 통해 모델은 "스크래치패드는 시를 만드는 과정이어야 한다"는 것을 내재화한다.

---

### Group 5 — 수정 태스크

비판을 받아들여 시를 개선하는 능력. DPO 이전에 기본 수정 능력을 SFT로 학습한다.

| 태스크 | 입력 | 출력 | 훈련 목적 |
|--------|------|------|---------|
| 비판 → 수정 | 시 + 구체적 비판 ("3행의 이미지가 진부하다") | 수정된 시 + 어디를 왜 바꿨는지 | 피드백 수용 |
| 진부한 이미지 교체 | 시 + "이 부분이 상투적이다" 표시 | 해당 부분만 수정된 시 | 미학적 자기 교정 |
| 두 버전 비교 + 설명 | 시 A, 시 B (같은 씨앗) | "A/B 중 어느 쪽이 더 낫고 왜인가" | 판단 근거 생성 |

비중 목표: 전체의 ~15%

---

### Group 6 — 도구 통합 (하네스 핵심)

창작 도구 호출 결과를 스크래치패드에 통합하는 능력. 이것이 하네스를 쓰는 모델의 고유 능력이다.

| 태스크 | 입력 | 출력 | 훈련 목적 |
|--------|------|------|---------|
| 랜덤 단어 → 스크래치패드 → 시 | 씨앗 + `random_words` 결과 5개 | 단어 일부를 활용한 스크래치패드 + 시 | 도구 결과 통합 |
| 뉴스 조각 → 시 | 씨앗 + `news_snippet` 결과 | 뉴스에서 영감을 받은 스크래치패드 + 시 | 현실 자극 통합 |
| 도구 호출 흔적 이어가기 | 씨앗 + 도구 호출이 포함된 미완성 스크래치패드 | 나머지 스크래치패드 + 시 | 도구 활용 흐름 파악 |
| 도구 없이 쓰기 | 씨앗 (도구 없음) | 도구 호출 없는 스크래치패드 + 시 | 도구 없이도 작동해야 함 |

비중 목표: 전체의 ~15%

학습 데이터 내 도구 사용 트레이스 비율: 도구 사용 60% / 미사용 40%로 시작 (Q26 실험에서 조정)

---

### Group 7 — 메타 판단

생성된 시의 품질을 평가하는 능력. 나중에 리워드 모델 학습의 사전 준비가 된다.

| 태스크 | 입력 | 출력 | 훈련 목적 |
|--------|------|------|---------|
| 씨앗 적합성 평가 | 씨앗 + 시 | "이 씨앗에서 이 시가 잘 나왔는가?" 점수 + 근거 | 자기 평가 |
| 참신성 판단 | 시 | "어떤 이미지/표현이 새롭고 어떤 것이 상투적인가" | novelty 감각 |
| sanity check | 시 | 비어 있는지 / 반복 과다인지 / 행 수 범위 확인 | 자동 필터 보조 |

비중 목표: 전체의 ~5%

---

## 전체 비중 요약

| 그룹 | 비중 | 주요 능력 |
|------|------|---------|
| Group 1 포맷 | 5% | 특수 토큰 정확성 |
| Group 2 이해 | 20% | 어휘·구조 감각, 판단력 |
| Group 3 생성 | 25% | 기본 생성, 제약 준수 |
| Group 4 스크래치패드 | 15% | 스크래치패드-시 인과 구조 |
| Group 5 수정 | 15% | 피드백 수용, 개선 |
| Group 6 도구 통합 | 15% | 하네스 핵심 능력 |
| Group 7 메타 판단 | 5% | 자기 평가, 필터 보조 |

> 이 비중은 가설이다. Phase 1 파일럿 학습 후 각 태스크의 기여도를 측정하여 조정한다.

---

## 훈련 데이터 생성 전략

**기존 시에서 추출 가능한 태스크:**
- Group 1, 2는 기존 시 코퍼스만으로 자동 생성 가능 (마스킹, 제거)
- Group 4 역방향 스크래치패드는 Claude/GPT-4로 생성

**새로 생성해야 하는 태스크:**
- Group 3 (씨앗 → 스크래치패드 → 시): Claude/GPT-4 + 페르소나로 생성
- Group 5 (수정): 시 + Claude/GPT-4 비판 생성 → 수정 쌍
- Group 6 (도구 통합): 도구 결과를 포함한 트레이스를 Claude/GPT-4로 생성

---

## 미결 사항

- 그룹별 비중 최적화: 파일럿 학습 후 어떤 태스크가 하네스 성능에 가장 기여하는지 측정 필요
- Group 4 역방향 스크래치패드의 품질 기준: Claude/GPT-4가 생성한 역방향 스크래치패드가 실제로 창작 과정처럼 보이는지, 아니면 사후 합리화처럼 보이는지 → 훈련 신호로서의 유효성 검증 필요
- Group 6 도구 통합 학습 시 도구 결과를 학습 데이터에 고정 기록할 것인가, 추론 시 실시간 호출로 대체할 것인가 (재현성 문제)
- Group 2 시 비교 태스크와 DPO의 관계: Group 2에서 판단 능력을 학습한 모델이 스스로 DPO 선호 쌍을 만들 수 있는가 (self-play 가능성)
