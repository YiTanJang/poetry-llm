# 평가 파이프라인

## 개요

이 문서는 생성된 시를 자동 지표, 인간 평가, 미학적 기준의 세 층위로 평가하고
그 결과를 다음 학습 이터레이션에 피드백하는 end-to-end 절차를 정의한다.

세 평가 유형의 역할은 명확히 구분된다:

| 평가 유형 | 역할 | 실행 빈도 |
|-----------|------|-----------|
| 자동 지표 | 1차 필터 — 스케일 확보, 예산 절감 | 매 체크포인트 |
| 인간 평가 | 2차 검증 — 자동 지표가 놓치는 미학 포착 | 마일스톤마다 |
| 미학적 평가 | 최종 게이트 — 출력물 공개 및 보고서 작성 기준 | 최종 릴리즈 전 |

---

## 1. 평가 단계 구성

### 1단계 — 자동 평가 (매 체크포인트)

학습 중 **모든 체크포인트**에서 자동으로 실행된다.
체크포인트는 250 스텝 단위로 설정하되, 풀 파인튜닝 시 1 epoch 완료 시점에도 추가 실행한다.

- [auto_metrics.md](auto_metrics.md)에 정의된 지표 전체를 계산
- 결과를 `eval_log/{checkpoint_id}/auto_scores.json`에 기록
- n-gram 독창성 < 0.7 **또는** 임베딩 거리 < 0.3인 시는 이후 단계로 진행 불가

### 2단계 — 인간 평가 (마일스톤)

마일스톤은 다음 세 시점으로 정의한다:

| 마일스톤 | 시점 | 설명 |
|----------|------|------|
| M1 | 파일럿 LoRA 완료 | Qwen2.5-10.7B 파인튜닝 검증 |
| M2 | 풀 파인튜닝 1차 완료 | Qwen2.5-32B 첫 버전 |
| M3 | 반복 수정 파이프라인 포함 완료 | CoT → 초안 → 수정 루프 포함 |

각 마일스톤에서 자동 평가 통과 시만 인간 평가를 진행한다.
[human_evaluation.md](human_evaluation.md)의 A(전문가)·B(일반 독자)·C(Turing test) 방식을 모두 적용한다.

### 3단계 — 미학적 최종 게이트 (릴리즈 전)

[aesthetic_quality.md](aesthetic_quality.md)의 6개 기준(경제성, 필연성, 긴장, 낯설게 하기, 열린 끝, 음악성)을 항목별로 체크한다.
전문가 평가단이 각 항목을 1~5점으로 평가하며, **평균 3.5점 이상**이어야 공개 가능 판정을 받는다.

---

## 2. 자동 평가 → 인간 평가 필터링

### 필터 설계 이유

인간 평가는 전문가 섭외, 보상 지급, 시간 비용이 수반된다.
자동 필터 없이 모든 생성시를 인간 평가에 넘기면 평가자 피로와 예산 낭비가 발생한다.
자동 지표로 명백히 실패한 시(표절에 가까운 n-gram 중복, 코퍼스와 거의 동일한 임베딩)를
사전 제거하면 인간 평가 투입량을 대폭 줄일 수 있다.

### 통과 기준

다음 두 조건을 **모두** 충족해야 인간 평가 대상이 된다:

```
조건 A: novelty_ngram ≥ 0.7   (4-gram 기준 코퍼스 중복 30% 미만)
조건 B: embedding_distance ≥ 0.3  (코퍼스 내 최근접 시와의 코사인 거리)
```

소재 novelty(material_novelty)는 필터 조건에 포함하지 않는다.
소재 novelty가 낮아도 수사·표현이 새로울 수 있으며, 이는 인간 평가에서 더 잘 포착되기 때문이다.

### 예상 통과율

> 가설: 파인튜닝 이전 베이스 모델 대비 파인튜닝 후 모델에서 통과율이 높아질 것으로 예상.
> 파일럿 단계에서 실측값으로 보정 필요.

인간 평가로 넘어가는 시의 수는 체크포인트당 **50편**으로 제한한다 (→ 4절 참조).

---

## 3. 복합 점수 산정 방식

### 수식

세 평가 유형을 하나의 최종 점수로 통합한다:

```
final_score = w_auto * score_auto
            + w_human * score_human
            + w_aesthetic * score_aesthetic
```

### 세부 구성 요소

**score_auto** (자동 점수, 0~1):
[auto_metrics.md](auto_metrics.md)의 종합 점수 수식을 그대로 사용:
```
score_auto = 0.4 * novelty_ngram
           + 0.2 * embedding_diversity
           + 0.2 * structure_accuracy
           + 0.1 * material_novelty
           + 0.1 * batch_diversity
```

**score_human** (인간 점수, 0~1):
전문가 평가(A) 5개 항목의 평균을 5점 만점 → 0~1로 정규화:
```
score_human = mean(언어밀도, 이미지선명도, 구조적필연성, 독창성, 여운) / 5
```

**score_aesthetic** (미학 점수, 0~1):
미학적 6개 기준의 평균을 5점 만점 → 0~1로 정규화:
```
score_aesthetic = mean(경제성, 필연성, 긴장, 낯설게하기, 열린끝, 음악성) / 5
```

### 가중치

| 구성 요소 | 가중치 | 근거 |
|-----------|--------|------|
| w_auto | **0.25** | 필요 조건이지 충분 조건이 아님; 자동 지표 단독으로 품질 보장 불가 |
| w_human | **0.45** | 독자 경험이 최종 목적; 감동·여운은 인간만 측정 가능 |
| w_aesthetic | **0.30** | 프로젝트의 핵심 목표인 novelty+전통 시학의 균형을 반영 |

> 가설: 이 가중치는 파일럿 인간 평가 결과와 상관 분석 후 조정.
> 특히 w_auto가 너무 낮으면 자동 필터의 예측력을 과소평가하게 됨.

### 최종 판정 기준

| final_score 범위 | 판정 |
|-----------------|------|
| ≥ 0.75 | 우수 — 릴리즈 후보 |
| 0.55 ~ 0.74 | 보통 — 추가 수정 후 재평가 |
| < 0.55 | 실패 — 실패 분석 프로토콜 적용 (→ 6절) |

## 4. 평가 데이터셋 구성

### 평가 규모

| 마일스톤 | 총 생성량 | 자동 필터 통과 예상 | 인간 평가 대상 |
|----------|-----------|---------------------|----------------|
| M1 (파일럿) | 200편 | ~80편 | 50편 |
| M2 (풀 파인튜닝) | 500편 | ~200편 | 50편 |
| M3 (CoT 루프 포함) | 500편 | ~200편 | 50편 |

인간 평가는 50편으로 고정한다. 이는 전문가 5~10인이 세션당 처리 가능한 적정량이다.

### 참조 코퍼스

자동 지표의 기준이 되는 비교 코퍼스는 다음으로 구성한다:

```
참조 코퍼스 = 학습 데이터 전체
            + 평가용 추가 수집 시(학습 데이터와 중복 없는 공개 시집 100~200편)
            + GPT-4o 생성시 50편
            + Claude 생성시 50편
```

GPT-4o·Claude 생성시를 포함하는 이유: 단순히 다른 LLM보다 novel한지를 구별하기 위함.

### 프롬프트 다양성 확보

생성 프롬프트는 다음 4개 축으로 교차 샘플링한다:

| 축 | 예시 |
|----|------|
| 소재 유형 | 전통 소재 / 현대·산업 소재 / 추상 소재 |
| 감정 톤 | 애도 / 긴장 / 무감각 / 경이 |
| 형식 조건 | 3연 이내 / 단시 / 산문시 |
| 창작 노트 유무 | CoT 창작 노트 포함 / 미포함 |

4축의 조합 중 50개를 무작위로 샘플링하여 평가 프롬프트 세트를 구성한다.
동일 프롬프트를 마일스톤 간 재사용하여 버전 간 직접 비교가 가능하게 한다.

---

## 5. 평가 결과 → 학습 루프 피드백

### 피드백 메커니즘

평가 결과는 다음 이터레이션의 학습 데이터 구성에 반영된다.

**우수 판정(final_score ≥ 0.75)인 생성시**:
- 해당 시의 창작 노트(CoT)와 시 쌍을 학습 데이터에 추가
- `data/generated_high_quality/` 디렉토리에 보관
- 다음 파인튜닝 라운드에서 이 데이터 비중을 최대 10%까지 허용

**실패 판정(final_score < 0.55)인 생성시**:
- 실패 유형별 분류(→ 6절 참조)
- 데이터 문제로 분류된 경우: 해당 소재·형식의 학습 데이터를 추가 수집
- 프롬프트 문제인 경우: 창작 노트 포맷 개정 (`generation/` 섹션 담당)

**자동 지표 가중치 재조정**:
파일럿(M1) 이후 `score_auto`와 `final_score`의 Pearson 상관계수를 계산한다.
상관계수 < 0.5이면 auto_metrics.md의 가중치 재검토.

### 소재별·형식별 통계 추적

매 마일스톤마다 다음 표를 갱신하여 취약 영역을 파악한다:

```
| 소재 유형 | 평균 final_score | 우수 비율 | 비고 |
|-----------|-----------------|-----------|------|
| 전통 소재 | ...             | ...       | ... |
| 현대 소재 | ...             | ...       | ... |
| 추상 소재 | ...             | ...       | ... |
```

우수 비율이 낮은 소재 유형은 해당 유형의 학습 데이터를 보강한다.

---

## 6. 실패 분석 프로토콜

### 실패 진단 트리

final_score < 0.55인 시에 대해 아래 순서로 진단한다:

```
[실패 시]
    │
    ├─ score_auto < 0.4?
    │       ├─ YES: "표절/반복 문제" → 데이터 문제 또는 과적합 의심
    │       └─ NO: 다음 단계로
    │
    ├─ score_human < 0.4?
    │       ├─ YES: "감동 실패" → 아래 세 원인 구분
    │       │        ├─ 평가자 코멘트에 "의미 없음/난해함" 언급
    │       │        │       → 프롬프트(창작 노트) 문제
    │       │        ├─ 평가자 코멘트에 "뻔함/예측 가능" 언급
    │       │        │       → 데이터 문제 (학습 데이터 novelty 부족)
    │       │        └─ 평가자 코멘트에 "이미지가 없음/공허함" 언급
    │       │                → 모델 용량 문제 (10.7B → 32B 스케일업 검토)
    │       └─ NO: 다음 단계로
    │
    └─ score_aesthetic < 0.4?
            └─ YES: "미학 실패" → 항목별 세분화
                     ├─ 경제성·필연성 낮음 → 창작 노트 구조 개선
                     ├─ 긴장·낯설게하기 낮음 → novelty 필터 임계값 상향 검토
                     └─ 열린끝·음악성 낮음 → 학습 데이터의 해당 유형 시 추가 수집
```

### 원인 유형별 대응

| 원인 유형 | 진단 신호 | 대응 방향 |
|-----------|-----------|-----------|
| 데이터 문제 | score_auto 낮음, 반복·표절 패턴 | 해당 소재·형식 데이터 추가 수집; 중복 제거 강화 |
| 프롬프트 문제 | score_human 낮음, "난해/의미없음" 코멘트 | 창작 노트(CoT) 포맷 개정 (`generation/` 담당자 협의) |
| 모델 용량 문제 | score_human 낮음, "이미지 없음" 코멘트, 소·대 모델 격차 | 10.7B → 32B 스케일업 또는 레이어별 분석 |
| 평가 설계 문제 | 평가자 간 불일치(inter-rater agreement < 0.5) | 평가 지침 재설계; 예시 시 추가 제공 |

### 실패 케이스 아카이브

실패 판정 시는 삭제하지 않고 `eval_log/failures/` 에 보관한다.
실패 이유 태그(데이터/프롬프트/모델/평가)와 진단 노트를 함께 기록한다.
이는 이후 실패 패턴 분석과 데이터 정제에 활용된다.

## 미결 사항

- [ ] `score_auto`와 `final_score` 간 상관계수 — 파일럿(M1) 완료 후 실측 필요. 현재 가중치(w_auto=0.25)는 가설 기반.
- [ ] 평가자 간 일치도(inter-rater agreement) 측정 방법 미정. Cohen's κ 또는 Krippendorff's α 중 선택 필요.
- [ ] 자동 평가 실행 환경 — GPU 서버에서 KoSimCSE 추론이 병목이 될 수 있음. 배치 크기·캐싱 전략 필요.
- [ ] 인간 평가 플랫폼 미정 — 자체 구축 vs. 설문 도구(Google Forms, Notion 등) 선택.
- [ ] 생성시의 창작 노트(CoT)를 학습 데이터 피드백에 포함할 때 저작권·출처 표기 방침.
- [ ] 일반 독자 평가(B) 30~50인 모집 방법 — 문학 커뮤니티, 온라인 리크루팅 등.
- [ ] 소재 novelty 지표를 필터 조건에서 제외한 결정이 타당한지 파일럿 이후 재검토.
