# 인간 평가 설계

## 평가의 목적

자동 지표로 측정할 수 없는 것들:
- 시가 독자를 **감동시키는가**
- 언어 선택이 **필연적으로 느껴지는가**
- 이미지가 **생생하게 느껴지는가**
- 시가 **여운을 남기는가**

## 평가 유형

### A. 전문가 평가 (Expert Evaluation)
**평가자**: 시인, 문학 연구자, 문예지 편집자
**규모**: 5~10인

**평가 방식**:
- 생성시 + 비교용 인간 시를 섞어 블라인드 제시
- 출처 모름 상태에서 각 시에 점수 부여
- 이후 출처 공개 후 이유 인터뷰

**평가 항목** (1~5점):
| 항목 | 설명 |
|------|------|
| 언어 밀도 | 최소 언어로 최대를 표현하는가 |
| 이미지 선명도 | 이미지가 구체적이고 생생한가 |
| 구조적 필연성 | 행갈이/연갈이가 의도적으로 보이는가 |
| 독창성 | 기존에 본 적 없는 언어 조합인가 |
| 여운 | 읽은 후 무언가가 남는가 |

### B. 일반 독자 평가 (General Reader)
**평가자**: 문학에 관심 있는 일반인
**규모**: 30~50인

**평가 방식**:
- 간단한 좋음/나쁨 투표 + 자유 코멘트
- 어떤 시가 "더 진짜 같은지" 선택

### C. Turing Test 방식
생성시와 인간 시를 섞어, 어느 것이 AI 생성인지 맞히는 실험.
AI 생성시가 인간 시와 구별되지 않는 비율을 측정.

## 평가 지문 설계

```
[시 A]
[원문 텍스트]

[시 B]
[원문 텍스트]

질문:
1. 더 마음에 드는 시를 고르시오.
2. 이 시들 중 AI가 쓴 것이 있다면 어느 것인가?
3. 고른 이유를 간략히 서술하시오.
```

## 비교 대상

| 조건 | 설명 |
|------|------|
| 생성시 (파인튜닝 모델) | 이 프로젝트의 모델 |
| 생성시 (GPT-4o) | 비교 기준 |
| 생성시 (Claude) | 비교 기준 |
| 인간 시 (공개 발표작) | 정답 기준 |
| 인간 시 (비공개 습작) | 하한선 기준 |

## 미결 사항

- 전문가 평가단 섭외 방법 (문학 커뮤니티, 대학 연구자)
- 평가 보상 설계 (원고료 수준)
- 블라인드 완결성 보장 방법 (생성시 표지 제거)
- 평가 결과를 학습 데이터에 피드백하는 방법 (RLHF 유사)
